### Remaining Integration Issues (2)

#### INT-04: QUOTATION_RECEIVED Never Dispatched (non-critical)

- **Type:** `QUOTATION_RECEIVED` defined in `notifications.types.ts:13`, labeled in frontend (`useNotifications.ts:12,29`), displayed in `NotificationBell.tsx:66` and `NotificationPreferencesPage.tsx:35`
- **Missing:** `quotations.service.ts` `registerProposal()` has zero `createNotification` calls
- **Impact:** When a proposal is registered, the quotation manager gets no notification. All other 14 notification types dispatch correctly.
- **Fix:** Add `createNotification(tx, ctx.organizationId, { type: 'QUOTATION_RECEIVED', ... })` in `registerProposal()` after upsert
- **Affects:** DASH-03 (partial — all other notification types work)

#### INT-05: Notification Preferences Channel Casing (bug)

- **Frontend:** `useNotificationPreferences.ts:63` sends `channel: 'badge' | 'email'` (lowercase)
- **Backend:** `notification-preferences.routes.ts:49` checks `channel !== 'BADGE' && channel !== 'EMAIL'` (uppercase)
- **Impact:** Toggle preferences on `/notification-preferences` page fails with 400. GET works fine.
- **Fix:** Either `channel.toUpperCase()` in frontend or lowercase comparison in backend
- **Affects:** DASH-03 (preference management broken, but notification dispatch itself works)
