### Issues Found: 3

#### INT-01: Kanban DnD API Path Mismatch (DASH-01) — NEEDS VERIFICATION

`usePurchasingKanban.ts` embeds `orgId` in API paths for 3 DnD action calls, but backend routes don't have `:orgId`:

| Transition                    | Frontend Call                                              | Backend Route                               | Issue          |
| ----------------------------- | ---------------------------------------------------------- | ------------------------------------------- | -------------- |
| RC_PENDENTE→RC_APROVADA       | `POST /org/${orgId}/purchase-requests/${cardId}/approve`   | No `/approve` route exists                  | 404            |
| RC_APROVADA→EM_COTACAO        | `POST /org/${orgId}/quotations`                            | `POST /org/quotations`                      | orgId mismatch |
| OC_EMITIDA→AGUARDANDO_ENTREGA | `PATCH /org/${orgId}/purchase-orders/${cardId}/transition` | `PATCH /org/purchase-orders/:id/transition` | orgId mismatch |

**Discrepancy:** Phase 13 verifier confirmed OC_EMITIDA→AGUARDANDO_ENTREGA as working. The `api` helper may handle orgId prefix transparently. **Requires browser testing.**

#### INT-02: Notification Preferences Route 404 (DASH-03) — LIKELY BROKEN

`useNotificationPreferences.ts` calls `GET/PUT /org/notification-preferences` but backend registers `/org/:orgId/notification-preferences`. The orgId segment is absent from frontend path → 404.

#### INT-03: DAILY_DIGEST Missing from Frontend Type (DASH-03) — MINOR

Backend dispatches `DAILY_DIGEST` notifications via cron. Frontend `NotificationType` union in `useNotifications.ts` omits it. Bell renders without label.
