  <implementation>
    Create `packages/shared/src/utils/rural-credit-schedule.ts` exporting:

    1. `computeMonthlyRate(annualRate: number): number` — compound rate: `Math.pow(1 + annualRate, 1/12) - 1`

    2. `capitalizeGracePeriod(principal: IMoney, monthlyRate: number, gracePeriodMonths: number): IMoney` — returns `principal.multiply(Math.pow(1 + monthlyRate, gracePeriodMonths))` or original if grace=0

    3. `generateSchedule(input: ScheduleInput): ScheduleRow[]` — main function:
       - Compute monthly rate from annualRate
       - Apply grace capitalization to get adjustedPrincipal
       - Switch on amortizationSystem:
         - **SAC**: basePrincipal = adjustedPrincipal / n (ROUND_DOWN to 2 decimal places via Decimal), residual to installment #1. Interest_k = balance * monthlyRate. Balance decrements by principal_k.
         - **PRICE**: PMT = PV * i * (1+i)^n / ((1+i)^n - 1) via Decimal. Interest_k = balance * monthlyRate. Principal_k = PMT - Interest_k. Last installment adjusts for residual (set principal = remaining balance).
         - **BULLET**: Interest_k = balance * monthlyRate (rows 1..n-1 have principal=0). Row n: principal = full balance, interest = balance * monthlyRate.

    4. `computeDueDate(year, month, dayOfMonth, offsetMonths): Date` — uses Date.UTC + setUTCMonth for offset, then clamps day overflow: if getUTCDate() !== dayOfMonth, set date to 0 (last day of previous month).

    Types exported: `ScheduleInput` (principalAmount, annualRate, termMonths, gracePeriodMonths, firstPaymentYear, firstPaymentMonth, paymentDayOfMonth, amortizationSystem), `ScheduleRow` (installmentNumber, dueDate, principal: IMoney, interest: IMoney, totalPayment: IMoney, outstandingBalance: IMoney).

    Use `Money()` from `@protos-farm/shared` for ALL monetary values. Use `Decimal` from decimal.js for intermediate computations (PMT formula denominator). Never use native JS floating point for monetary amounts.

  </implementation>
</feature>
