    **receivables.service.ts:**

    1. `createReceivable(ctx, input)` — validate costCenterItems. If funruralRate provided, compute funruralAmount = Money(totalAmount).multiply(funruralRate) and store both on Receivable (Pitfall 3 from research: store at creation, not runtime). Generate installments. Prisma tx: Receivable + ReceivableInstallment[] + ReceivableCostCenterItem[].

    2. `listReceivables(ctx, query)` — paginated with filters, include installments and costCenterItems.

    3. `getReceivable(ctx, id)` — with all relations.

    4. `updateReceivable(ctx, id, input)` — only PENDING. Recalculate FUNRURAL if rate or totalAmount changes.

    5. `deleteReceivable(ctx, id)` — CANCELLED status (soft delete).

    6. `settleReceivable(ctx, receivableId, input)` — atomic Prisma tx: update Receivable status to RECEIVED, INCREMENT BankAccountBalance.currentBalance (opposite of payables — money comes IN), create FinancialTransaction type='CREDIT' referenceType='RECEIVABLE'.

    7. `reverseReceivable(ctx, receivableId)` — atomic: reset to PENDING, decrement bank balance, create DEBIT FinancialTransaction "Estorno CR #...".

    8. `renegotiateReceivable(ctx, id, input)` — mark original as RENEGOTIATED, create new Receivable with newDueDate and newAmount (or same amount), link via notes. Original becomes read-only.

    **receivables.routes.ts:**
    Router under `/org/receivables` with `checkPermission('financial', ...)`.
    - GET `/` — list
    - GET `/:id` — get
    - POST `/` — create
    - PUT `/:id` — update
    - DELETE `/:id` — delete (cancel)
    - POST `/:id/settle` — settle
    - POST `/:id/reverse` — reverse
    - POST `/:id/renegotiate` — renegotiate

    Register receivablesRouter in app.ts.
