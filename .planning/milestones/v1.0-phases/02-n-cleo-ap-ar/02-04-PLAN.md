  <action>
    **cnab.adapter.ts:**
    Define interfaces exactly as in Research Pattern 3: CnabHeaderData (companyName, companyDocument, convenioCode, agency, accountNumber, fileDate, sequentialNumber), CnabPaymentRecord, CnabReturnRecord, CnabAdapter interface with generateRemessa240(), generateRemessa400(), parseRetorno(). Export getCnabAdapter(bankCode) registry function and CNAB_ADAPTERS Map.

    **cnab.generator.ts:**
    Shared utility functions for fixed-width formatting: padLeft(value, length, char='0'), padRight(value, length), formatDate(date, format='DDMMYYYY'), formatAmount(cents, length) — amount in centavos zero-padded. These are used by both adapters.

    **cnab.parser.ts:**
    Shared retorno parsing utilities: sliceField(line, start, end) — 1-based positions to 0-based. parseDateFromCnab(str, format). parseAmountFromCnab(str) — centavos to BRL number. detectFormat(content) — returns '240' or '400' based on line length (240 chars vs ~400 chars).

    **bb-001.adapter.ts:**
    Implements CnabAdapter for Banco do Brasil (code '001').
    - generateRemessa240: File Header (record type 0) + Lot Header (type 1) + Detail records (type 3, segment P for payment + segment Q for beneficiary) + Lot Trailer (type 5) + File Trailer (type 9). Each line exactly 240 chars. Follow BB layout manual positions from Research code example.
    - generateRemessa400: Header (record type 0) + Detail (type 1) + Trailer (type 9). Each line exactly 400 chars.
    - parseRetorno: parse return file, extract ourNumber, status (map BB-specific codes: '00'=LIQUIDATED, '02'=RETURNED, etc.), liquidationDate, amountPaid.

    **sicoob-756.adapter.ts:**
    Implements CnabAdapter for Sicoob (code '756').
    - Key difference from BB: positions 53-72 in File Header contain "Banco Cooperado" identification (Pitfall 2 from research). Use Sicoob-specific layout.
    - generateRemessa240, generateRemessa400, parseRetorno — same structure as BB but with Sicoob field positions.

    **Add CNAB routes to payables.routes.ts:**
    - POST `/cnab/remessa` — body: { bankAccountId, format: '240'|'400', payableIds: string[] }. Loads bank account (for CNAB header data including convenioCode/carteira/variacao), loads payables, calls getCnabAdapter(bankCode).generateRemessa{format}(), returns text file with Content-Type: text/plain and Content-Disposition: attachment.
    - POST `/cnab/retorno/preview` — multipart file upload (multer). Parse file with adapter.parseRetorno(), match ourNumber to payableIds, return preview records (matched/unmatched).
    - POST `/cnab/retorno/confirm` — body: { records: { payableId, amountPaid, liquidationDate }[] }. Call settlePayment for each matched record in a single Prisma transaction.

    Register these routes BEFORE /:id routes in payables.routes.ts.

    **cnab.spec.ts:**
    Tests: generate CNAB 240 for BB produces correct line lengths (240 chars each), header has bank code '001' in positions 1-3, detail records have correct segment markers. Parse sample retorno extracts records with correct status mapping. Sicoob header differs in positions 53-72. Round-trip test: generate remessa then parse its structure validates consistent formatting.

Bucket into 7 faixas: vencidas (dueDate < today), 7_dias (0-7 days ahead), 15_dias (8-15), 30_dias (16-30), 60_dias (31-60), 90_dias (61-90), acima_90 (91+). Return: buckets[] with { label, displayLabel, count, totalAmount }, grandTotal, overdueCount. Use Money for totalAmount aggregation.
