## Spacing Scale

Declared values (all multiples of 4px):

| Token | Value               | Usage                                                |
| ----- | ------------------- | ---------------------------------------------------- |
| xs    | 4px (`--space-1`)   | Icon gaps, badge padding, inline row gaps            |
| sm    | 8px (`--space-2`)   | Compact element spacing, table cell padding vertical |
| md    | 16px (`--space-4`)  | Default element spacing, input padding horizontal    |
| lg    | 24px (`--space-6`)  | Section padding, modal padding, card padding         |
| xl    | 32px (`--space-8`)  | Layout gaps between major sections                   |
| 2xl   | 48px (`--space-12`) | Major section breaks, page-level vertical rhythm     |
| 3xl   | 64px (`--space-16`) | Empty state icon size, page-level spacing            |

Exceptions:

- Input padding: 12px vertical (`--space-3`) × 16px horizontal (`--space-4`) — matches existing `fin-dashboard` inputs
- Touch targets minimum: 48×48px on all interactive elements (buttons, row action icons)
- Modal panel max-width: 700px for import/mapping modals (matches PayableModal pattern), 900px for reconciliation line list modal
- Reconciliation line row height: minimum 56px to accommodate inline action buttons at 48px touch target

## Component Inventory

### New Components (to build)

#### ReconciliationPage (`pages/ReconciliationPage.tsx`)

- Layout: full-width page with breadcrumb header, main content area
- Initial state: import history table + "Nova Importação" primary button (top-right)
- Switching state: clicking an import row opens reconciliation session for that import

#### ImportHistoryTable (`components/reconciliation/ImportHistoryTable.tsx`)

- `<table>` with `<th scope="col">`, `<caption>` for accessibility
- Columns: Data, Arquivo, Importado por, Total de linhas, Importadas, Pendentes, Status
- Row click: opens reconciliation session (navigates to `?importId=xxx` query param)
- Empty state: FileSearch icon (48px) + "Nenhum extrato importado" + "Importe o primeiro extrato bancário" + "Nova Importação" CTA button
- Mobile (<768px): cards empilhados com data, arquivo e status visíveis

#### ImportPreviewModal (`components/reconciliation/ImportPreviewModal.tsx`)

- Multi-step modal (2 steps): Step 1 = CSV column mapping, Step 2 = row selection table
- Step 1 (CSV only): auto-suggested column mapping with dropdowns (data, valor, descrição, tipo). OFX skips to step 2.
- Step 2: table with checkboxes — all rows selected by default. Columns: Tipo (CRÉDITO/DÉBITO badge), Data, Valor, Descrição. Footer: selected count + "Importar N linhas" CTA.
- Limit warning: if file exceeds 5.000 linhas, show inline alert before proceeding
- Modal width: 900px
- Stepper: text-based "1. Mapeamento → 2. Seleção" in header

#### ReconciliationLineList (`components/reconciliation/ReconciliationLineList.tsx`)

- Grouped sections by confidence: EXATO → PROVÁVEL → SEM MATCH (collapsible sections with count badge)
- Each row: [Type badge] [Date] [Amount mono] [Bank memo] | [Match description] [Match amount mono] | [Aceitar] [Recusar] [Vincular]
- "Aceitar" button: filled primary green — confirms suggested match
- "Recusar" button: outlined red — rejects suggested match, sets line to PENDENTE
- "Vincular" button: outlined neutral — opens ManualLinkModal
- N:N indicator: when a line has multiple candidates, show "+ N mais" expandable
- Filter bar above list: status dropdown (Todos / Pendente / Conciliado / Ignorado) + search input
- Bottom summary bar: N conciliados, N pendentes, N sem match. "Gerar Relatório" secondary button.

#### ManualLinkModal (`components/reconciliation/ManualLinkModal.tsx`)

- Search input (autofocus) with debounce 300ms — searches CP + CR + Transferências
- Results list with type tag, description, date, amount (mono), status
- Multi-select: checkboxes for N:N reconciliation. Running sum shown below list.
- Sum validation: shows error in red if selected sum ≠ statement line amount
- "Vincular selecionados" CTA button (disabled until sum matches)
- Modal width: 700px

#### CashflowPage (`pages/CashflowPage.tsx`)

- Layout: full-width page with breadcrumb, farm filter dropdown (local state, "Todas as fazendas" default), export buttons (PDF, Excel) top-right
- Negative balance alert banner: appears at top of content when projection has any date with saldo < 0. Uses `--color-error-100` bg, `--color-error-500` icon+text, AlertCircle icon (20px). Copy: "Saldo negativo previsto em [data]. Revise seus compromissos financeiros."
- Loading state: skeleton screen — chart skeleton (rect 100%×320px) + table skeleton (3 rows)

#### CashflowChart (`components/cashflow/CashflowChart.tsx`)

- Recharts `ComposedChart` inside `ResponsiveContainer` (100% width, 320px height)
- Area: Realista (solid green fill, `--color-primary-600` stroke, `--color-primary-100` fill opacity 0.4)
- Line: Otimista (dashed, `--color-success-500`, no dots)
- Line: Pessimista (dashed, `--color-error-500`, no dots)
- ReferenceLine y=0: `--color-error-500`, strokeWidth 2
- XAxis: monthly labels (Jan, Fev, ...), `--font-body` 14px (`--text-sm`)
- YAxis: `formatBRLCompact` (e.g. "R$ 12k"), `--font-mono` 14px (`--text-sm`)
- CartesianGrid: `strokeDasharray="3 3"`, `--color-neutral-200`
- Tooltip: custom `CashflowTooltip` component — shows date, Saldo realista, Entradas previstas, Saídas previstas, Cheques pendentes — all values in `--font-mono`
- Legend: bottom, text labels "Realista", "Otimista", "Pessimista"
- Negative balance zone: chart area below y=0 visually indicates risk — achieved through ReferenceLine + pessimistic line color

#### DfcTable (`components/cashflow/DfcTable.tsx`)

- Three expandable sections: Operacional, Investimento, Financiamento
- Each section: category rows with 12-month columns (Jan–Dez) + Total column
- Values: JetBrains Mono, 14px. Positive in `--color-neutral-700`, negative in `--color-error-500`
- Mobile (<768px): horizontal scroll, sticky first column (category name)
- Collapsible: click section header to collapse/expand. Default: all expanded.

#### Dashboard alert card (addition to FinancialDashboardPage.tsx)

- New card in existing alerts section: "Saldo negativo previsto"
- Card: `--color-error-100` background, AlertCircle icon 20px in `--color-error-500`, heading "Saldo negativo previsto", body "em [data] — R$ [valor]" (mono), Link to `/cashflow`
- Only rendered when backend returns a `negativeBalanceDate` in the dashboard response

## Interaction Contract

### File Upload Flow (ImportPreviewModal)

1. User clicks "Nova Importação" → modal opens, Step 1 active
2. Drag-and-drop area or "Selecionar arquivo" button — accepts `.ofx`, `.csv`
3. On file select: show file name, size, type badge. "Avançar" button becomes enabled.
4. Backend detects file type. OFX: auto-detects bank account (shows name or "Conta não identificada" + dropdown fallback). CSV: shows column mapping UI.
5. CSV mapping: dropdowns pre-filled with auto-detected columns. User can override. "Avançar" enabled when data/valor/descrição columns are mapped.
6. Step 2: preview table loads. All rows checked. User can uncheck rows. Running count in footer.
7. "Importar N linhas" → POST → loading state on button (spinner icon, disabled). On success: modal closes, history table refreshes, toast "X linhas importadas. Y duplicatas ignoradas."
8. On error: error message inline below upload area. Button re-enables.

### Reconciliation Actions

- "Aceitar": immediate optimistic update — row moves to CONCILIADO section, summary counter updates. Toast: "Lançamento conciliado."
- "Recusar": row moves to PENDENTE group, match candidate removed. Toast: "Match recusado. Linha permanece pendente."
- "Vincular" (ManualLinkModal): on confirm, same optimistic update as Aceitar. Toast: "Vínculo manual criado."
- N:N: selecting multiple candidates shows running sum. Sum mismatch shows red inline: "Soma selecionada (R$ X) ≠ valor do extrato (R$ Y)". CTA disabled.

### Cashflow Page Load

1. Page mounts → skeleton screens displayed immediately
2. Backend computes projection on demand (no cache) — expect 1–3s
3. On data load: skeleton replaced with chart + DFC table
4. Farm filter change → skeleton + reload (no stale data shown)
5. Negative balance alert banner appears above chart if any projected date < 0

### Export Actions

- "Exportar PDF" / "Exportar Excel" buttons: secondary outlined buttons, top-right of each page
- Click → loading indicator on button (spinner, disabled state, 200ms delay before showing)
- File download triggered by `Content-Disposition: attachment` response header
- On error: toast "Não foi possível exportar. Tente novamente."

### Destructive: Ignorar linha do extrato

- "Ignorar" action on SEM MATCH lines (via overflow menu `...`)
- Confirmation: inline confirmation row replaces action buttons — "Confirmar ignorar esta linha? [Sim] [Cancelar]" — proportional risk, no separate modal needed
- On confirm: line marked IGNORED, removed from active list, summary updates

## Accessibility Contract

- All tables: `<table>` with `<th scope="col">` headers and `<caption>` (visually hidden ok)
- Confidence badges: never rely on color alone — text label always present
- Chart: add `aria-label` to `ResponsiveContainer` wrapper: `aria-label="Gráfico de projeção de fluxo de caixa por cenário"`; chart data also available in DFC table for non-visual access
- Chart tooltips: not keyboard-accessible (Recharts limitation) — DFC table provides full data access
- Action buttons in reconciliation rows: `aria-label="Aceitar match para {memo} em {date}"`, `aria-label="Recusar match para {memo} em {date}"`, `aria-label="Vincular manualmente {memo} em {date}"`
- File upload: `<input type="file" accept=".ofx,.csv">` with visible `<label>` — never placeholder-only
- Modal focus trap: focus moves to first interactive element on open, returns to trigger on close. Escape closes modal.
- Alert banner: `role="alert"` on negative balance banner so screen readers announce it on page load
- Status badges with icon: icon is `aria-hidden="true"`, text label is the status text
- Form errors: `role="alert"` + `aria-live="polite"` on inline error messages
- Focus visible: `outline: 2px solid var(--color-primary-500); outline-offset: 2px` — never removed
