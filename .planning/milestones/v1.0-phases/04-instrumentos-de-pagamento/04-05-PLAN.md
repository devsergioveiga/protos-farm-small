```

EMITIDO -> A_COMPENSAR (deliver/confirm)
A_COMPENSAR -> COMPENSADO (compensate: updates BankAccountBalance)
A_COMPENSAR -> DEVOLVIDO (returned by bank)
EMITIDO|A_COMPENSAR -> CANCELADO
DEVOLVIDO -> A_COMPENSAR (re-present)

````

  type: string;
  status: string;

  amount: number;
  bankAccountId: string;
  bankAccountName: string;
  issueDate: string;
  deliveryDate: string | null;
  expectedCompensationDate: string | null;
  compensationDate: string | null;
  payeeName: string;
  description: string | null;
  notes: string | null;
  createdAt: string;
}

// Valid state transitions map
export const VALID_TRANSITIONS: Record<string, string[]> = {
  EMITIDO: ['A_COMPENSAR', 'CANCELADO'],
  A_COMPENSAR: ['COMPENSADO', 'DEVOLVIDO', 'CANCELADO'],
  DEVOLVIDO: ['A_COMPENSAR', 'CANCELADO'],
  COMPENSADO: [],
  CANCELADO: [],
};

2. **Create checks.service.ts** with functions:

- **`createCheck(ctx, input)`**: Validate bankAccountId exists and isActive. Create Check record with status EMITIDO. Return CheckOutput.

- **`listChecks(ctx, query)`**: List with optional filters (status, type, date range on issueDate). Include bankAccount name. Return CheckOutput[].

- **`getCheck(ctx, checkId)`**: Find by id + organizationId. Return CheckOutput or throw 404.

- **`markACompensar(ctx, checkId)`**: Validate current status is EMITIDO (use VALID_TRANSITIONS). Update status to A_COMPENSAR. Optionally set deliveryDate if not already set. Return updated check.

- **`compensateCheck(ctx, checkId, compensationDate?)`**: Inside withRlsContext: validate status is A_COMPENSAR. Determine transaction type: EMITIDO -> DEBIT, RECEBIDO -> CREDIT. Determine balance operation: EMITIDO -> decrement, RECEBIDO -> increment. Create FinancialTransaction with referenceType 'CHECK_COMPENSATION', referenceId = check.id. Update BankAccountBalance atomically. Update check: status=COMPENSADO, compensationDate=compensationDate || new Date(). Return updated check.

- **`returnCheck(ctx, checkId)`**: Validate status is A_COMPENSAR. Update status to DEVOLVIDO. Return updated check.

- **`resubmitCheck(ctx, checkId, expectedCompensationDate?)`**: Validate status is DEVOLVIDO. Update status to A_COMPENSAR, set new expectedCompensationDate if provided. Return updated check.

- **`cancelCheck(ctx, checkId)`**: Validate current status allows CANCELADO transition (EMITIDO, A_COMPENSAR, DEVOLVIDO). Update status to CANCELADO. Return updated check.

- **`getAlertCount(ctx)`**: Count checks with status IN ['A_COMPENSAR', 'DEVOLVIDO']. Return `{ count: number }`.

- **`getAccountingBalanceData(ctx, farmId?)`**: Sum amount of checks with status A_COMPENSAR grouped by type. Returns `{ pendingEmitidos: number, pendingRecebidos: number }`. Used by dashboard endpoint. Filter by farmId if provided (via bankAccount.farms relation).

Helper **`validateTransition(currentStatus, targetStatus)`**: Check VALID_TRANSITIONS map. If invalid, throw CheckError('Transicao de status invalida', 422).
