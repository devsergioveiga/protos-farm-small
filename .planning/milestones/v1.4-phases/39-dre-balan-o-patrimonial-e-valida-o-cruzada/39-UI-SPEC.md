## Spacing Scale

Declared values (4px base scale, from `tokens.css`):

| Token | Value               | Usage                                                           |
| ----- | ------------------- | --------------------------------------------------------------- |
| xs    | 4px (`--space-1`)   | Icon gaps, inline label padding, cell internal gaps             |
| sm    | 8px (`--space-2`)   | Compact element spacing, breadcrumb gaps                        |
| md    | 16px (`--space-4`)  | Default element spacing, filter group gap, table cell padding-x |
| lg    | 24px (`--space-6`)  | Page padding, section padding, card padding                     |
| xl    | 32px (`--space-8`)  | Layout gaps between major page sections                         |
| 2xl   | 48px (`--space-12`) | Major section breaks (indicator cards row to table)             |
| 3xl   | 64px (`--space-16`) | Page-level top padding (empty state vertical centering)         |

Exceptions:

- Table cell padding-y: 12px (`--space-3`) — financial tables require denser rows
- Input/select padding: 12px vertical, 16px horizontal — CLAUDE.md minimum for touch
- Touch targets minimum: 48x48px — all filter dropdowns, toggle button, "Investigar" button
- Indicator card sparkline area height: 48px — fixed for recharts sparkline

Source: `tokens.css` spacing scale + CLAUDE.md input padding rules.

## Component Inventory

### 1. DRE Table (`DreTable.tsx`)

A `<table>` with hierarchical rows for the 10 DRE sections.

**Structure:**

- `<caption>`: "Demonstração do Resultado do Exercício — {Mês} {Ano}"
- `<thead>`: columns defined below
- `<tbody>`: rows with 3 visual levels
- `<tfoot>`: Resultado Líquido total row (bold, top border 2px)

**Columns (base — V/H toggle OFF):**

| #   | Header                 | Content                     | Alignment |
| --- | ---------------------- | --------------------------- | --------- |
| 1   | Descrição              | Section name / account name | Left      |
| 2   | Mês Atual              | R$ amount, JetBrains Mono   | Right     |
| 3   | Acumulado              | R$ amount, JetBrains Mono   | Right     |
| 4   | Mesmo Período Ano Ant. | R$ amount, JetBrains Mono   | Right     |

**Extra columns when V/H toggle is ON (appended right):**

| #   | Header | Content                                                    | Alignment                    |
| --- | ------ | ---------------------------------------------------------- | ---------------------------- |
| 5   | % AV   | Vertical analysis percentage                               | Right, `--color-neutral-600` |
| 6   | Δ% AH  | Horizontal delta percentage — green positive, red negative | Right                        |

**Row visual hierarchy:**

- Section header row (e.g., "Receita Operacional Bruta"): Source Sans 3 14px weight 600, `background: var(--color-neutral-50)`, left-padding 16px
- Sub-account row (analytic accounts): Source Sans 3 14px weight 400, `background: var(--color-neutral-0)`, left-padding 32px (level-based, 16px per level)
- Subtotal row (Receita Líquida, Lucro Bruto, etc.): Source Sans 3 14px weight 600, `background: var(--color-neutral-100)`, top border `1px solid var(--color-neutral-200)`
- CPC 29 section row: same as section header but with `--color-sun-100` (#FFF9C4) background tint — signals special accounting treatment

**States:**

- Loading: skeleton with 12 rows (alternating 2 indent depths), pulse animation
- Empty (no fiscal year selected): empty state — TrendingUp icon 48px + "Selecione o exercício fiscal e o período para gerar a DRE." + no CTA
- Empty (period has no data): TrendingUp icon 48px + "Nenhum lançamento no período selecionado." + no CTA

### 2. Margin Ranking Chart (`MarginRankingChart.tsx`)

Recharts horizontal `BarChart`. Only rendered when cost center filter = "Consolidado".

**Dimensions:** 100% width, 240px height (5-10 bars typical)

**Data:** Top cost centers sorted by gross margin descending. Each bar = 1 cost center.

**Bar configuration:**

- X axis: percentage values (0–100%)
- Y axis: cost center names, Source Sans 3 12px
- Bar fill: `var(--color-primary-400)` (#66BB6A)
- Bar label: margin % inside bar, white, JetBrains Mono 12px
- Tooltip: "{CC name}: Receita R$ {X} | CPV R$ {X} | Margem {X}%"

**Section wrapper:** white card with 16px padding, 8px radius, `--shadow-sm`, heading "Ranking por Margem Bruta" (DM Sans 16px 500)

### 3. Indicator Card (`IndicatorCard.tsx`)

Six cards in a responsive grid on the Balance Sheet page. Each card shows one financial ratio.

**Card anatomy:**

- Label row: ALL CAPS label (e.g., "LIQUIDEZ CORRENTE"), Source Sans 3 12px 600, `--color-neutral-500`
- Value row: ratio value, JetBrains Mono 24px 400, `--color-neutral-800` — large and readable
- Sparkline row: recharts `LineChart` with `width={120} height={48}`, no axes, no tooltip. Line color `--color-primary-500`. Stroke width 2.
- Tooltip on card hover (CSS title attribute): formula description (e.g., "AC / PC")

**Grid:** CSS grid `repeat(auto-fill, minmax(180px, 1fr))`, gap 16px. Minimum 3 per row on desktop.

**Card style:** white surface, 16px padding, `--radius-md` (8px), `--shadow-sm`, `border: 1px solid var(--color-neutral-200)`

**The 6 indicators and their labels:**

| Label                    | Formula              | Unit                      |
| ------------------------ | -------------------- | ------------------------- |
| LIQUIDEZ CORRENTE        | AC / PC              | ratio (e.g., 1.85)        |
| LIQUIDEZ SECA            | (AC − Estoques) / PC | ratio                     |
| ENDIVIDAMENTO GERAL      | PE / AT              | % (e.g., 42%)             |
| COMPOSIÇÃO ENDIVIDAMENTO | PC / PE              | %                         |
| ROE                      | RL / PL              | %                         |
| PL POR HECTARE           | PL / área total ha   | R$/ha (e.g., R$ 1.240/ha) |

Source: CONTEXT.md D-09, REQUIREMENTS.md BP-02.

### 4. Balance Sheet Table (`BalanceSheetTable.tsx`)

A `<table>` with rural classification (AC, ANC, PC, PNC, PL).

**Columns:**

| #   | Header         | Content                   |
| --- | -------------- | ------------------------- |
| 1   | Descrição      | Group / account name      |
| 2   | Saldo Atual    | R$ amount, JetBrains Mono |
| 3   | Saldo Anterior | R$ amount, JetBrains Mono |

**Row hierarchy:** Same visual pattern as DRE table — group header rows (`--color-neutral-50` bg, 14px 600), account rows (white bg, indented, 14px 400), subtotal rows (`--color-neutral-100` bg, 14px 600).

**Two-column layout:** Ativo (groups 1.1 AC + 1.2 ANC) on the left half, Passivo + PL (groups 2.1 PC + 2.2 PNC + 3 PL) on the right half. On screens < 1024px: single column, Ativo first then Passivo+PL, with visible section divider (`<hr>` with `--color-neutral-200`).

### 5. Invariant Card (`InvariantCard.tsx`)

Four cards in a 2×2 grid. Each card represents one cross-validation invariant.

**Card anatomy (PASSED state):**

- Status icon: CheckCircle 24px, `--color-success-500`
- Title: invariant name, Source Sans 3 14px 600, `--color-neutral-800`
- Detail row: "Esperado: {value} | Encontrado: {value}" — Source Sans 3 13px 400, `--color-neutral-600`, JetBrains Mono for values
- Background: `--color-success-100` (#E8F5E9)
- Border: `1px solid var(--color-success-500)` at 30% opacity

**Card anatomy (FAILED state):**

- Status icon: XCircle 24px, `--color-error-500`
- Title: invariant name, Source Sans 3 14px 600, `--color-neutral-800`
- Difference row: "Diferença: R$ {X}" — JetBrains Mono 13px, `--color-error-500`, `font-weight: 600`
- "Investigar" button: secondary button (outline), 36px height, `--color-error-500` border + text, ArrowRight icon 16px
- Background: `--color-error-100` (#FFEBEE)
- Border: `1px solid var(--color-error-500)` at 30% opacity

**Card anatomy (PENDING — DFC placeholder):**

- Status icon: Clock 24px, `--color-neutral-400`
- Title: "Variação Caixa DFC ↔ BP", Source Sans 3 14px 400 (dimmed), `--color-neutral-500`
- Body text: "Aguardando DFC (Phase 40)" — italic, Source Sans 3 13px, `--color-neutral-400`
- Background: `--color-neutral-100` (#F5F3EF)
- Border: `1px dashed var(--color-neutral-300)`
- Entire card `opacity: 0.7`

**Grid:** `display: grid; grid-template-columns: 1fr 1fr; gap: 16px`. On screens < 768px: single column.

## Interaction Contracts

### V/H Toggle (DRE page)

- Default state: OFF — table has 4 columns (Descrição + 3 period columns)
- ON state: table adds 2 columns (AV% and ΔAH%), total 6 columns
- Toggle is a `<button type="button">` with `aria-pressed` attribute (`"false"` / `"true"`)
- Label text changes: "Análise V/H" + ToggleLeft icon (OFF) → "Análise V/H" + ToggleRight icon (ON)
- Transition: columns appear with `opacity 0 → 1` over 200ms, no layout shift (columns pre-allocated via `col-width`)

### Cost Center Filter (DRE page)

- Default: "Consolidado (todas as fazendas)"
- When CC selected: Margin Ranking Chart section disappears with `display: none` (not hidden, removed from flow)
- Filter change triggers new API call; table shows skeleton while loading
- Filter applies only to DRE page — BP page has no CC filter (BP is always consolidated)

### "Investigar" Button (Cross-Validation page)

- Appears only on FAILED invariant cards
- Opens Razão Contábil (`/ledger`) or Balancete (`/trial-balance`) in a new browser tab (`target="_blank"`)
- Specific routing per invariant:
  - Invariant 1 (RL DRE vs Delta PL): links to Razão filtered on "Lucros Acumulados" account
  - Invariant 3 (AT = PT + PL): links to Balancete with fiscal year + month pre-selected
  - Invariant 4 (Débitos = Créditos): links to Balancete with fiscal year + month pre-selected
- Button is `aria-label="Investigar divergência em {invariant name}"` (screen reader context)

### Period Selectors (all 3 pages)

- Both Exercício Fiscal and Mês must be selected before API call fires
- If only one is selected: no fetch, empty state persists
- Exercício Fiscal dropdown: populated from `useFiscalYears()` existing hook
- Mês dropdown: static list Janeiro–Dezembro (same as TrialBalancePage)
- Both selects have `min-width: 160px`, `flex: 1` within filter bar

### Sparklines (BP page — IndicatorCard)

- Data source: 6 most recent months of the calculated indicator values
- Recharts `LineChart`, `width={120} height={48}`, no axes, no tooltip, no legend
- `dot={false}` — clean trend line only
- Line color: `var(--color-primary-500)` (#388E3C)
- If fewer than 2 data points: sparkline area shows "--" text instead of chart

### Loading States (skeleton screens, never spinner)

| Page section           | Skeleton description                                         |
| ---------------------- | ------------------------------------------------------------ |
| DRE table              | 12 skeleton rows, alternating indent levels, pulse animation |
| BP indicator cards     | 6 skeleton cards (same grid), gray rectangle 80px tall each  |
| BP balance sheet table | 10 skeleton rows                                             |
| Cross-validation cards | 4 skeleton cards in 2×2 grid, gray rectangles                |

## Accessibility Contract

- Every `<table>` has `<caption>` with period context
- Column headers use `<th scope="col">`, row group headers use `<th scope="row">`
- All filter `<select>` elements have associated `<label htmlFor>`
- V/H toggle: `<button aria-pressed="true|false">` — state announced to screen readers
- Invariant card status icons: `aria-hidden="true"` — status communicated by card text
- "Investigar" button: `aria-label` includes invariant name for context
- Focus visible: `outline: 2px solid var(--color-primary-500); outline-offset: 2px` (global rule, do not override)
- Error states: toast uses `role="status" aria-live="polite"` for informational, `role="alert" aria-live="assertive"` for errors
- Sparklines: `aria-hidden="true"` on the recharts SVG — the numeric value on the card is the accessible value
- Color-only rule: invariant card status communicated via icon (CheckCircle/XCircle/Clock) + color + text simultaneously
- `prefers-reduced-motion`: `@media (prefers-reduced-motion: reduce)` — disable pulse skeleton animation and column fade-in transitions

## Implementation Notes for Executor

1. **CSS file per page:** `DrePage.css`, `BalanceSheetPage.css`, `CrossValidationPage.css`. Follow exact pattern of `TrialBalancePage.css` — BEM-style class names, CSS custom properties only, no hardcoded hex.

2. **Shared components live in:** `apps/frontend/src/components/financial-statements/` — `IndicatorCard.tsx`, `DreTable.tsx`, `MarginRankingChart.tsx`, `InvariantCard.tsx`. Each has its own `.css` file.

3. **Recharts sparkline import:** `import { LineChart, Line, ResponsiveContainer } from 'recharts'` — wrap in `<ResponsiveContainer width="100%" height={48}>` to avoid SSR issues.

4. **Number formatting:** All monetary values use `Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' })`. Percentages use `Intl.NumberFormat('pt-BR', { style: 'percent', minimumFractionDigits: 1 })`.

5. **Negative values in DRE:** Deductions and expenses are shown as positive numbers with context (section label communicates sign). No red parenthetical notation — keep the table clean.

6. **Two-column BP layout:** Use `display: grid; grid-template-columns: 1fr 1fr; gap: 32px` for ≥1024px. Single column below 1024px.

7. **Sidebar group:** CONTABILIDADE already exists. Add 3 new `<li>` links: "DRE" (TrendingUp 20px), "Balanço Patrimonial" (Scale 20px), "Validação Cruzada" (GitMerge 20px). Follow existing sidebar item class pattern.
