    **chart-of-accounts.types.ts:**
    - Export `ChartOfAccountNode` interface: `{ id, code, name, parentId, level, accountType, nature, isSynthetic, allowManualEntry, isActive, isFairValueAdj, spedRefCode, children?: ChartOfAccountNode[] }`
    - Export `CreateAccountInput`: `{ code, name, parentId?, accountType, nature, isSynthetic?, allowManualEntry?, isFairValueAdj?, spedRefCode? }`
    - Export `UpdateAccountInput`: `Partial<CreateAccountInput> & { isActive?: boolean }`
    - Export `ChartOfAccountOutput`: flat fields from Prisma model
    - Export `ChartOfAccountError` class extending Error with `code` field (e.g. 'DUPLICATE_CODE', 'HAS_CHILDREN', 'SYNTHETIC_MANUAL_ENTRY', 'PARENT_NOT_FOUND', 'MAX_DEPTH_EXCEEDED')

    **chart-of-accounts.service.ts:**
    - `getAccountTree(prisma, organizationId)` — Use `prisma.$queryRaw` with `WITH RECURSIVE coa_tree AS (...)` CTE. Select all columns. Order by `code`. Return flat array ordered by code (frontend builds tree from parentId). Annotate return type as `Promise<ChartOfAccountNode[]>`.
    - `createAccount(prisma, organizationId, input: CreateAccountInput)` — Validate: (a) if parentId provided, parent must exist in same org and level must be parent.level + 1; (b) compute `level` from `code.split('.').length`; (c) if `isSynthetic === true`, force `allowManualEntry = false`; (d) level must be <= 5; (e) check code uniqueness (catch Prisma P2002 unique violation). Use `prisma.chartOfAccount.create()`.
    - `updateAccount(prisma, organizationId, id, input: UpdateAccountInput)` — Find account first. If setting `isActive: false`, check no active children exist. If setting `isSynthetic: true`, force `allowManualEntry = false`. Cannot change `accountType` or `nature` if AccountBalance rows exist (future guard — check for linked rows).
    - `deactivateAccount(prisma, organizationId, id)` — Checks: (a) no active children; (b) no AccountBalance rows linked. Sets `isActive: false`.
    - `getUnmappedSpedAccounts(prisma, organizationId)` — `prisma.chartOfAccount.findMany({ where: { organizationId, isSynthetic: false, isActive: true, spedRefCode: null } })`. Returns flat list of analytic accounts without SPED mapping.
    - `getAccountById(prisma, organizationId, id)` — Single account lookup.

    **chart-of-accounts.routes.ts:**
    - Import `requireAuth` and `requireRole` middleware from auth module (follow existing pattern).
    - `GET /api/org/:orgId/chart-of-accounts` — calls `getAccountTree`. Returns 200 with array.
    - `POST /api/org/:orgId/chart-of-accounts` — calls `createAccount`. Returns 201.
    - `PUT /api/org/:orgId/chart-of-accounts/:id` — calls `updateAccount`. Returns 200.
    - `DELETE /api/org/:orgId/chart-of-accounts/:id` — calls `deactivateAccount` (soft delete). Returns 200.
    - `GET /api/org/:orgId/chart-of-accounts/unmapped-sped` — calls `getUnmappedSpedAccounts`. Returns 200.
    - `POST /api/org/:orgId/chart-of-accounts/seed` — calls `seedRuralTemplate` (Task 2). Returns 201.
    - All routes use `req.params.orgId as string` and `req.params.id as string` per CLAUDE.md Express 5 rules.
    - Error handling: catch `ChartOfAccountError` and map to appropriate HTTP status (409 for DUPLICATE_CODE, 422 for SYNTHETIC_MANUAL_ENTRY/MAX_DEPTH_EXCEEDED, 404 for PARENT_NOT_FOUND, 409 for HAS_CHILDREN).

    - Mount: `app.use('/api/org/:orgId/chart-of-accounts', requireAuth, chartOfAccountsRouter);` following the same pattern as existing module mounts.
