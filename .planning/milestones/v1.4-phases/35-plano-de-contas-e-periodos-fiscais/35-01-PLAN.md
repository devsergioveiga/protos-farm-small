    - Throws when items array is empty
    - Throws when percentages do not sum to 100 (e.g. 50+40=90)

    - Fields: `id` (uuid PK), `organizationId` (String), `parentId` (String?), `code` (String), `name` (String), `accountType` (AccountType), `nature` (AccountNature), `isSynthetic` (Boolean default false), `allowManualEntry` (Boolean default true), `isActive` (Boolean default true), `isFairValueAdj` (Boolean default false), `spedRefCode` (String?), `level` (Int), `createdAt`, `updatedAt`
    - Self-relation named `"CoaTree"`: `parent ChartOfAccount? @relation("CoaTree", fields: [parentId], references: [id])` and `children ChartOfAccount[] @relation("CoaTree")`
    - Relation to Organization
    - `@@unique([organizationId, code])`
    - `@@index([organizationId, accountType])`
    - `@@index([parentId])`
    - `@@map("chart_of_accounts")`

    3. **FiscalYear model**:
    - Fields: `id` (uuid PK), `organizationId`, `name` (String, e.g. "2026" or "Safra 2025/2026"), `startDate` (DateTime @db.Date), `endDate` (DateTime @db.Date), `isActive` (Boolean default true), `createdAt`, `updatedAt`
    - Relation to Organization
    - Relations: `accountingPeriods AccountingPeriod[]`, `accountBalances AccountBalance[]`
    - `@@unique([organizationId, startDate])`
    - `@@index([organizationId])`
    - `@@map("fiscal_years")`

    4. **AccountingPeriod model**:
    - Fields: `id` (uuid PK), `organizationId`, `fiscalYearId`, `month` (Int), `year` (Int), `status` (PeriodStatus default OPEN), `openedAt` (DateTime default now()), `closedAt` (DateTime?), `closedBy` (String?), `reopenedAt` (DateTime?), `reopenedBy` (String?), `reopenReason` (String?), `createdAt`, `updatedAt`
    - Relations to Organization, FiscalYear, User (closedByUser via "PeriodClose", reopenedByUser via "PeriodReopen")
    - `@@unique([organizationId, fiscalYearId, month, year])`
    - `@@index([organizationId, status])`
    - `@@map("accounting_periods")`

    5. **AccountBalance model**:
    - Fields: `id` (uuid PK), `organizationId`, `accountId`, `fiscalYearId`, `month` (Int), `openingBalance` (Decimal(14,2) default 0), `debitTotal` (Decimal(14,2) default 0), `creditTotal` (Decimal(14,2) default 0), `closingBalance` (Decimal(14,2) default 0), `costCenterId` (String?), `updatedAt`
    - Relations to Organization, ChartOfAccount, FiscalYear, CostCenter (optional)
    - `@@unique([organizationId, accountId, fiscalYearId, month])` (NOTE: if costCenterId is part of the granularity, use `@@unique([organizationId, accountId, fiscalYearId, month, costCenterId])` instead — but per research, cost center is optional dimension so use the simpler unique without costCenterId and handle cost-center granularity at query time)
    - `@@index([organizationId, fiscalYearId])`
    - `@@map("account_balances")`

    - On `User`: add `closedPeriods AccountingPeriod[] @relation("PeriodClose")`, `reopenedPeriods AccountingPeriod[] @relation("PeriodReopen")`
    - On `CostCenter`: add `accountBalances AccountBalance[]`
