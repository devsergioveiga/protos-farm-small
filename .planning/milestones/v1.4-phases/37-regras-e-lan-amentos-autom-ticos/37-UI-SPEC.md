#### `AccountingRuleModal.tsx` + `AccountingRuleModal.css`

Location: `apps/frontend/src/components/accounting/`

Purpose: CRUD modal for AccountingRule configuration (D-42).

Structure:
- Modal header: rule sourceType label (readonly), `isActive` toggle switch
- Modal body (scrollable):
  - Section "Linhas de Lançamento": editable table with columns — Ordem | Tipo (Débito/Crédito) | Conta (searchable select, analytic accounts only) | Centro de Custo Obrigatório (checkbox)
  - Add row button below table: "Adicionar linha"
  - Section "Histórico": textarea for `{{placeholder}}` template with inline helper showing available variables for that sourceType
  - Section "Configuração": `isActive` repeated as labeled toggle (redundant but scan-friendly for long modals)
- Modal footer: "Cancelar" (secondary) | "Pré-visualizar" (secondary, calls preview endpoint) | "Salvar" (primary, disabled if no lines defined)

Preview panel: when "Pré-visualizar" is clicked, renders an inline read-only JournalEntry card below footer showing the resolved template with real data from the last operation of that sourceType. Card uses `--color-neutral-100` background. "Pré-visualizar" button shows spinner while loading.

Destructive: no delete action on rules — only `isActive` toggle (rules are organizational config, not user data).

#### `PendingPostingsTab.tsx` + `PendingPostingsTab.css`

Location: `apps/frontend/src/components/accounting/`

Purpose: Tab content for the "Pendências" tab in JournalEntriesPage (D-35).

Structure:
- Filter bar (horizontal, `--color-neutral-100` background, `--space-3` padding):
  - Select: status filter (Todos / PENDING / PROCESSING / COMPLETED / ERROR)
  - Select: tipo de operação (sourceType enum values, pt-BR labels)
  - Button: "Retry em Lote" — primary, visible only when filtered to ERROR/PENDING rows
- Summary chips (inline, below filter bar): "N com erro" (red chip) + "N pendentes" (yellow chip) — per D-38. Hidden when both counts are 0.
- Table with accordion rows (D-37):

## Interaction Contracts

### Tab Navigation

- Tab switching: immediate (no loading), active tab stores in URL query param `?tab=lancamentos|pendencias|regras` for deep-linkability
- Navigating to Pendências from a COMPLETED posting ("Ver lançamento") updates active tab + applies `?tab=lancamentos&sourceId={id}` filter
- Keyboard: tabs navigable with arrow keys, `role="tablist"` + `role="tab"` + `aria-selected`

### Accordion Row (ERROR postings)

- Click anywhere on ERROR row expands accordion — chevron rotates 180° on expand (`--duration-fast: 200ms`, `ease-out`)
- Accordion content: `max-height` transition from 0 to auto using measured height — respects `prefers-reduced-motion` (instant expand if reduced motion)
- Keyboard: `Enter` or `Space` on row triggers expand; focus moves into expanded content

### Retry Button

- Individual retry: `RefreshCw` icon (16px) + label "Tentar novamente" — secondary button style
- After retry click: button enters loading state (`aria-busy="true"`, spinner replaces icon, label "Tentando..."), row status badge updates optimistically to PROCESSING
- On success: row badge transitions to COMPLETED, retry button removed, "Ver lançamento" appears
- On failure: row badge returns to ERROR, error message in accordion updates with new error text, toast: "Não foi possível reprocessar. Verifique a mensagem de erro."
- Batch retry: confirmation NOT required (non-destructive operation) — button triggers immediately, count badge updates

### Rule isActive Toggle

- Toggle switch component: 40×22px, thumb slides left (inactive, `--color-neutral-300`) / right (active, `--color-primary-600`)
- On toggle: optimistic update, PATCH request. On error: revert + toast "Não foi possível alterar o status da regra."
- Changing from active to inactive: no confirmation needed (non-destructive — just stops future auto-posting, per D-09)

### AccountingRuleModal — Line Table

- Add line: appends new row with empty selects; focus moves to Tipo select of new row
- Remove line: `X` button on each row (except when only 1 line remains — minimum 1 line enforced)
- Account select: searchable combobox — type to filter analytic accounts by name or code; shows code + name; keyboard navigable
- Validation: "Salvar" disabled if: (a) any line has empty conta, or (b) total lines < 1
- Validation inline: on blur of conta select, shows "Selecione uma conta analítica" if empty

### Preview Panel in AccountingRuleModal

- Shows after "Pré-visualizar" click — not visible by default (saves vertical space)
- Loading state: skeleton pulse (3 lines)
- When no operations exist for that sourceType: info card "Nenhuma operação deste tipo encontrada. O preview usa dados de exemplo."
- Preview card shows: data, histórico (resolved), linhas (conta + valor + D/C), CC

### Origin Link Column (Pendências tab)

- Renders as `<a>` tag navigating to the source module page
- Source type to route mapping (Claude's discretion — sensible defaults):
  - `PAYABLE_SETTLEMENT` → `/payables?id={sourceId}`
  - `RECEIVABLE_SETTLEMENT` → `/receivables?id={sourceId}`
  - `PAYROLL_RUN` → `/payroll-runs?id={sourceId}`
  - `DEPRECIATION_RUN` → `/depreciation?id={sourceId}`
  - `STOCK_ENTRY` → `/stock-entries?id={sourceId}`
  - `STOCK_OUTPUT_CONSUMPTION` / `STOCK_OUTPUT_TRANSFER` / `STOCK_OUTPUT_DISPOSAL` → `/stock-outputs?id={sourceId}`
  - `PAYROLL_PROVISION_VACATION` / `PAYROLL_PROVISION_13TH` → `/payroll-provisions?id={sourceId}`
- Opens in same tab (not new tab) — standard SPA navigation

## Accessibility Contract

- All status badges: `aria-label="Status: {label}"` (already established in existing StatusBadge)
- Tab nav: `role="tablist"`, each tab `role="tab"`, `aria-selected`, `aria-controls="{panel-id}"`
- Tab panels: `role="tabpanel"`, `aria-labelledby="{tab-id}"`, `tabindex="0"`
- Accordion rows: `aria-expanded={true/false}` on trigger button, `aria-controls="{detail-id}"`
- Toggle switch: `role="switch"`, `aria-checked={true/false}`, `aria-label="Ativar/desativar regra"`
- Account combobox: `role="combobox"`, `aria-autocomplete="list"`, `aria-expanded`, options in `role="listbox"`
- Origin links: `aria-label="Ver {sourceType} de origem"` (descriptive, not just "clique aqui")
- Error messages: `role="alert"` on error accordion content
- Retry button loading: `aria-busy="true"` + `aria-label="Tentando novamente..."` during load
- Focus trap in AccountingRuleModal (Escape closes, Tab cycles within modal)
- Focus returns to triggering "Editar" button after modal closes

| Modal close | 200ms | ease-in | `--duration-fast`, `--ease-in` |
| Badge status transition (optimistic) | 200ms | ease-out | `--duration-fast`, `--ease-out` |
| Toggle switch thumb | 200ms | ease-out | `--duration-fast`, `--ease-out` |
| Hover on table row | 100ms | ease-out | `--duration-instant`, `--ease-out` |
| Skeleton pulse | 1500ms | linear, infinite | opacity 0.4→0.7 |

When `prefers-reduced-motion: reduce`: all transitions set to `0ms` (instant). Skeleton pulse switches to static opacity 0.6.
