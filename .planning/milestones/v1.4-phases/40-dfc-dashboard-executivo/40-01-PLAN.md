    1. Verify fiscal year exists (same pattern as getDre)
    2. Find prior fiscal year (same pattern)
    3. Derive date ranges from fiscal year:
       - currentMonth: `new Date(fiscalYear.year, filters.month - 1, 1)` to `new Date(fiscalYear.year, filters.month, 0, 23, 59, 59, 999)`
       - YTD: `fiscalYear.startDate` to current month end

    4. Load paid payables for each period:
       ```typescript
       const paidPayables = await prisma.payable.findMany({

       });
       ```
    5. Load settled receivables for each period (use `receivedAt` NOT `settledAt` per Pitfall 1):
       ```typescript
       const settledReceivables = await prisma.receivable.findMany({
         where: { organizationId, receivedAt: { gte: startDate, lte: endDate } },

       });
       ```
    6. Build DfcPaidItem arrays: payables → type 'outflow' with amount from amountPaid; receivables → type 'inflow' with amount from amountReceived
    7. Load cash account balances (accounts with code starting '1.1.01'):
       ```typescript
       const cashAccounts = await prisma.chartOfAccount.findMany({
         where: { organizationId, isActive: true, isSynthetic: false, code: { startsWith: '1.1.01' } },
         select: { id: true },
       });
       const cashAccountIds = cashAccounts.map(a => a.id);
       ```
       For each period, load AccountBalance and sum closingBalance/openingBalance.
       - currentMonthOpening: sum of openingBalance for current month
       - currentMonthClosing: sum of closingBalance for current month
       - ytdOpening: sum of openingBalance for month 1 of the fiscal year
       - ytdClosing: same as currentMonthClosing
       - priorYear: same logic for prior fiscal year's same month
    8. Call `calculateDfcDireto(diretoInput)` to get direto result
    9. For indireto, gather adjustments:
       - lucroLiquido: call `getDre(organizationId, filters)` and extract `resultadoLiquido.currentMonth`, `.ytd`, `.priorYear` — if getDre fails, use '0'
       - depreciacao: sum AccountBalance `debitTotal - creditTotal` for accounts with code starting '5.2.03' (despesas-depreciacao); for YTD use groupBy sum months 1..filters.month
       - provisoes delta: AccountBalance closingBalance delta (current month vs prior month) for accounts with code starting '2.1.03' or '2.1.04'
       - cpc29FairValue: AccountBalance movement (debitTotal - creditTotal) for accounts with `isFairValueAdj = true`
       - workingCapitalDeltas — each is closingBalance(current month) - closingBalance(prior month):
         - deltaContasReceber: accounts starting '1.1.03' (if no match, try '1.1.02' range — but check COA template)
         - deltaEstoques: accounts starting '1.1.02'
         - deltaContasPagar: accounts starting '2.1.01'
         - deltaObrigacoes: accounts starting '2.1.02' + '2.1.03' + '2.1.04'
       - For YTD deltas: closingBalance(current month) - openingBalance(month 1)
       - For priorYear: same deltas from prior fiscal year's same month
    10. Pass investimentoSection and financiamentoSection from direto result, plus cash summary
    11. Call `calculateDfcIndireto(indiretoInput)` to get indireto result
    12. Return `{ direto, indireto }`
