## Spacing Scale

Declared values (multiples of 4px only). Source: `tokens.css` `--space-*` scale.

| Token | CSS Var | Value | Usage |
|-------|---------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, inline separators |
| sm | `--space-2` | 8px | Compact element spacing (badge padding, list item gaps) |
| md | `--space-4` | 16px | Default element spacing (card internal padding sides) |
| lg | `--space-6` | 24px | Section padding (card top/bottom padding, filter bar) |
| xl | `--space-8` | 32px | Layout gaps (between KPI row and chart section) |
| 2xl | `--space-12` | 48px | Major section breaks (between dashboard zones) |

- Touch targets (alert link buttons, tab controls): minimum 48x48px — enforced via `min-height: 48px; padding: 12px 16px`
- DFC table row height: 40px minimum for readability of tabular financial data
- Input fields: 12px vertical padding + 16px horizontal padding (per CLAUDE.md)

## Typography

Source: `tokens.css` + CLAUDE.md font rules.

| Role | CSS Var | Size | Font | Weight | Line Height |
|------|---------|------|------|--------|-------------|
| Body / table cell | `--text-sm` | 14px (0.875rem) | Source Sans 3 | 400 (`--font-regular`) | 1.5 (`--leading-normal`) |
| Label / badge / filter | `--text-sm` | 14px (0.875rem) | Source Sans 3 | 600 (`--font-semibold`) | 1.5 |
| Card heading / section title | `--text-base` | 16px (1rem) | DM Sans | 500 (`--font-medium`) | 1.2 (`--leading-tight`) |

- Monetary values in KPI cards: JetBrains Mono 400 (`--font-mono`) at `--text-xl` (24px) — ensures digit alignment

## Color

Source: CLAUDE.md + `tokens.css` semantic colors.

| Role | CSS Var | Value | Usage |
|------|---------|-------|-------|
| Dominant (60%) | `--color-neutral-50` | #FAFAF8 | Page background, main surface |
| Secondary (30%) | `--color-neutral-0` | #FFFFFF | Cards (KPI cards, chart cards, DFC table container, alert cards) |
| Accent (10%) | `--color-primary-600` | #2E7D32 | See reserved list below |
| Destructive | `--color-error-500` | #C62828 | Failed cross-validation badge, negative monetary values, error state |

Accent (`--color-primary-600`) reserved for:
- Active tab indicator underline (DfcPage tabs: Direto / Indireto)
- Sidebar active item text and left-border indicator
- "Investigar" button on cross-validation DFC↔BP card (primary action only)
- Positive sparkline trend line color in indicator cards
- Progress bar fill on accounting dashboard indicators when value is healthy

Color semantics for DFC data:
- Cash inflows (entradas): `--color-success-500` (#2E7D32) — text and bar fill
- Cash outflows (saídas): `--color-error-500` (#C62828) — text and bar fill
- Net cash flow positive: `--color-success-500`
- Net cash flow negative: `--color-error-500`
- Neutral / zero: `--color-neutral-500` (#7A7267)

Color semantics for dashboard alerts:
- Períodos não fechados: `--color-warning-500` (#F57F17) + AlertTriangle icon
- Lançamentos pendentes na fila: `--color-warning-500` + Clock icon
- Contas sem mapeamento SPED: `--color-info-500` (#0277BD) + Info icon

Color semantics for cross-validation card:
- Invariant PASSED: left border `--color-success-500`, icon CheckCircle success-500
- Invariant FAILED: left border `--color-error-500`, icon AlertTriangle error-500
- Invariant PENDING (invariant #2 before DFC computed): left border `--color-neutral-300`, icon Clock neutral-500

BP indicator card thresholds (AccountingDashboardPage):
- Liquidez Corrente ≥ 1.5: value text `--color-success-500`
- Liquidez Corrente 1.0–1.49: value text `--color-warning-500`
- Liquidez Corrente < 1.0: value text `--color-error-500`
- ROE positive: `--color-success-500`; negative: `--color-error-500`
- Endividamento Geral ≤ 0.4: `--color-success-500`; > 0.6: `--color-error-500`

Recharts chart palette (fixed order, accessible):
1. Receita / inflow: `#2E7D32` (primary-600)
2. Despesa / outflow: `#C62828` (error-500)
3. Donut slice order for cost composition: primary-600, sun-500 (#F9A825), sky-500 (#0288D1), earth-500 (#8D6E63), neutral-400 (#A8A196)

## Interaction Contracts

### DfcPage

- Seletor exercício fiscal: `<select>` with label "Exercício fiscal", populated from `useFiscalYears()`. Default: current open fiscal year.
- Seletor mês: `<select>` with label "Mês", populated from MONTHS constant. Default: current month.
- Both selectors trigger automatic data fetch on change (no "Calcular" button) — loading state with skeleton while fetching.
- Export CSV button: disabled during loading; triggers `GET /dfc?format=csv`; success shows no toast (browser downloads file); error shows toast "Não foi possível exportar. Tente novamente."
- Tab switching: instant (no loading — data already fetched, both methods returned in same API response).
- DFC table: no row expansion/collapse — all rows visible. Section headers not interactive.

### AccountingDashboardPage

- Seletor exercício fiscal + mês: same pattern as DfcPage. Auto-fetch on change.
- Alert rows: each is a `<Link>` (react-router-dom) with full row clickable. Underline on hover. Arrow icon (ChevronRight size 16) at right edge.
- Chart lazy loading: `React.lazy` + `Suspense` for both recharts components. Suspense fallback is skeleton div at chart height (288px for line chart, 248px for donut).
- BP indicator cards: sparkline data rendered inline. No interactive drill-down from dashboard (link to /balance-sheet not required per D-08 — they are a subset without full redundancy).
- Recharts tooltip: custom formatter for BRL values using `value.toLocaleString('pt-BR', { style: 'currency', currency: 'BRL' })`.

### Navigation (Sidebar)

- "DFC" and "Dashboard Contábil" added to CONTABILIDADE group.
- Sidebar order: Plano de Contas → Lançamentos → Fechamento → Balancete → DRE → Balanço Patrimonial → **DFC** → Validação Cruzada → **Dashboard Contábil**.
- Both items use standard sidebar link pattern. No sub-items.

## Accessibility Checklist

- `<main id="main-content">` on both pages — enables skip-to-content link
- Tab navigation: `role="tablist"` + `role="tab"` + `aria-selected` + `role="tabpanel"` + `aria-labelledby`
- DfcTable uses `<table>` with `<caption>` (e.g., "DFC Método Direto — Janeiro 2026"), `<th scope="col">` for column headers, `<th scope="row">` for section/row labels
- All monetary values in table cells: include `aria-label` with formatted BRL value (screen readers announce commas inconsistently in mono font)
- Alert section: `role="status"` for non-urgent, `role="alert"` only if new alert appears dynamically (not on load)
- Clickable alert rows: `<a>` element (not `<div>`), full accessible name via text content
- Charts: each recharts container has `role="img"` + `aria-label` describing the chart purpose; a data table alternative is not required (data accessible via DFC and BP pages)
- Focus outline: `outline: 2px solid var(--color-primary-500); outline-offset: 2px` — inherited from global CSS, must not be overridden
- Color not sole indicator: positive/negative cash flows use color + arrow icon (ArrowUp/ArrowDown) simultaneously
- KPI delta badges: `aria-label` with full sentence (e.g., `aria-label="Aumento de 12,3% em relação ao período anterior"`)
