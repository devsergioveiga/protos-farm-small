    **3. Create sped-ecd.writer.ts — Pure class (NO Prisma imports)**

    Class SpedEcdWriter:
    - Constructor receives SpedEcdInput
    - Private `lines: string[]` accumulator
    - Private `registerCounts: Map<string, number>` for Bloco 9 counts
    - Private helper `line(fields: (string | number | null)[]): void` that formats as `|f1|f2|...|fN|\r\n` and increments lineCount and registerCounts
    - Private `formatDate(d: Date): string` — returns DDMMAAAA using date-fns `format(d, 'ddMMyyyy')`
    - Private `formatAmount(val: string): string` — parse with Decimal, return `toFixed(2)` (period decimal, no thousand sep)
    - Private `codNat(accountType: string): string` — ATIVO=01, PASSIVO=02, PL=03, RECEITA=04, DESPESA=04
    - Private `indDc(accountNature: string, balance: string): string` — if DEVEDORA and amount >= 0 return "D", if CREDORA and amount >= 0 return "C", if amount = 0 return natural side

    Public `generate(): string`:
    1. writeBloco0() — 0000 (LECD header), 0001 (open), 0007 (book info: tipo G per D-08), 0990 (close with count)
    2. writeBlocoI() — I001 (open), I010 (writer ID: "Protos Farm", "1.0"), I050+I051 for each active account (D-05: all active, synthetic+analytic; I051 only for analytic with spedRefCode), I100 (cost centers), I150/I155 (monthly balances per analytic account only), I200/I250 (journal entries per analytic account only), I990 (close)
    3. writeBlocoJ() — J001 (open), J005 (period header), J100 (BP rows per D-04), J150 (DRE rows per D-04), J210 (DLPA rows per D-04), J990 (close)
    4. writeBloco9() — 9001 (open), 9900 (one line per register type with count), 9990 (close), 9999 (total file line count including 9999 itself)
    5. Return `this.lines.join('')`

    Key field layouts per RESEARCH.md register reference:
    - 0000: `|0000|LECD|{DT_INI}|{DT_FIN}|{NOME}|{CNPJ}|{UF}|{IE}|{COD_MUN}|{IM}||0|0|0|||||`
    - 0007: `|0007|Livro 1|1|G|{DT_INI}|{DT_FIN}|1||`
    - I050: `|I050|{DT_ALT}|{COD_NAT}|{IND_CTA: S or A}|{NIVEL}|{COD_CTA}|{COD_CTA_SUP}|{CTA_NAME}|`
    - I051: `|I051|{spedRefCode}||` (immediately after parent I050 for analytic accounts with spedRefCode)
    - I150: `|I150|{DT_INI_month}|{DT_FIN_month}|`
    - I155: `|I155|{COD_CTA}|{COD_CTA_CUS}|{VL_SLD_INI}|{IND_DC_INI}|{VL_DEB}|{VL_CRED}|{VL_SLD_FIN}|{IND_DC_FIN}|`
    - I200: `|I200|{NUM_LCTO}|{DT_LCTO}|{VL_LCTO}|{IND_LCTO: N}|`
    - I250: `|I250|{COD_CTA}|{COD_CTA_CUS}|{VL_DC}|{IND_DC: D or C}|{NUM_LCTO}|{HIST}|{DT_LCTO}|`
    - J100: `|J100|{COD_AGL}|{IND_COD_AGL: T or D}|{NIVEL_AGL}|{COD_AGL_SUP}|{IND_GRP_BAL: A or P}|{DESCR}|{VL_CTA_INI}|{IND_DC_INI}|{VL_CTA_FIN}|{IND_DC_FIN}|`
    - J150: `|J150|{COD_AGL}|{IND_COD_AGL}|{NIVEL_AGL}|{COD_AGL_SUP}|{DESCR}|{VL_CTA}|{IND_DC}|{IND_GRP_DRE: D or R}|`
    - J210: `|J210|0|{COD_AGL}|{DESCR}|{VL_CTA_INI}|{IND_DC_INI}|{VL_CTA_FIN}|{IND_DC_FIN}|`
    - 9900: `|9900|{REG_TYPE}|{QTD_LINES}|`
    - 9999: `|9999|{TOTAL_LINE_COUNT}|`

    **1. Create sped-ecd.service.ts**

    Two exported functions:

    **validateSpedEcd(organizationId: string, fiscalYearId: string): Promise&lt;SpedValidationResult&gt;**

    Validation checks (per D-14, D-17, SPED-02):
    1. **UNMAPPED_SPED (ERROR)**: Call `getUnmappedSpedAccounts(organizationId)`. If count > 0, add item with `severity: 'ERROR'`, `code: 'UNMAPPED_SPED'`, `message: '${count} conta(s) sem mapeamento SPED (codigo referencial ausente)'`, `navigateTo: '/chart-of-accounts'`.
    2. **OPEN_PERIODS (ERROR)**: Query `prisma.accountingPeriod.findMany({ where: { organizationId, fiscalYearId, status: 'OPEN' } })`. For each open period, add ERROR item with `message: 'Periodo ${MM}/${YYYY} nao esta fechado'`, `navigateTo: '/monthly-closing'`.
    3. **UNBALANCED_TRIAL (ERROR)**: For each month in fiscal year, call `getTrialBalance(organizationId, { fiscalYearId, month })` and check `totalDebits === totalCredits`. If unequal, add ERROR with `message: 'Balancete desequilibrado: debitos != creditos em ${MM}/${YYYY}'`.
    4. **DUPLICATE_I050 (ERROR)**: Query all active accounts for org and check for duplicate `code` values. If found, add ERROR.
    5. **I155_INCONSISTENCY (ERROR per D-17)**: For each month, verify sum of JournalEntryLine debits/credits per account matches AccountBalance totalDebits/totalCredits. If mismatch, add ERROR with `message: 'Totalizacao I155 inconsistente em ${MM}/${YYYY}: soma dos lancamentos != saldo apurado'`.
    6. **NO_MOVEMENT (WARNING)**: Count active analytic accounts with zero totalDebits and zero totalCredits across all periods. If count > 0, add WARNING `message: '${count} conta(s) sem movimento no periodo (serao incluidas com saldo zero)'`.
    7. **INACTIVE_ACCOUNTS (WARNING)**: Count inactive accounts that have historical data. If found, add WARNING.

    Return `{ items, hasErrors: items.some(i => i.severity === 'ERROR') }`.

    **generateSpedEcd(organizationId: string, fiscalYearId: string): Promise&lt;{ content: string; filename: string }&gt;**

    1. Load Organization (name, document, accountantName, accountantCrc, accountantCpf)
    2. Load FiscalYear (startDate, endDate) — derive year as `fiscalYear.startDate.getFullYear()` (per STATE.md Phase 40 decision)
    3. Load all active ChartOfAccount for org (code, name, accountType, accountNature, isSynthetic, level, parent.code, spedRefCode)
    4. Load AccountBalance for all months in fiscal year (analytic accounts only, isSynthetic=false)
    5. Load JournalEntry + JournalEntryLine for fiscal year date range (where status = 'POSTED')
    6. Load CostCenters for org
    7. Call getDre() and getBp() from financial-statements.service for J-block data
    8. Map DRE output sections to SpedDreRow[] (extract spedRefCode from account mapping)
    9. Map BP output groups to SpedBpRow[]
    10. Build SpedDlpaRow from PL accounts (opening balance from prior FY last month, closing from current FY last month, with net income from DRE)
    11. Construct SpedEcdInput and call `new SpedEcdWriter(input).generate()`
    12. Build filename: `SPED_ECD_${cnpjDigitsOnly}_${year}.txt` (per D-12)
    13. Return `{ content, filename }`
