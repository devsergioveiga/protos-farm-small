    **Step 3: Service** (`journal-entries.service.ts`)

    Implement these functions (all using `prisma` from `../../lib/prisma`):

    1. `createJournalEntryDraft(organizationId, input, createdBy)`:
       - Validate all line accountIds: fetch ChartOfAccount records, check `isActive`, `!isSynthetic`, `allowManualEntry`
       - Call `assertBalanced(input.lines.map(l => ({ side: l.side, amount: l.amount })))`
       - Fetch period by `input.periodId`, call `assertPeriodOpen(period)`
       - Create JournalEntry with `entryNumber: 0`, `status: 'DRAFT'`, nested lines create
       - Return with lines + account select {code, name, nature}

    2. `postJournalEntry(organizationId, entryId, postedBy)`:
       - Wrap in `prisma.$transaction(async (tx) => { ... }, { isolationLevel: 'Serializable' })`
       - Fetch entry with lines + period; guard status === 'DRAFT'
       - `assertPeriodOpen(entry.period)`, `assertBalanced(entry.lines...)`
       - `SELECT MAX(entryNumber)` via `tx.journalEntry.aggregate`, compute `nextNumber`
       - For each line: `tx.accountBalance.upsert` — increment debitTotal or creditTotal
       - Recompute closingBalance: for each affected account, fetch account nature, then:
         - DEVEDORA: `closingBalance = openingBalance + debitTotal - creditTotal`
         - CREDORA: `closingBalance = openingBalance - debitTotal + creditTotal`
         - Use raw SQL UPDATE within tx for efficiency:
           ```sql
           UPDATE account_balances ab
           SET closing_balance = ab.opening_balance +
             CASE WHEN coa.nature = 'DEVEDORA' THEN ab.debit_total - ab.credit_total
                  ELSE ab.credit_total - ab.debit_total END
           FROM chart_of_accounts coa
           WHERE coa.id = ab.account_id
             AND ab.organization_id = $1
             AND ab.fiscal_year_id = $2
             AND ab.month = $3
             AND ab.account_id = ANY($4)
           ```
       - Update entry: `status: 'POSTED'`, `entryNumber: nextNumber`, `postedAt: new Date()`

    3. `reverseJournalEntry(organizationId, entryId, reason, reversedBy)`:
       - Validate reason non-empty (trim), throw `REASON_REQUIRED` if empty
       - Wrap in `prisma.$transaction`
       - Fetch original with lines + period; guard status === 'POSTED' (not 'REVERSED')
       - `assertPeriodOpen(original.period)`
       - Create reversal entry: `entryType: 'REVERSAL'`, `reversalOf: entryId`, `reversalReason: reason`, lines with inverted sides
       - Mark original: `status: 'REVERSED'`, `reversedById: reversal.id`
       - Call internal postJournalEntry logic (within same tx) to assign number and update balances

    4. `saveTemplate(organizationId, input, createdBy)`: Create JournalEntry with `templateName` set, `status: 'DRAFT'`, `entryNumber: 0`
    5. `listTemplates(organizationId)`: Find entries where `templateName IS NOT NULL`
    6. `deleteTemplate(organizationId, templateId)`: Delete where `templateName IS NOT NULL`
    7. `deleteDraft(organizationId, entryId)`: Delete where `status: 'DRAFT'` and `templateName IS NULL`; reject if POSTED
    8. `listEntries(organizationId, filters)`: Paginated findMany with optional periodId, status, entryType, dateRange filters; include lines count
    9. `getEntry(organizationId, entryId)`: findFirst with full lines + account details

    10. `importCsvJournalEntries(organizationId, csvBuffer: Buffer)` **(per LANC-03 CSV import)**:
       - Parse CSV buffer using the same simple CSV parsing approach as `reconciliation/csv-parser.ts` (split lines, handle quoted fields).
       - Expected CSV columns: `entryDate`, `periodId`, `description`, `accountCode`, `side` (DEBITO/CREDITO or DEBIT/CREDIT), `amount`, `costCenterId` (optional).
       - For each row: look up `accountCode` via `prisma.chartOfAccount.findFirst({ where: { organizationId, code: accountCode } })`. If not found, add to errors array with reason `ACCOUNT_NOT_FOUND`.
       - Normalize `side`: accept "DEBITO"/"D" as DEBIT, "CREDITO"/"C" as CREDIT.
       - Group valid rows by `entryDate + description` into `CreateJournalEntryInput[]`.
       - For each group, call `assertBalanced` — if unbalanced, move group rows to errors with reason `UNBALANCED`.
       - If buffer is empty or has no data rows, throw `JournalEntryError('Arquivo CSV vazio', 'EMPTY_CSV')`.
       - Return `CsvImportPreview` with `entries` (valid groups ready to create as drafts), `errors`, `totalEntries`, `totalErrors`.
       - Do NOT auto-create drafts — return preview only. Frontend will call `createJournalEntryDraft` for each approved entry.
