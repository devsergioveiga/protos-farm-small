## Typography

All sizes from `tokens.css`. Use CSS vars — never hardcode.

Two weights only: 400 (`--font-regular`) and 600 (`--font-semibold`). No `--font-bold` (700) is used in this phase — aligns with CLAUDE.md's DM Sans specification (500, 700 for headlines is the font capability, but the project token scale declares 400 and 600 as the two allowed weights in UI).

| Role    | Size                 | Weight                  | Line Height              | Font           | Usage                                                               |
| ------- | -------------------- | ----------------------- | ------------------------ | -------------- | ------------------------------------------------------------------- |
| Body    | 16px (`--text-base`) | 400 (`--font-regular`)  | 1.5 (`--leading-normal`) | Source Sans 3  | Row labels, table cells, form help text                             |
| Label   | 14px (`--text-sm`)   | 600 (`--font-semibold`) | 1.3 (`--leading-snug`)   | Source Sans 3  | Table headers, form labels, badge text, status chips                |
| Heading | 20px (`--text-md`)   | 600 (`--font-semibold`) | 1.2 (`--leading-tight`)  | DM Sans        | Page section titles, modal titles                                   |
| Display | 24px (`--text-xl`)   | 600 (`--font-semibold`) | 1.2 (`--leading-tight`)  | DM Sans        | Page `<h1>` title                                                   |
| Mono    | 14px (`--text-sm`)   | 400 (`--font-regular`)  | 1.5 (`--leading-normal`) | JetBrains Mono | Timestamps (08:07), durations (7h32m), hour totals, GPS coordinates |

Notes:

- Minimum 12px — never go below (outdoor readability, CLAUDE.md)
- Mono font is mandatory for all time values — operators compare registers visually; consistent character width prevents misreading
- ALL CAPS only for short column headers ("ENTRADA", "SAÍDA", "HE") — never in body or description text

```

  <h1> Controle de Ponto
  [Filtros: fazenda, colaborador, período]
  [Tabs: Apontamentos | Banco de Horas | Vincular Operações]

```

- Max-width: 1280px, centered — matches `.payroll-params` pattern
- Padding: 32px top/bottom, 24px horizontal (desktop); 16px horizontal (mobile)
- Table on desktop (≥768px); card stack on mobile (<768px)
- Single primary CTA: "Registrar Ponto" (manager manual entry) — opens `ManualPunchModal`

### Web: TimesheetPage (`/timesheets`)

Structure:

```

  Breadcrumb: RH > Espelho de Ponto
  <h1> Espelho de Ponto
  [Filtros: fazenda, mês/ano, colaborador, status]
```

- Status flow chips: PENDENTE (warning) → APROVADO_GERENTE (info) → APROVADO_RH (primary) → FECHADO (neutral)

## Interaction Contracts

### Clock-in / Clock-out (Mobile)

1. User taps primary circular button
2. `Haptics.impactAsync(Light)` on press
3. GPS location captured — loading spinner inside button (300ms max)
4. If geofence valid (turf.js offline check): immediate optimistic commit to SQLite; show entry in list; sync dot changes to syncing → synced
5. If outside geofence: warning modal "Você está fora do perímetro da fazenda. Deseja registrar mesmo assim?" — yes/no — if yes, entry flagged `outsideGeofence: true`
6. `Haptics.notificationAsync(Success)` on confirmed entry
7. If offline: entry stored with `syncStatus: 'PENDING'`; banner updated to show pending count

### Manual Punch Entry (Web — Manager)

1. Manager opens `ManualPunchModal` from primary CTA
2. Fields: Colaborador* (searchable select), Data*, Tipo* (ENTRADA / SAÍDA / INTERVALO_INÍCIO / INTERVALO_FIM), Horário*, Justificativa\* (textarea, min 10 chars)
3. Validation inline on blur: required fields, time format HH:MM, justificativa length
4. Submit: POST `/time-entries` — on success, close modal + success toast + refetch list
5. Justificativa stored in audit log — visible in timesheet view as info badge with `Info` icon

### Timesheet Approval (Web)

1. RH/Manager opens `TimesheetApprovalModal` from "Aprovar" or "Rejeitar" action
2. Approve path: confirmation prompt "Aprovar espelho de [Nome] — [Mês]?" — single confirm button (primary)
3. Reject path: requires `justificativa` field (min 20 chars); confirm button labeled "Rejeitar e devolver"; button uses `--color-error-500`
4. After approval: entries become immutable if `payrollRunId` is set; row chip updates to approved state
5. "Aceite do colaborador" — mobile screen shows pending timesheets; tap to view, long-press or swipe to accept

### Inconsistency Correction (Web)

1. Row flagged with `AlertCircle` icon (16px, `--color-error-500`) + red background
2. Click row or "Corrigir" action opens `TimeEntryEditModal`
3. Fields: horário corrigido, justificativa\* (required for any edit)
4. Audit trail: original value shown in modal as read-only "Valor original: 07:58", new value in editable field
5. ConfirmModal with `variant="warning"` before saving — "Confirmar correção? Esta alteração ficará no histórico de auditoria."

### Overtime Bank Alert (Web)

1. `OvertimeBankCard` shows: saldo atual, saldo expiring in ≤30 days (warning chip), saldo expiring in ≤7 days (error chip)
2. Chip: `Clock` icon (16px) + "X horas vencem em DD/MM/AAAA"
3. No destructive action — bank entries are system-generated, read-only in UI

## Accessibility Checklist

- All `<Pressable>` (mobile): `accessible={true}`, `accessibilityLabel`, `accessibilityRole="button"`
- Clock-in/out button: `accessibilityLabel="Registrar entrada"` / `"Registrar saída"` (dynamic)
- GPS status chip: `accessibilityRole="text"`, `accessibilityLabel="Status: Dentro da fazenda"`
- Inconsistency row: `role="alert"` on the error badge — `aria-live="polite"` on web
- Timesheet table: `<th scope="col">` for all column headers; `<caption>` describing the month
- Status chips: never rely on color alone — always include icon + text label
- Focus management: `TimeEntryEditModal` traps focus; on close, returns focus to triggering row
- Form error messages: `role="alert"` + `aria-live="polite"`, displayed below field with `AlertCircle` icon
- Modal close: `Escape` key on web; hardware back button + swipe-down on mobile
- Offline banner: `role="status"` + `aria-live="polite"` on web
- `prefers-reduced-motion`: disable clock-in button pulse animation and sync dot animation

## Animation Contracts

All durations from `tokens.css` — never custom values.

| Interaction                       | Duration        | Easing          | Notes                                                                     |
| --------------------------------- | --------------- | --------------- | ------------------------------------------------------------------------- |
| Tab switch                        | 200ms           | `--ease-out`    | Content fade, not slide                                                   |
