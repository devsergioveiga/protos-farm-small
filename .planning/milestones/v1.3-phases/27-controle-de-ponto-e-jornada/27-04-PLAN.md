**useTimeEntries.ts** — Follow existing `useState+useCallback` pattern (per STATE.md decision — no SWR):
- `fetchTimeEntries(orgId, query)` — GET `/api/org/${orgId}/time-entries`
- `createTimeEntry(orgId, employeeId, data)` — POST `/api/org/${orgId}/employees/${employeeId}/time-entries`
- `addActivity(orgId, timeEntryId, data)` — POST `/api/org/${orgId}/time-entries/${timeEntryId}/activities`
- State: `timeEntries`, `loading`, `error`, `successMessage`
- Return: `{ timeEntries, loading, error, successMessage, fetchTimeEntries, createTimeEntry, addActivity }`

**useTimesheet.ts** — Same pattern:
- `fetchTimesheets(orgId, query)` — GET `/api/org/${orgId}/timesheets`
- `fetchTimesheet(orgId, id)` — GET `/api/org/${orgId}/timesheets/${id}`
- `calculateTimesheet(orgId, id)` — POST `/api/org/${orgId}/timesheets/${id}/calculate`
- `approveTimesheet(orgId, id, data)` — PATCH `/api/org/${orgId}/timesheets/${id}/approve`
- `addCorrection(orgId, id, data)` — POST `/api/org/${orgId}/timesheets/${id}/corrections`
- `exportPdf(orgId, id)` — GET `/api/org/${orgId}/timesheets/${id}/pdf` with `responseType: 'blob'`
- State: `timesheets`, `timesheet`, `loading`, `error`, `successMessage`

**TeamLinkingTab.tsx** — Third tab in AttendancePage ("Vincular Operacoes"):

This component implements PONTO-02 "modo rapido por equipe" — bulk linking of hours to an operation for all members of a team.

Layout:

- Team select (searchable dropdown, fetches from `/api/org/${orgId}/field-teams?farmId=${farmId}`)
- Date picker (defaults to today)
- Operation select (fetches from `/api/org/${orgId}/field-operations?farmId=${farmId}`)
- Talhao/Pasto select (fetches from `/api/org/${orgId}/plots?farmId=${farmId}`)
- Centro de Custo select (pre-filled from team's costCenterId if present)
- Minutos\* input (number, required)
- Observacoes textarea (optional)
- "Vincular para Equipe" primary button

On submit: POST `/api/org/${orgId}/time-entries/team/${teamId}/activities` with `{ date, operationType, fieldOperationId, fieldPlotId, costCenterId, minutes, notes }`

Response handling:

- Show summary toast: "Vinculado para {created} de {total} membros da equipe"
- If `skipped > 0`: show info toast listing skipped members (no time entry for the date)
- Refetch time entries list

Also supports individual linking (existing LinkOperationModal) — show toggle or two sub-sections:

- "Por Equipe" (TeamLinkingTab bulk mode described above)
