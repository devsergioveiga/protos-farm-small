Create `payroll-runs.service.ts`:

**`createRun(rls, input: { referenceMonth: string, runType: PayrollRunType, notes?: string })`**

1. Parse referenceMonth as `new Date(input.referenceMonth + '-01')`.
2. Check uniqueness: `findFirst({ where: { organizationId, referenceMonth, runType }})`. Throw PayrollRunError('DUPLICATE_RUN') if exists.
3. Create PayrollRun with status PENDING, triggeredBy = rls.userId.
4. Return created run.

**`processRun(rls, runId: string)`**

1. Fetch run, validate status is PENDING (via VALID_PAYROLL_TRANSITIONS.START).
2. Update status to PROCESSING.
3. Fetch all active employees for org (Employee where status='ACTIVE').
4. Fetch engine params: get effective INSS brackets, IRRF table, FGTS rate, salary family limits from payroll-tables service (`getEffectiveTables(referenceMonth)`).
5. For each employee (per-employee transaction pattern from depreciation-batch):
   ```typescript
   await prisma.$transaction(async (tx) => {
     // a. Get salary from EmployeeSalaryHistory (latest effectiveAt <= referenceMonth)
     // b. Get timesheet for this employee+referenceMonth with status=APPROVED
     // c. If no approved timesheet: create PayrollRunItem with status='PENDING_TIMESHEET', baseSalary, grossSalary=0, netSalary=0
     // d. If approved: build EmployeePayrollInput from salary, timesheet, employee data
     // e. Get pending SalaryAdvance for this employee+referenceMonth (not yet deducted)
     // f. For MONTHLY: call calculateEmployeePayroll()
     //    For THIRTEENTH_FIRST/SECOND: call calculateThirteenthSalary() with HE averages
     // g. Create PayrollRunItem with all calculated fields + lineItemsJson from result.lineItems
   });
   ```
6. Count calculated items, update run: employeeCount, totalGross, totalNet, totalCharges. Transition to CALCULATED.

**For THIRTEENTH runs:** Before per-employee loop, fetch all APPROVED timesheets for referenceYear. For each employee, calculate avgOvertime50/100/nightPremium = sum/monthsWorked. MonthsWorked = months from max(admissionDate, Jan 1) to Nov 30 (FIRST) or Dec 31 (SECOND).

**`recalculateEmployee(rls, runId: string, employeeId: string)`**

1. Validate run status is CALCULATED (transition RECALC: CALCULATED->PROCESSING).
2. Delete existing PayrollRunItem for this employee.
3. Re-run calculation for this employee (same as processRun per-employee logic).
4. Recalculate totals. Transition back to CALCULATED.

**`closeRun(rls, runId: string)`**

1. Validate status CALCULATED (transition CLOSE: CALCULATED->COMPLETED).
2. In a transaction per employee:
   - Create Payable with `originType: 'PAYROLL_RUN_ITEM', originId: item.id`, category: 'PAYROLL', `dueDate: fifthBusinessDay(followingMonth)`, supplierName = employee name.
   - Lock timesheet: `tx.timesheet.update({ where: { employeeId_referenceMonth }, data: { status: 'LOCKED', payrollRunId: runId } })`.
   - Mark SalaryAdvance as deducted: `deductedInRunId = runId`.
3. Create employer CPs:
   - INSS patronal: sum of all items' inssPatronal, originType='PAYROLL_EMPLOYER_INSS', originId=runId, dueDate=20th following month.
   - FGTS: sum of all items' fgtsAmount, originType='PAYROLL_EMPLOYER_FGTS', originId=runId, dueDate=7th following month.
4. Generate payslip PDF for each CALCULATED item (call generatePayslipPdf), send email to employees with email via sendMail (per D-05). Store payslipSentAt.
5. Update run: status=COMPLETED, closedAt=now(), closedBy=rls.userId.

**`revertRun(rls, runId: string)`** (per D-03)

1. Validate status COMPLETED (transition REVERT: COMPLETED->REVERTED).
2. Cancel all Payables linked to this run: `updateMany({ where: { originId: { in: [runId, ...itemIds] }, originType: { in: ['PAYROLL_RUN_ITEM','PAYROLL_EMPLOYER_INSS','PAYROLL_EMPLOYER_FGTS'] }}, data: { status: 'CANCELLED' }})`.
3. Unlock timesheets: `updateMany({ where: { payrollRunId: runId }, data: { status: 'APPROVED', payrollRunId: null }})`.
4. Reset SalaryAdvance deductedInRunId: `updateMany({ where: { deductedInRunId: runId }, data: { deductedInRunId: null }})`.
5. Update run: status=REVERTED, revertedAt=now(), revertedBy=rls.userId.

**`listRuns(rls, filters)`** and **`getRun(rls, runId)`** — standard CRUD with pagination and includes.

**`downloadPayslipsZip(rls, runId: string): Promise<Buffer>`** (per D-06)

1. Fetch run with CALCULATED items.
2. For each item: generate PDF from lineItemsJson via generatePayslipPdf.
3. Use JSZip: `const zip = new JSZip(); zip.file(filename, pdfBuffer)` where filename = `holerite_${YYYY-MM}_${NAME.toUpperCase().replace(/\s+/g, '-')}.pdf`.
4. Return `zip.generateAsync({ type: 'nodebuffer', compression: 'DEFLATE' })`.

**`getEmployeePayslips(rls, employeeId: string)`** — for employee ficha tab (per D-05):

1. Fetch PayrollRunItems for employee where run.status=COMPLETED, last 12 months.
2. Return with run referenceMonth, runType, grossSalary, netSalary.

```typescript
import { Router } from 'express';
import { requireAuth, requirePermission } from '../../shared/rbac/middleware';
import * as service from './payroll-runs.service';

const router = Router();
router.use(requireAuth);

// POST / — create run (wizard step 1)
router.post('/', requirePermission('payroll-params', 'write'), async (req, res, next) => {
  try {
    const orgId = req.params.orgId as string;
    const run = await service.createRun(
      { organizationId: orgId, userId: (req as any).user.id },
      req.body
    );
    res.status(201).json(run);
  } catch (err) { next(err); }
});

// GET / — list runs with filters
router.get('/', requirePermission('payroll-params', 'read'), async (req, res, next) => { ... });

// GET /preview/:runId — preview employees for wizard step 2
// Register BEFORE /:id to prevent Express 5 route shadowing
router.get('/preview/:runId', requirePermission('payroll-params', 'read'), async (req, res, next) => {
  // Returns employees with timesheet status for the reference month
});

// GET /:id — get run details
router.get('/:id', requirePermission('payroll-params', 'read'), async (req, res, next) => { ... });

// POST /:id/process — start batch processing (wizard step 4)
router.post('/:id/process', requirePermission('payroll-params', 'write'), async (req, res, next) => {
  // req.body.employeeIds = selected employee IDs from wizard step 3
});

// POST /:id/recalculate/:employeeId — recalculate individual
router.post('/:id/recalculate/:employeeId', requirePermission('payroll-params', 'write'), async (req, res, next) => { ... });

// POST /:id/close — finalize run (per D-03 flow)
router.post('/:id/close', requirePermission('payroll-params', 'write'), async (req, res, next) => { ... });

// POST /:id/revert — estorno (per D-03)
router.post('/:id/revert', requirePermission('payroll-params', 'write'), async (req, res, next) => { ... });

// GET /:id/payslips — download ZIP of all PDFs (per D-06)
router.get('/:id/payslips', requirePermission('payroll-params', 'read'), async (req, res, next) => {

  res.send(buffer);
});

// GET /:id/items/:itemId/payslip — download individual PDF
router.get('/:id/items/:itemId/payslip', requirePermission('payroll-params', 'read'), async (req, res, next) => { ... });

export default router;
```
