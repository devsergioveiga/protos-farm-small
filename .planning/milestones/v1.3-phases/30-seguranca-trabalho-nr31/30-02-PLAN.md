**epi-products.service.ts** — Implement using withRlsContext pattern:

1. `createEpiProduct(ctx, input: CreateEpiProductInput)` — verify Product exists and belongs to org, create EpiProduct row. Return EpiProductOutput with product name and current stock from StockBalance.
2. `updateEpiProduct(ctx, id: string, input: UpdateEpiProductInput)` — update caNumber/caExpiry/epiType. Verify EpiProduct belongs to org.
3. `deleteEpiProduct(ctx, id: string)` — delete EpiProduct (only if no EpiDelivery references it). Return 409 if deliveries exist.
4. `listEpiProducts(ctx, query)` — list all EpiProducts for org with Product name, current stock (from StockBalance), CA expiry status. Support search by name and filter by epiType. Paginate.
5. `getEpiProduct(ctx, id: string)` — single EpiProduct with full details.
6. `createPositionEpiRequirement(ctx, input: CreatePositionEpiRequirementInput)` — create requirement. Default quantity=1.
7. `deletePositionEpiRequirement(ctx, id: string)` — delete requirement.
8. `listPositionEpiRequirements(ctx, positionId?: string)` — list all requirements, optionally filtered by position. Join Position.name and EpiProduct product name.

Key: `listEpiProducts` must join `StockBalance` to get `currentQuantity` for the product. Use `select` to include `product: { select: { name: true, stockBalances: { select: { currentQuantity: true } } } }`.

**epi-deliveries.service.ts** — Critical: stock deduction inside same withRlsContext transaction (NO nested service calls per STATE.md anti-pattern):

1. `createEpiDelivery(ctx, input: CreateEpiDeliveryInput)`:
   - Inside `withRlsContext(ctx, async (tx) => { ... })`:
   - Verify EpiProduct exists, get its productId
   - Verify Employee exists and belongs to org
   - Get employee name for StockOutput responsibleName
   - Create StockOutput: `tx.stockOutput.create({ data: { organizationId, type: 'CONSUMPTION', outputDate: new Date(input.date), responsibleName: employeeName, notes: 'EPI entregue ao colaborador', status: 'CONFIRMED', createdBy: ctx.userId } })`
   - Create StockOutputItem: `tx.stockOutputItem.create({ data: { stockOutputId, productId, quantity: input.quantity, unitCost: averageCost from StockBalance } })`
   - Update StockBalance: `tx.stockBalance.update({ where: { organizationId_productId: { organizationId, productId } }, data: { currentQuantity: { decrement: input.quantity }, totalCost: { decrement: costReduction } } })`
   - Check stock sufficiency BEFORE deduction — if insufficient, throw EpiDeliveryError('INSUFFICIENT_STOCK')
   - Create EpiDelivery: `tx.epiDelivery.create({ data: { ...input, organizationId, stockOutputId, createdBy: ctx.userId } })`
   - Return EpiDeliveryOutput with employee name, product name, CA number

2. `deleteEpiDelivery(ctx, id: string)`:
   - Inside withRlsContext:
   - Find delivery with stockOutputId
   - Restore StockBalance (increment quantity, add back cost)
   - Delete StockOutputItem + StockOutput
   - Delete EpiDelivery
   - Per CONTEXT.md: "Remover esta entrega irá restaurar o saldo de estoque"

3. `listEpiDeliveries(ctx, query: EpiDeliveryListQuery)`:
   - Paginated list with employee name, product name, CA number
   - Filter by employeeId, epiType (join through EpiProduct), reason, date range
   - Include count for pagination

4. `getEpiDelivery(ctx, id: string)` — single delivery with full details

5. `listEmployeeDeliveries(ctx, employeeId: string)` — all deliveries for a specific employee (for Ficha tab)

6. `generateEpiFichaPdf(ctx, employeeId: string)`:
   - Dynamic import: `const PDFDocument = (await import('pdfkit')).default;`
   - Fetch employee data (name, position, admission date)
   - Fetch all deliveries for employee with EpiProduct details
   - Build A4 PDF (margin 50):
     - Header: org name (from ctx or query), employee name, position, admission date
     - Title: "FICHA DE CONTROLE DE EPI" (centered, bold)
     - Subtitle: "Conforme NR-6 e NR-31"
     - Table columns: Data | Descrição EPI | Nº CA | Qtd | Motivo | Assinatura
     - Each delivery row with date formatted dd/MM/yyyy
     - Footer: "Declaro ter recebido os EPIs listados acima, ter sido treinado quanto ao uso correto e conservação, e me comprometo a usá-los durante a jornada de trabalho."
     - Signature lines: collaborator + employer
   - Return Buffer (same pattern as pesticide-prescriptions)
