**training-types.service.ts:**

1. `seedNr31TrainingTypes()` — Seed function (NO RLS context needed since system types have organizationId=null):
   - Import `NR31_TRAINING_TYPES` from types file
   - For each type, upsert by `{ name, isSystem: true }` composite or just create if not exists:
     ```typescript
     for (const tt of NR31_TRAINING_TYPES) {
       await prisma.trainingType.upsert({
         where: { organizationId_name: { organizationId: null!, name: tt.name } },
         // If unique constraint doesn't support null, use a findFirst + create pattern:
         // const existing = await prisma.trainingType.findFirst({ where: { name: tt.name, isSystem: true } });
         // if (!existing) await prisma.trainingType.create({ data: { ...tt } });
         create: { ...tt, organizationId: undefined },
         update: {},
       });
     }
     ```
   - Actually, since `organizationId` is nullable and the unique constraint `@@unique([organizationId, name])` may not handle null correctly in Postgres (null != null), use the findFirst + create pattern instead:
     ```typescript
     const existing = await prisma.trainingType.findFirst({ where: { name: tt.name, isSystem: true } });
     if (!existing) { await prisma.trainingType.create({ data: { name: tt.name, description: `Treinamento obrigatório ${tt.nrReference}`, minHours: tt.minHours, defaultValidityMonths: tt.defaultValidityMonths, nrReference: tt.nrReference, isSystem: true, isGlobal: tt.isGlobal } }); }
     ```
   - Call this function from a seed route `POST /training-types/seed` (admin only) or auto-call on first `listTrainingTypes` if no system types exist. Recommended: expose as `POST /training-types/seed` with admin auth, and also call from prisma seed script.

2. `createTrainingType(ctx, input: CreateTrainingTypeInput)` — Create custom type (isSystem=false, organizationId=ctx.organizationId). Validate name unique within org.

3. `updateTrainingType(ctx, id: string, input: UpdateTrainingTypeInput)` — Update custom type only. If isSystem=true, throw TrainingTypeError('SYSTEM_TYPE_READONLY', 'Treinamentos do sistema não podem ser editados').

4. `deleteTrainingType(ctx, id: string)` — Delete custom type only. If isSystem=true, throw error. If TrainingRecord references it, throw 409.

5. `listTrainingTypes(ctx)` — List both system types (isSystem=true, organizationId=null) AND org-specific types (organizationId=ctx.organizationId). `WHERE isSystem = true OR organizationId = $orgId`. No pagination needed (small dataset).

6. `getTrainingType(ctx, id: string)` — Single type.

7. `createPositionTrainingRequirement(ctx, input)` — Create requirement linking position to training type.

8. `deletePositionTrainingRequirement(ctx, id: string)` — Delete requirement.

9. `listPositionTrainingRequirements(ctx, positionId?: string)` — List requirements with position name and training type name.

**training-records.service.ts:**

1. `createTrainingRecord(ctx, input: CreateTrainingRecordInput)`:
   - Inside withRlsContext(ctx, async (tx)):
   - Verify TrainingType exists (system or org-scoped)
   - Get TrainingType.defaultValidityMonths for expiry calculation
   - Create TrainingRecord row
   - For each employeeId in input.employeeIds:
     - Verify employee exists and belongs to org
     - Calculate expiresAt: `new Date(input.date)` + `defaultValidityMonths` months. Use: `const d = new Date(input.date); d.setMonth(d.getMonth() + defaultValidityMonths);`
     - Create EmployeeTrainingRecord: `{ trainingRecordId, employeeId, expiresAt }`
   - Return TrainingRecordOutput with participants list and count

2. `deleteTrainingRecord(ctx, id: string)`:
   - Delete all EmployeeTrainingRecords for this record first (cascade), then delete TrainingRecord.
   - Or rely on Prisma cascade if configured.

3. `listTrainingRecords(ctx, query: TrainingRecordListQuery)`:
   - Paginated list with training type name, instructor info, participant count
   - Filter by trainingTypeId, instructorType, date range, farmId
   - Include `_count: { select: { employees: true } }` for participant count

4. `getTrainingRecord(ctx, id: string)` — Full record with all participant details.

5. `generateCertificatePdf(ctx, trainingRecordId: string, employeeId: string)`:
   - Dynamic import pdfkit
   - Fetch TrainingRecord + TrainingType + Employee data
   - Fetch EmployeeTrainingRecord for expiresAt
   - Build A4 PDF (margin 50):
     - Header: Organization name (centered, bold, 16pt)
     - Title: "CERTIFICADO DE TREINAMENTO" (centered, bold, 20pt)
     - Body: "Certificamos que {employeeName}, portador(a) do CPF {cpf}, participou do treinamento de {trainingTypeName} ({nrReference}), com carga horária de {effectiveHours} horas, realizado em {date formatted dd/MM/yyyy}, ministrado por {instructorName} ({instructorType})."
     - "Este certificado tem validade até {expiresAt formatted dd/MM/yyyy}."
     - Footer: date + instructor signature line + employer signature line
   - Return Buffer
