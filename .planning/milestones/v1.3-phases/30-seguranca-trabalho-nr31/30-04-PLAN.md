**safety-compliance.service.ts** — Read-only aggregation layer that joins EPI + training + ASO:

1. `getComplianceSummary(ctx, farmId?: string)`:
   - Get all active employees (status ACTIVE) in org, optionally filtered by farmId via EmployeeFarm
   - For each employee, compute compliance across 3 dimensions:
     a) **EPI compliance**: Get employee's Position → PositionEpiRequirements → check if employee has at least `quantity` EpiDeliveries per required EpiProduct where CA is not expired
     b) **Training compliance**: Get employee's Position → PositionTrainingRequirements + all isGlobal TrainingTypes → check if employee has EmployeeTrainingRecord where expiresAt > today
     c) **ASO compliance**: Get latest MedicalExam per employee → check nextExamDate vs today using classifyExpiryAlert
   - Aggregate: employee is "compliant" only if ALL three dimensions are OK (no YELLOW/RED/EXPIRED)
   - Return ComplianceSummary: { totalEmployees, compliantCount, compliantPercent, pendingCount, expiringIn30Days }
   - `expiringIn30Days` = count of unique employees with ANY item (CA, training, ASO) expiring within 30 days

   **Performance optimization:** Do NOT iterate per-employee in N+1 queries. Instead:
   - Query 1: All PositionEpiRequirements for positions held by active employees
   - Query 2: All EpiDeliveries for those employees (latest per epiProductId)
   - Query 3: All PositionTrainingRequirements + global training types
   - Query 4: All EmployeeTrainingRecords for those employees (where expiresAt relevant)
   - Query 5: Latest MedicalExam per employee (GROUP BY employeeId, MAX date)
   - Merge in memory by employeeId

2. `listNonCompliantEmployees(ctx, query: ComplianceDashboardQuery)`:
   - Same 5 queries as getComplianceSummary but returns EmployeeComplianceOutput[] with details
   - Filter by pendingType (EPI/TRAINING/ASO) — only return employees with issues in that dimension
   - Filter by search (employee name LIKE)
   - Paginate

3. `getEmployeeCompliance(ctx, employeeId: string)`:
   - Full compliance detail for single employee across all 3 dimensions

4. `generateComplianceReportCsv(ctx, farmId?: string)`:
   - Generate CSV with columns: Nome, Cargo, Tipo Pendencia, Detalhe, Vencimento
   - One row per pending item per employee
   - Return string (CSV content)

5. `generateComplianceReportPdf(ctx, farmId?: string)`:
   - Dynamic import pdfkit
   - Build A4 PDF:
     - Header: "RELATÓRIO DE CONFORMIDADE NR-31" + farm name + date
     - Summary section: total, conformes, pendentes
     - Section "EPIs Pendentes": table of employees with missing EPIs
     - Section "Treinamentos Vencidos": table of employees with expired training
     - Section "ASOs Pendentes": table of employees with expired/missing ASOs
     - Footer: "Gerado automaticamente pelo sistema Protos Farm"
   - Return Buffer
