    **fiscal-periods.service.ts:**
    Import `date-fns` for month iteration (`eachMonthOfInterval`, `getMonth`, `getYear`).

    - `createFiscalYear(prisma, organizationId, input: CreateFiscalYearInput)` — Parse startDate/endDate. Validate: endDate > startDate, no overlapping year for org (query existing years where ranges overlap). Create FiscalYear row. Then auto-generate AccountingPeriod rows for each month in the range using `eachMonthOfInterval({ start: parsedStart, end: parsedEnd })`. For each month: `{ organizationId, fiscalYearId, month: getMonth(d) + 1, year: getYear(d), status: 'OPEN' as const }`. Use `prisma.accountingPeriod.createMany()` for batch insert. Return the created fiscal year with periods.

    - `getFiscalYears(prisma, organizationId)` — `prisma.fiscalYear.findMany({ where: { organizationId }, include: { accountingPeriods: { orderBy: [{ year: 'asc' }, { month: 'asc' }] } }, orderBy: { startDate: 'desc' } })`.

    - `closePeriod(prisma, organizationId, periodId, input: ClosePeriodInput)` — Find period. Validate current status is OPEN (if CLOSED throw INVALID_TRANSITION "Periodo ja esta fechado", if BLOCKED throw INVALID_TRANSITION "Periodo bloqueado"). Update: `status: 'CLOSED' as const, closedAt: new Date(), closedBy: input.closedBy`.

    - `reopenPeriod(prisma, organizationId, periodId, input: ReopenPeriodInput)` — Find period. Validate current status is CLOSED. Validate `reopenReason` is non-empty (throw REASON_REQUIRED). Update: `status: 'OPEN' as const, reopenedAt: new Date(), reopenedBy: input.reopenedBy, reopenReason: input.reopenReason`.

    - `blockPeriod(prisma, organizationId, periodId)` — Find period. Validate current status is OPEN or CLOSED (not already BLOCKED). Update: `status: 'BLOCKED' as const`.

    - `getPeriodForDate(prisma, organizationId, date: Date)` — Determine month and year from date. Find the accounting period where `month` and `year` match and the fiscalYear is active. Return the period or null.

    Valid transitions:
    - OPEN -> CLOSED (closePeriod)
    - CLOSED -> OPEN (reopenPeriod, requires reason)
    - OPEN -> BLOCKED (blockPeriod)
    - CLOSED -> BLOCKED (blockPeriod)
    - All other transitions throw INVALID_TRANSITION

    **fiscal-periods.routes.ts:**
    - `GET /api/org/:orgId/fiscal-years` — calls `getFiscalYears`. Returns 200.
    - `POST /api/org/:orgId/fiscal-years` — calls `createFiscalYear`. Returns 201.
    - `GET /api/org/:orgId/fiscal-years/:yearId/periods` — returns periods for a fiscal year.
    - `POST /api/org/:orgId/accounting-periods/:periodId/close` — calls `closePeriod`. Returns 200.
    - `POST /api/org/:orgId/accounting-periods/:periodId/reopen` — calls `reopenPeriod`. Returns 200.
    - `POST /api/org/:orgId/accounting-periods/:periodId/block` — calls `blockPeriod`. Returns 200.
    - `GET /api/org/:orgId/accounting-periods/for-date?date=YYYY-MM-DD` — calls `getPeriodForDate`. Returns 200 or 404.
    - All routes use `req.params.orgId as string`, `req.params.yearId as string`, `req.params.periodId as string` per Express 5 rules.

    - Mount: `app.use('/api/org/:orgId', requireAuth, fiscalPeriodsRouter);`
    - The router internally defines `/fiscal-years` and `/accounting-periods` sub-paths.
