    **2. Create `rural-credit.service.ts`:**

    Implement these exported functions (all use `withRlsContext` for RLS):

    - `simulateSchedule(input)` — calls `generateSchedule()` from shared, returns ScheduleRow[] serialized to plain objects. No DB access.

    - `createContract(ctx, input)` — single `withRlsContext` transaction:
      1. Validate farmId and bankAccountId belong to org
      2. Call `generateSchedule()` with input params
      3. Create `RuralCreditContract` with `outstandingBalance = principalAmount`
      4. For each schedule row, create `Payable` (category='FINANCING', originType='RURAL_CREDIT', originId=contract.id, supplierName=bank name, description=`Parcela ${n}/${total} - ${creditLineLabel}`, totalAmount=row.totalPayment, dueDate=row.dueDate, bankAccountId, installmentCount=1) + `PayableInstallment` (number=1, amount=row.totalPayment, dueDate=row.dueDate) + `RuralCreditInstallment` (contractId, payableId, installmentNumber, principal=row.principal.toDecimal(), interest=row.interest.toDecimal(), outstandingBalanceAfter=row.outstandingBalance.toDecimal())
      5. Register credit release: `tx.bankAccountBalance.update` with `increment: Money(principalAmount).toDecimal()` + `tx.financialTransaction.create` with type='CREDIT', referenceType='RURAL_CREDIT_RELEASE', referenceId=contract.id
      6. `logAudit` for contract creation
      7. Return contract with installments

    - `listContracts(ctx, query)` — list with filters: farmId, status, creditLine. Include computed `nextPayment` (earliest PENDING PayableInstallment dueDate+amount via subquery). Sort by createdAt DESC.

    - `getContract(ctx, id)` — fetch contract with all RuralCreditInstallments (joined to Payable for status). Recompute and persist status via `computeContractStatus()`. Return full detail.

    - `computeContractStatus(contract, payables)` — pure function: if status=='CANCELADO' return CANCELADO; if all payables status=='PAID' return QUITADO; if any PENDING payable dueDate > 30 days overdue return INADIMPLENTE; else ATIVO. 30 days hardcoded constant `OVERDUE_THRESHOLD_DAYS = 30`.

    - `updateContract(ctx, id, input)` — for non-schedule fields (contractNumber, notes, alertDaysBefore, guaranteeDescription): simple update. For schedule fields (annualRate, termMonths, etc.): delete all PENDING Payables + their RuralCreditInstallments, recompute schedule from current outstandingBalance, create new Payables. Keep PAID payables intact.

    - `cancelContract(ctx, id)` — set status=CANCELADO, cancelledAt=now(). Find all Payables where originType='RURAL_CREDIT' AND originId=contract.id AND status='PENDING', update status to 'CANCELLED'. Set outstandingBalance=0.

    - `settleInstallment(ctx, contractId, payableId, input)` — single transaction:
      1. Find RuralCreditInstallment by contractId+payableId
      2. Settle the PayableInstallment (same logic as settlePayment: create FinancialTransaction DEBIT, decrement BankAccountBalance, update installment status to PAID, update payable status to PAID)
      3. Update contract: `outstandingBalance -= meta.principal`, `totalPrincipalPaid += meta.principal`, `totalInterestPaid += meta.interest`
      4. Recompute status

    - `applyExtraordinaryAmortization(ctx, contractId, input)` — single transaction:
      1. Validate contract is ATIVO
      2. Debit bank account for extraAmount
      3. Create FinancialTransaction DEBIT (referenceType='RURAL_CREDIT_AMORTIZATION')
      4. Update contract: outstandingBalance -= extraAmount, totalPrincipalPaid += extraAmount
      5. Delete all PENDING Payables + RuralCreditInstallments
      6. If REDUCE_TERM: recompute with same payment amount, fewer months. If REDUCE_INSTALLMENT: recompute with same term, lower outstanding balance
      7. Create new Payables + RuralCreditInstallments from recomputed schedule

    - `getAlertCount(ctx)` — count distinct contracts where: status='ATIVO' AND exists a PENDING PayableInstallment with dueDate within contract's alertDaysBefore days from today. Lightweight query for sidebar badge.

    **3. Create `rural-credit.routes.ts`:**

    Router mounted at `/org/rural-credit`. All routes use `authenticate` + `authorize('financial', action)`.

    **CRITICAL route ordering** — register fixed paths BEFORE `/:id`:

    ```
    POST   /simulate          → simulateSchedule (financial:read)
    GET    /alert-count       → getAlertCount (financial:read)
    GET    /                  → listContracts (financial:read)
    POST   /                  → createContract (financial:create)
    GET    /:id               → getContract (financial:read)
    PUT    /:id               → updateContract (financial:update)
    DELETE /:id/cancel        → cancelContract (financial:delete)
    POST   /:id/settle-installment/:payableId → settleInstallment (financial:update)
    POST   /:id/extraordinary-amortization    → applyExtraordinaryAmortization (financial:update)
    ```

    Error handling: catch RuralCreditError and return `res.status(err.statusCode).json({ error: err.message })`, catch unknown errors as 500.
