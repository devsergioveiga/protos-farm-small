## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status chip padding, inline gaps between icon and label |
| sm | 8px | Compact table cell padding, badge inner padding |
| md | 16px | Default element spacing, form field gaps, card padding |
| lg | 24px | Section padding inside panels, wizard step body padding |
| xl | 32px | Layout column gaps, between header and body |
| 2xl | 48px | Major section breaks, page top padding |
| 3xl | 64px | Empty state icon size |

Exceptions:
- Touch targets minimum 48x48px — applies to all interactive elements (buttons, wizard nav arrows, row action icons).
- Input padding: 12px vertical × 16px horizontal — minimum for field legibility in field conditions.
- Wizard step header height: 64px fixed — accommodates stepper dots + step label.

## Wizard Specification (D-01)

The wizard is a modal dialog (`role="dialog"`) with a fixed header (stepper + title), scrollable body, and fixed footer (navigation buttons).

### Steps

| Step | Title | Content | Validation to advance |
|------|-------|---------|----------------------|
| 1 | Competência | Select mês/ano (native `<select>`s) + runType radio group (Mensal / Adiantamento / 13ª Parcela 1 / 13ª Parcela 2) | mês, ano, and runType selected |
| 2 | Colaboradores | Read-only list — employee name, status do ponto chip (APPROVED / PENDING), salary. Employees with APPROVED timesheet show green check. Employees with PENDING_TIMESHEET show warning chip. Counter shows "X de Y colaboradores aptos" | at least 1 eligible employee |
| 3 | Confirmação | Checkbox list — all eligible pre-checked; contador can uncheck to exclude. Ineligible employees shown greyed out with tooltip "Espelho de ponto não aprovado". Shows "X colaboradores serão processados". | at least 1 employee checked |
| 4 | Processando | Non-interactive. Shows spinner + progress text "Processando X de Y colaboradores...". Auto-advances to success/error state on completion | n/a — wait for API |

### Wizard footer buttons

| Position | Step 1 | Steps 2–3 | Step 4 |
|----------|--------|-----------|--------|
| Left | "Cancelar" (text button) | "Voltar" (secondary button) | hidden |
| Right | "Próximo" (primary, accent) | "Próximo" / "Processar" on step 3 (primary, accent) | hidden (auto-progresses) |

Wizard header: 4 step indicator dots. Active dot: filled accent. Completed dots: filled primary-200. Future dots: neutral-300 outline. Current step label shown below dots.

Source: D-01 from CONTEXT.md, frontend CLAUDE.md wizard pattern.

## Page Layout: PayrollRunsPage

- Breadcrumb: RH > Folha de Pagamento
- Header row: page title "Folha de Pagamento" (DM Sans 20px 700) + primary CTA button "Iniciar Folha" (right-aligned, 1 per page)
- Filter row: mês/ano selects + runType filter select + fazenda filter select (all inline, left-aligned)
- Tab bar: "Rodadas" | "Adiantamentos" — active tab has 2px accent underline
- Table (Rodadas tab): columns — COMPETÊNCIA, TIPO, COLABORADORES, BRUTO, ENCARGOS, LÍQUIDO, STATUS, AÇÕES
- Table (Adiantamentos tab): columns — DATA, COLABORADOR, VALOR, STATUS CP, AÇÕES
- Table rows: 48px min-height (touch target). Hover: neutral-100 background.
- AÇÕES column: icon buttons only — `Eye` (view), `Download` (ZIP), `Mail` (reenviar), `RotateCcw` (estornar). Each has `aria-label`. Destructive actions (estorno) use error-500 color with danger variant.
- Mobile: table collapses to stacked cards below 768px. Each card shows competência, tipo chip, status chip, líquido value, and expand chevron to see details.

## Estorno Flow (D-03)

Estorno is high-criticality. Pattern: `ConfirmDeleteModal` with typing confirmation.

- Trigger: "Estornar" button in PayrollRunDetailModal (only visible when status=COMPLETED)
- Button: secondary button with `color: var(--color-error-500)`, icon `RotateCcw` 20px + label "Estornar Folha"
- Confirmation modal: user must type the run reference (e.g. "2026-03 MENSAL") to confirm
- Modal title: "Estornar Folha de Pagamento"
- Modal body: "Esta ação é irreversível. Todos os holerites serão destruídos, as Contas a Pagar serão canceladas e os espelhos de ponto serão liberados. Digite [referência] para confirmar."
- Confirm button: "Estornar" with error-500 background, disabled until typed text matches

Source: D-03 from CONTEXT.md, CLAUDE.md ConfirmDeleteModal rule.

## Accessibility Requirements

- All wizard steps: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to step title.
- Wizard `Escape` key: closes wizard and discards uncommitted changes (steps 1–3 only; step 4 processing cannot be cancelled).
- Status chips: include `role="status"` only when dynamically updating (PROCESSING → CALCULATED). Static chips use plain `<span>`.
- PROCESSING chip with spinning `Loader2`: add `aria-label="Processando"` to the icon element.
- Tables: `<th scope="col">` on all column headers. Currency columns right-aligned.
- Monetary values: use `<span class="font-mono" aria-label="R$ [value]">R$ [value]</span>` pattern — not required to read "reais" aloud but column context makes it clear.
- Destructive buttons: `aria-describedby` pointing to a visually hidden warning text before the ConfirmDeleteModal opens.
- Focus trap in wizard and in confirmation modal — Tab cycles within the modal.
- After wizard closes (success or cancel): return focus to "Iniciar Folha" trigger button.

## Animation Timing

| Interaction | Duration | Curve | Notes |
|-------------|----------|-------|-------|
| Wizard step transition | 200ms | ease-out | Slide next step in from right; slide out to left |
| Modal open | 300ms | ease-out | Opacity 0→1 + translateY(8px→0) |
| Modal close | 200ms | ease-in | Opacity 1→0 |
| Status chip update | 200ms | ease-out | Background color transition when status changes |
| Skeleton pulse | 1.5s infinite | ease-in-out | Opacity 0.4→0.7 |
| Toast enter | 200ms | ease-out | translateX(100%→0) from right |
| Toast exit | 200ms | ease-in | translateX(0→100%) |
| `Loader2` spin | 1s linear infinite | n/a | CSS animation `spin` |

`prefers-reduced-motion: reduce` — all transitions and animations disabled; state changes are instant.
