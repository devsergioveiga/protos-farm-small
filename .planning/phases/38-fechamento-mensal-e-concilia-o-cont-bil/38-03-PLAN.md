    **1. MonthlyClosingPage.tsx — Vertical Stepper Page (D-01, D-03, D-04, D-05, D-06):**

    Page reads `periodId` from URL search params (`useSearchParams`). If no periodId, show message "Selecione um periodo na pagina de Periodos Fiscais."

    **Layout structure:**
    ```
    Breadcrumb: Home > Contabilidade > Fechamento Mensal
    Header: "Fechamento Mensal — {MES}/{ANO}" with status badge (Em Andamento / Concluido / Reaberto)

    Vertical Stepper (6 steps):
      Each step is a card/row with:
        - Step number circle (1-6) with color: green (OK), red (FAILED), gray (PENDING), dimmed (BLOCKED)
        - Step label from STEP_LABELS
        - Status badge: "Aprovado" (green), "Falhou" (red), "Pendente" (gray), "Bloqueado" (dimmed)
        - Summary text (from stepResults)
        - Expand/collapse chevron for details
        - When FAILED: link to module (STEP_MODULE_LINKS) + "Revalidar" button (D-05)
        - When PENDING and previous step OK: "Validar" button
        - When BLOCKED (previous step not OK): no button, dimmed appearance

    Footer actions:
      - "Fechar Periodo" button — primary, disabled unless all 6 steps are OK. Uses ConfirmModal (variant="warning") before calling completeClosing.
      - "Reabrir Periodo" button — only shown when status COMPLETED and user.role is ADMIN/SUPER_ADMIN. Uses ConfirmModal (variant="warning") requiring reason text input.
    ```

    **Behavior flow:**
    1. On mount: call `useMonthlyClosing(periodId)` to load existing closing
    2. If no closing exists and period is OPEN: show "Iniciar Fechamento" button that calls `useStartClosing().startClosing(periodId)`
    3. If closing exists with status IN_PROGRESS: show stepper with current stepResults
    4. User clicks "Validar" on step N → `useValidateStep().validateStep(closingId, N)` → refetch closing to update stepResults
    5. If step fails: show error summary + module link + "Revalidar" button (same as Validar)
    6. When all 6 steps OK: "Fechar Periodo" button enables → ConfirmModal → `useCompleteClosing().completeClosing(closingId)` → toast "Periodo fechado com sucesso" → refetch
    7. Reopen: ConfirmModal with textarea for reason → `useReopenClosing().reopenClosing(closingId, reason)` → toast "Periodo reaberto" → refetch

    **Status determination for each step:**
    - `stepResults[stepN]?.status === 'OK'` → green, show summary
    - `stepResults[stepN]?.status === 'FAILED'` → red, show summary + module link + Revalidar
    - `stepResults[stepN]` is undefined and previous step OK (or step 1) → gray PENDING, show Validar button
    - `stepResults[stepN]` is undefined and previous step NOT OK → dimmed BLOCKED, no button

    **Icons from lucide-react:**
    - Step OK: `CheckCircle2` (green)
    - Step FAILED: `XCircle` (red)
    - Step PENDING: `Circle` (gray)
    - Step BLOCKED: `Lock` (dimmed)
    - Completed closing: `ShieldCheck`
    - Reopened: `Unlock`

    Use `useNavigate` for module links. Use `useSearchParams` for periodId.

    **IMPORTANT CSS rules per CLAUDE.md:**
    - Font: DM Sans for headings, Source Sans 3 for body
    - Colors via CSS custom properties only: `var(--color-primary-600)`, `var(--color-error-500)`, `var(--color-neutral-*)` etc.
    - Spacing: 4px grid multiples only
    - Touch targets minimum 48x48px
    - HTML semantic: `<main>`, `<section>`, `<button>` (never div onClick)
    - Focus visible on all interactive elements
    - aria-label on icon-only buttons
