### EM_COTACAO -> OC_EMITIDA Transition (CHANGED)

The `EM_COTACAO` column drop zone lights up as a valid target (unchanged â€” `VALID_TRANSITIONS` map entry kept). When the user drops a card:

1. `ConfirmModal` opens with updated copy (see Copywriting section below)
2. On confirm: `navigate('/quotations?purchaseRequestId={card.id}')` â€” no API call
3. Modal closes, user lands on Quotations page filtered to the RC

Animation: Modal entrance at 300ms (`--duration-normal`) with `ease-out` â€” existing `ConfirmModal` behavior, no change.

### OC_EMITIDA -> AGUARDANDO_ENTREGA Transition (BUG FIX â€” no visual change)

Drop behavior is identical to before from the user's perspective. The fix is internal: changes the API call from `PUT /purchase-orders/{id}/status` to `PATCH /purchase-orders/{id}/transition` with `{ status: 'EM_TRANSITO' }`. The `ConfirmModal` copy for this transition is unchanged.

## Accessibility Contract

No new interactive elements. Existing `ConfirmModal` contract applies:

- Focus moves into modal on open; returns to trigger element on close
- `Escape` closes modal without confirming
- Confirm button: `<button type="button">` with visible text label (no aria-label needed â€” always has text)
- Modal root: `role="dialog"` with `aria-modal="true"` and `aria-labelledby` pointing to title â€” existing implementation, no change
