Create `purchase-orders.service.ts` with these exported functions:

1. **`createEmergencyPO(ctx: RlsContext & { userId: string }, input: CreateEmergencyPOInput)`**
   - Validate supplier exists and is ACTIVE; throw PurchaseOrderError(400) if blocked/inactive
   - Validate justification is non-empty (required for emergency)
   - Generate OC-YYYY/NNNN using findFirst+startsWith pattern scoped to organizationId + year
   - Create PurchaseOrder with isEmergency=true, status=RASCUNHO, quotationId=null
   - Create PurchaseOrderItem rows with snapshot fields (productName, unitName, quantity, unitPrice, totalPrice=unitPrice\*quantity)
   - Return created PO with items and supplier

2. **`duplicatePO(ctx: RlsContext & { userId: string }, input: DuplicatePOInput)`**
   - Load source PO with items; throw 404 if not found
   - Generate new OC-YYYY/NNNN
   - Create new PO in RASCUNHO status with same supplierId, copy all items (new IDs, same snapshot data)
   - Set isEmergency=false, quotationId=null, notes=input.notes or `"Duplicado de ${source.sequentialNumber}"`
   - Return new PO

3. **`listPurchaseOrders(ctx: RlsContext, query: ListPurchaseOrdersQuery)`**
   - Paginated (default 20, max 100)
   - Filter by status, supplierId, isEmergency, overdue (where expectedDeliveryDate < now AND status in EMITIDA/CONFIRMADA/EM_TRANSITO)
   - Search by sequentialNumber (contains, insensitive) or supplier.name
   - Include supplier (select: id, name, tradeName), item count, quotation sequentialNumber
   - Computed field: isOverdue = expectedDeliveryDate && expectedDeliveryDate < now && status not in ENTREGUE/CANCELADA
   - Filter `deletedAt: null`
   - Return `{ data, total, page, limit, totalPages }`

4. **`getPurchaseOrderById(ctx: RlsContext, id: string)`**
   - Include supplier, items, quotation (with purchaseRequest sequentialNumber), creator
   - Throw PurchaseOrderError(404) if not found or deleted

5. **`updatePO(ctx: RlsContext, id: string, input: UpdatePOInput)`**
   - BLOCK if status !== RASCUNHO: throw PurchaseOrderError(400, 'OC emitida nao pode ser editada')
   - Update notes, internalReference, expectedDeliveryDate
   - If items provided: delete existing items and create new ones (full replace)
   - Return updated PO

6. **`transitionPO(ctx: RlsContext, id: string, input: TransitionPOInput)`**
   - Guard with canOcTransition; throw PurchaseOrderError(400) if invalid
   - Special handling per target status:
     - EMITIDA: set issuedAt = new Date()
     - CONFIRMADA: set confirmedAt = new Date()
     - CANCELADA: set cancelledAt = new Date()
   - Update status
   - Return updated PO

7. **`deletePO(ctx: RlsContext, id: string)`**
   - Only if status === RASCUNHO; throw 400 otherwise
   - Soft delete: deletedAt = new Date()

8. **`generatePurchaseOrderPdf(ctx: RlsContext, id: string, res: Response)`**
   - Load PO with supplier, items, quotation, creator, organization
   - Throw 404 if not found
   - Create PDFDocument with margin 50
   - Set response headers: Content-Type application/pdf, Content-Disposition attachment filename="OC-{sequentialNumber}.pdf"
   - Pipe doc to res
   - PDF layout sections:
     a. **Header:** Organization name (bold, 18pt), address, CNPJ
     b. **Title:** "PEDIDO DE COMPRA" centered, OC number, date
     c. **Supplier block:** Name, CNPJ, address, contact
     d. **Items table:** columns: #, Produto, Unidade, Qtd, Preco Unit., Total — use Number() for Decimal fields
     e. **Totals:** Subtotal, notes
     f. **Conditions:** Payment terms (from quotation proposal if available), expected delivery date
     g. **Footer:** "Documento gerado em {date}" + organization name
   - doc.end()

9. **`checkOverduePOs(ctx: RlsContext)`**
   - Find POs where expectedDeliveryDate < now AND status in ['EMITIDA','CONFIRMADA','EM_TRANSITO'] AND overdueNotifiedAt is null AND deletedAt is null
   - For each: create notification (type='PO_OVERDUE', referenceId=po.id, referenceType='purchase_order') and set overdueNotifiedAt=now()
   - Return count of overdue POs notified

**Step 1: Create `purchase-orders.routes.ts`** — Express Router at `/org/purchase-orders`:

```
GET    /                 — listPurchaseOrders (purchases:read)
POST   /                 — createEmergencyPO (purchases:manage)
POST   /duplicate        — duplicatePO (purchases:manage) — BEFORE /:id
GET    /:id/pdf          — generatePurchaseOrderPdf (purchases:read) — BEFORE /:id
GET    /:id              — getPurchaseOrderById (purchases:read)
PATCH  /:id/transition   — transitionPO (purchases:manage)
PATCH  /:id              — updatePO (purchases:manage)
DELETE /:id              — deletePO (purchases:manage)
```

CRITICAL: Register `/duplicate` and `/:id/pdf` routes BEFORE `/:id` to prevent Express matching them as IDs.

PDF handler: call `generatePurchaseOrderPdf(ctx, id, res)` — the service function handles streaming directly to response. Do NOT send res.json() after calling it.

Error handling: catch PurchaseOrderError, return err.statusCode + { error: err.message }. For PDF handler, if error occurs before pipe starts, return 4xx JSON.
