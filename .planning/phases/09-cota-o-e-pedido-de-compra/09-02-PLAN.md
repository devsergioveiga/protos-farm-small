**Step 2: Create `quotations.service.ts`** with these exported functions:

1. **`createQuotation(ctx: RlsContext & { userId: string }, input: CreateQuotationInput)`**
   - Validate RC exists and `status === 'APROVADA'`; throw QuotationError(400) if not
   - Validate all supplierIds exist, are ACTIVE status, and belong to org; throw 400 if any blocked
   - Generate SC-YYYY/NNNN using same findFirst+startsWith pattern as purchase-requests (scoped to organizationId + year)
   - Create Quotation with status RASCUNHO, plus QuotationSupplier rows for each supplier
   - Transition to AGUARDANDO_PROPOSTA immediately after creation
   - Return created quotation with suppliers included

2. **`listQuotations(ctx: RlsContext, query: ListQuotationsQuery)`**
   - Paginated (default 20, max 100)
   - Filter by status, purchaseRequestId
   - Search by sequentialNumber or purchaseRequest.sequentialNumber (contains, insensitive)
   - Include purchaseRequest (select: id, sequentialNumber, type, urgency), supplier count
   - Filter `deletedAt: null`
   - Return `{ data, total, page, limit, totalPages }`

3. **`getQuotationById(ctx: RlsContext, id: string)`**
   - Include purchaseRequest with items, quotationSuppliers with supplier and proposal+proposalItems
   - Include quotationItemSelections
   - Throw QuotationError(404) if not found or deleted

4. **`registerProposal(ctx: RlsContext, quotationId: string, quotationSupplierId: string, input: RegisterProposalInput, file?: { url: string; name: string })`**
   - Validate quotation exists, status is AGUARDANDO_PROPOSTA or EM_ANALISE
   - Validate quotationSupplierId belongs to this quotation
   - Validate each item's purchaseRequestItemId belongs to the quotation's purchaseRequest
   - Calculate totalPrice = unitPrice \* quantity for each item (using Number() for Decimal safety)
   - Upsert QuotationProposal (create or update if already exists) with items
   - If file provided, set fileUrl and fileName on proposal
   - If this is the last supplier to submit (all suppliers have proposals), auto-transition SC to EM_ANALISE
   - Return updated quotation

5. **`getComparativeMap(ctx: RlsContext, quotationId: string): Promise<ComparativeMapData>`**
   - Load quotation with purchaseRequest.items and all suppliers+proposals+proposalItems
   - Build items array from RC items with productName, unitName, quantity
   - For lastPricePaid: query PurchaseOrderItem where purchaseRequestItemId matches AND purchaseOrder.status in ['EMITIDA','CONFIRMADA','EM_TRANSITO','ENTREGUE'], orderBy createdAt desc, take first unitPrice. Fall back to null.
   - Build suppliers array with proposal data, cast Decimal fields with Number()
   - Compute perItemMinPrice and perItemMaxPrice across all proposals that have items for each RC item
   - Return ComparativeMapData shape

6. **`approveQuotation(ctx: RlsContext & { userId: string }, quotationId: string, input: ApproveQuotationInput)`**
   - Validate SC status is EM_ANALISE; throw 400 if not
   - Validate all selectedItems reference valid quotationSupplierIds and purchaseRequestItemIds
   - Check if any selected supplier is NOT the lowest price for their item — if so, justification is REQUIRED; throw 400 if missing
   - In a SINGLE Prisma transaction ($transaction):
     a. Save QuotationItemSelection rows (delete existing + create new)
     b. Update Quotation: status=APROVADA, approvedBy=userId, approvedAt=now(), approvalJustification=input.justification
     c. Group selectedItems by quotationSupplierId to determine which suppliers won items
     d. For EACH winning supplier, create a PurchaseOrder:
     - Generate OC-YYYY/NNNN sequential number
     - Status = EMITIDA, issuedAt = now()
     - Copy selected items as PurchaseOrderItem with frozen unitPrice/quantity/totalPrice from QuotationProposalItem
     - Set productName and unitName from PurchaseRequestItem (snapshot)
       e. Transition SC to FECHADA after all OCs created
   - After transaction: fire-and-forget createNotification with referenceId=quotationId, referenceType='quotation', type='QUOTATION_APPROVED'
   - Return { quotation, purchaseOrders: createdPOs }

7. **`transitionQuotation(ctx: RlsContext, id: string, toStatus: string)`**
   - Guard with canScTransition; throw 400 if invalid
   - Update status
   - Return updated quotation

8. **`deleteQuotation(ctx: RlsContext, id: string)`**
   - Only allow if status is RASCUNHO; throw 400 otherwise
   - Soft delete: set deletedAt = new Date()

**Step 1: Create `quotations.routes.ts`** — Express Router at `/org/quotations` with these endpoints:

```
GET    /                          — listQuotations (purchases:read)
POST   /                          — createQuotation (purchases:manage)
GET    /:id/comparative            — getComparativeMap (purchases:read) — BEFORE /:id
GET    /:id                        — getQuotationById (purchases:read)
PATCH  /:id/transition             — transitionQuotation (purchases:manage)
POST   /:id/suppliers/:qsId/proposal — registerProposal (purchases:manage) with multer upload.single('file')
PATCH  /:id/approve                — approveQuotation (purchases:manage)
DELETE /:id                        — deleteQuotation (purchases:manage)
```

CRITICAL: Register `/`:id/comparative`route BEFORE`/:id` route to prevent Express matching "comparative" as an ID.

Each handler follows the purchase-requests pattern:

- Extract ctx from `req.user` (organizationId from RLS)
- Call service function
- Return JSON with appropriate status (201 for create, 200 for others)
- Catch module-specific errors (QuotationError) and return err.statusCode + message
- Multer config: `multer({ dest: 'uploads/', limits: { fileSize: 10 * 1024 * 1024 } })` for proposal upload

For the proposal upload handler, after multer processes the file, pass `{ url: req.file.path, name: req.file.originalname }` to registerProposal.
