**Models to add (7 total):**

1. `Quotation` — org-scoped, links to PurchaseRequest, has sequentialNumber unique per org, status defaults to RASCUNHO, createdBy/approvedBy user FKs, deletedAt for soft delete
2. `QuotationSupplier` — links quotation to supplier, unique(quotationId, supplierId), has optional isSelected boolean
3. `QuotationProposal` — 1:1 with QuotationSupplier (unique quotationSupplierId), freight/tax Decimal(12,2), paymentTerms, validUntil, deliveryDays, fileUrl/fileName for upload
4. `QuotationProposalItem` — unitPrice Decimal(12,4), quantity Decimal(12,3), totalPrice Decimal(14,2), unique(proposalId, purchaseRequestItemId)
5. `QuotationItemSelection` — NEW model for split selection: `id cuid(), quotationId, purchaseRequestItemId, quotationSupplierId, unique(quotationId, purchaseRequestItemId)` — represents "item X is sourced from supplier Y"
6. `PurchaseOrder` — org-scoped, optional quotationId (null for emergency), supplierId, sequentialNumber unique per org, isEmergency boolean, expectedDeliveryDate, issuedAt, overdueNotifiedAt, cancelledAt, createdBy user FK, deletedAt
7. `PurchaseOrderItem` — snapshot fields: productName String, unitName String, quantity Decimal(12,3), unitPrice Decimal(12,4), totalPrice Decimal(14,2), optional purchaseRequestItemId
