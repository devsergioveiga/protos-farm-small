## Page Layouts

### TransfersPage

- **Layout:** Page header (title "Transferências" + "Nova Transferência" CTA) → filter bar (date range + type enum + conta origem/destino) → data table
- **Table columns:** Data | Tipo (badge) | Conta Origem | Conta Destino | Valor | Tarifa | Descrição | Ações
- **Type badges:** INTERNA (neutral), TED (info), APLICACAO (success), RESGATE (info) — colored pill, max 80px wide
- **Responsive <768px:** Table collapses to stacked cards showing: Data + Tipo badge / Origem → Destino (ArrowRight icon) / Valor em destaque (mono font) / Tarifa if present
- **Empty state:** ArrowLeftRight icon (48px, `--color-neutral-300`) + "Nenhuma transferência registrada" + "Registre movimentações entre contas bancárias." + "Nova Transferência" CTA button

### CreditCardsPage

- **Layout:** Two-panel within page. Left: list of card cards (max 280px, hidden <1024 → stacked above on mobile). Right: selected card detail with bill tabs.
- **Card card element:** Card brand icon + name + last 4 digits + limit + closing day | usage bar (spent/limit) | badge "Fatura aberta" if OPEN bill exists
- **Bill detail:** Tab per billing period (current + last 2). Tab content: period dates header + expense table (Data | Descrição | Parcela | Valor) + total footer + "Fechar Fatura" button (only if status OPEN)
- **Fatura FECHADA:** Shows linked CP status badge ("Pago" or "Pendente") with link to payables
- **Responsive <1024px:** Card list stacks above bill detail (accordion pattern — tap card to expand its detail)
- **Empty state (no cards):** CreditCard icon (48px) + "Nenhum cartão cadastrado" + "Cadastre um cartão corporativo para controlar faturas." + "Novo Cartão" CTA

### ChecksPage

- **Layout:** Page header (title "Cheques" + "Novo Cheque" CTA) → filter bar (status multiselect + type EMITIDO/RECEBIDO + date range) → data table with row actions
- **Table columns:** N° Cheque | Tipo (EMITIDO/RECEBIDO badge) | Beneficiário/Emitente | Valor | Data Emissão | Data Compensação Prevista | Status (state badge) | Ações
- **Status badges:** EMITIDO (neutral/gray) | A_COMPENSAR (warning/amber) | COMPENSADO (success/green) | DEVOLVIDO (error/red) | CANCELADO (neutral/strikethrough text)
- **Row actions per status:**
  - EMITIDO: "Marcar A Compensar" | "Cancelar"
  - A_COMPENSAR: "Compensar" | "Devolver" | "Cancelar"
  - DEVOLVIDO: "Re-apresentar" (→ A_COMPENSAR) | "Cancelar"
  - COMPENSADO / CANCELADO: "Ver detalhes" (read-only)
- **Responsive <768px:** Cards with status badge pill prominent, value in mono font large, action buttons below
- **Empty state:** CheckSquare icon (48px) + "Nenhum cheque registrado" + "Controle cheques emitidos e recebidos com rastreamento de compensação." + "Novo Cheque" CTA

| Field         | Type           | Required | Validation                                |

### Close Bill Modal (CloseBillModal)

- Shows: period summary (from → to) + expense count + total amount in large mono font
- Single confirm button: "Fechar fatura e gerar CP de R$ X,XX"
- Warning note: "Após fechar, não é possível adicionar despesas a esta fatura."
- If bill has 0 expenses: button is disabled + inline message "Adicione ao menos uma despesa antes de fechar."

### Check Modal fields

| Field                        | Type                       | Required | Validation                              |
| ---------------------------- | -------------------------- | -------- | --------------------------------------- |
| Tipo                         | Radio (EMITIDO / RECEBIDO) | Yes      | —                                       |
| N° Cheque                    | Text                       | Yes      | string livre, max 20 chars              |
| Beneficiário / Emitente      | Text                       | Yes      | max 100 chars (label changes with Tipo) |
| Conta Bancária               | Select                     | Yes      | —                                       |
| Valor                        | Currency input             | Yes      | > 0                                     |
| Data de Emissão              | Date picker                | Yes      | —                                       |
| Data de Entrega              | Date picker                | No       | ≥ emissão                               |
| Data Prevista de Compensação | Date picker                | No       | —                                       |
| Observações                  | Textarea                   | No       | —                                       |

### Compensate Check Modal (CompensateCheckModal)

- Compact — shows: cheque number + value summary
- One field: "Data de compensação" (date, defaults to today, required)
- Confirm button: "Confirmar compensação"
- Note: "O saldo bancário real será atualizado neste momento."

### Destructive Actions

| Action                               | Confirmation approach                                                                                                                                                                     |
| ------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Cancelar cheque                      | Inline confirmation row expand: "Confirmar cancelamento do cheque Nº {n}? Esta ação não pode ser desfeita." + Confirmar/Cancelar buttons — no separate modal (low risk, recoverable data) |
| Fechar fatura (gera CP irreversível) | CloseBillModal with expense review — user must read summary before confirming                                                                                                             |
| Excluir cartão (if implemented)      | Typing confirmation: "Digite o nome do cartão para confirmar a exclusão" — per CLAUDE.md modal destructive pattern                                                                        |

## Accessibility Contract

All components must satisfy WCAG 2.1 AA:

- All modals: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal title `<h2>`
- Focus trap inside open modal; Escape closes modal
- Status badges: `aria-label` includes full text (e.g. `aria-label="Status: A compensar"`)
- Check state machine action buttons: `aria-label` includes cheque number (e.g. `aria-label="Compensar cheque Nº 001234"`)
- Monetary values in mono font: wrap in `<span aria-label="R$ {value} reais">` when value is meaningful in isolation
- Tooltip on accounting balance: triggered by Info button with `aria-label="Entender saldo contábil"`, tooltip uses `role="tooltip"` with `id` and `aria-describedby` on the trigger
- All form labels: `<label htmlFor>` — never placeholder-only
- Required fields: `aria-required="true"` + visual asterisk `*`
- Inline errors: `role="alert"` with ícone + message
- Tables: `<th scope="col">` headers, `<caption>` for screen readers

Source: CLAUDE.md Acessibilidade section, `apps/frontend/CLAUDE.md` HTML Semântico rules
