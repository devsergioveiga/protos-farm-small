Monetary and fiscal data values (asset tag, invoice number, installment amounts, NF-e total):

- Font: `JetBrains Mono` (`--font-mono`), 14px, weight 400
- Justification: right-align for numerical columns in installment preview tables
- NF-e parsed total (large preview card): 20px JetBrains Mono, weight 400

**Font weight exception вЂ” pre-existing project constraint:** This phase declares 4 weights (400, 500, 600, 700) across 2 font families. Source Sans 3 uses 400 (body) and 600 (label/UI), DM Sans uses 500 (display numerics) and 700 (headings). These are separate families serving distinct semantic roles; the 4-weight set is an inherent consequence of the 3-family type system committed in `CLAUDE.md` and `tokens.css`.

Constraints from CLAUDE.md:

- Minimum 12px вЂ” nothing smaller anywhere in this phase
- ALL CAPS only for short classification labels (e.g., "ГЂ VISTA", "FINANCIADO") вЂ” never in sentences
- Never center-align text blocks longer than 2 lines

**Source:** `apps/frontend/src/styles/tokens.css` (token values), `CLAUDE.md` (font family, min size, caps rule)

## Screen-by-Screen Interaction Contract

### AssetModal вЂ” "Dados Financeiros" Section (AQUI-01, AQUI-02, AQUI-07)

**Trigger:** Collapsible section below the existing "Dados de AquisiГ§ГЈo" section. Expanded by default when `acquisitionValue > 0` and in create mode. Collapsed when editing an asset with no acquisition data.

**Section header:** "Dados Financeiros" at `--text-sm` / `--font-semibold` / Source Sans 3 + `ChevronDown` / `ChevronUp` icon (16px) right-aligned. Click to toggle. Background `--color-neutral-100` on hover. No animation on expand/collapse вЂ” `display:none` / `display:block` per STATE.md locked decision for conditional modal fields.

**Fields inside section:**

1. **Tipo de pagamento** (required when `acquisitionValue > 0`): radio card group вЂ” "ГЂ Vista" and "Financiado". Each card: label + 1-line description (Source Sans 3 14px). "ГЂ Vista" shows single-installment dueDate field. "Financiado" shows financing fields below.
   - Unselected card: white background, `--color-neutral-200` border, `--radius-md`
   - Selected card: "ГЂ Vista" в†’ neutral `--color-neutral-100` background; "Financiado" в†’ `--color-sky-100` background вЂ” `--color-primary-600` border 2px
   - Focus: 2px `--color-primary-500` outline per accessibility rules
   - Radio input visually hidden; entire card is click target

2. **Vencimento** (shown when "ГЂ Vista" selected): date input, `aria-label="Data de vencimento"`, required when payment type present.

3. **Financing block** (shown only when "Financiado" selected вЂ” `display:none` when not selected):
   - NГєmero de parcelas: numeric input, min 1 max 360, label "NГєmero de parcelas *"
   - Primeira parcela: date input, label "Vencimento da 1ВЄ parcela *"
   - Taxa de juros a.m.: optional decimal input, label "Taxa de juros a.m. (%)", placeholder "0,00 вЂ” deixe em branco para parcelas iguais"
   - Below those fields: `InstallmentPreviewTable` вЂ” live-updating installment schedule preview (recalculates on blur of any financing field)

4. **Centro de custo** (shown in this section when not already visible in base form): selector pre-populated from asset's `costCenterId`. Required when `acquisitionValue > 0`.

5. **ClassificaГ§ГЈo contГЎbil**: read-only display of the asset's `classification` field (already set in the base form above). Shown as an informational chip вЂ” `--color-neutral-100` background, 14px Source Sans 3 вЂ” confirming which CPC applies to this acquisition.

**CP generation banner** (displayed below the section once all required fields are filled): info banner вЂ” `--color-info-100` background, `--color-info-500` left border 4px, `Info` icon 16px. Copy: "Uma conta a pagar serГЎ criada automaticamente ao salvar este ativo." No user action required; dismissible with `Г—` button.

### AssetNfeImportModal вЂ” 3-step wizard (AQUI-03, AQUI-04)

**Body content:**

#### Step 2 вЂ” Revisar Itens

**Despesas acessГіrias summary** (below item list if vFrete > 0 or vSeg > 0 or vOutro > 0):

(20px DM Sans 700) + "Conta a pagar registrada automaticamente." (16px Source Sans 3 `--color-neutral-700`) + "Ver ativos" link button

### InstallmentPreviewTable (AQUI-02)

**Purpose:** Read-only table rendered inside `AssetModal` "Dados Financeiros" section when payment type is "Financiado" and installment count is set.

## Interaction States

### "Dados Financeiros" Section вЂ” Conditional Rendering

All conditional sub-sections use `display:none` / `display:block` вЂ” zero animation, per STATE.md locked decision (same as Phase 17 DepreciationConfigModal and Phase 18 WorkOrderCloseWizard DIFERIMENTO field).

- Financing block (`installmentCount`, `firstDueDate`, `interestRate`, `InstallmentPreviewTable`): only shown when payment type = "FINANCIADO"
- Single due date field: only shown when payment type = "ГЂ VISTA"
- CP generation banner: only shown when `acquisitionValue > 0` AND `supplierId` is set AND at least one due date/payment field is filled

### Form Validation

- Inline validation on blur per field (not on submit)
- Required fields marked with `*` in label
- Error messages with `role="alert"` + `AlertCircle` 16px icon left of text, `--color-error-500`
- Currency fields (BRL decimal inputs): accept comma as decimal separator; format on blur to "R$ X.XXX,XX" using `Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' })`
- Percentage field (interest rate): accept comma as decimal separator; format on blur to "X,XX%"
- Date fields: Brazilian format day/month/year in display; ISO 8601 for API payload
- `installmentCount` field: integer only, min 2 when FINANCIADO (1 installment = ГЂ Vista), max 360; inline validation message "MГ­nimo 2 parcelas para financiamento"

### InstallmentPreviewTable вЂ” Live Update

Recalculates on every `onBlur` event on `installmentCount`, `firstDueDate`, or `interestRate` fields. Shows skeleton rows during recalculation. On calculation error (invalid inputs): hides table and shows inline field error instead.

### NF-e Upload вЂ” File Validation

- File type: accepts only `.xml` (MIME `text/xml` or `application/xml`)
- File rejected: upload zone border switches to `--color-error-500`, red error message below zone with `AlertCircle` 16px icon: "Apenas arquivos XML sГЈo aceitos."
- File too large (> 2MB): "O arquivo Г© muito grande. Tamanho mГЎximo permitido: 2 MB."
- Parse failure: error shown inside upload zone, file chip removed, upload zone resets to empty state

### NF-e Step 2 вЂ” Item Assignment

- Each item's asset dropdown is searchable (type to filter existing assets)
- "Criar novo ativo" option at top of dropdown list вЂ” when selected, item row shows inline mini-form: "Nome do ativo *" text input (pre-filled with `xProd` trimmed to 80 chars)
- Confirmar button disabled until all items have an assignment (existing asset OR new asset name provided)
