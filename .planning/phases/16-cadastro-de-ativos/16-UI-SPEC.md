## Interaction Specifications

### AssetModal — Type Selector

- Asset type is selected FIRST, before any other field renders
- Type options displayed as a horizontal button group: MAQUINA | VEICULO | IMPLEMENTO | BENFEITORIA | TERRA | EQUIPAMENTO
- Selected type gets `--color-primary-600` border + background tint `--color-primary-50`
- Each button minimum 48px height (touch target rule)
- Type-specific field sections render below the common fields with a `<section>` heading (DM Sans 500 18px)
- On type change: type-specific fields clear; common fields (name, farm, acquisition) persist

### AssetModal — Common Sections (all types)

1. **Identificação** — name*, asset tag (auto-generated, read-only with JetBrains Mono), fazenda*, status\*
2. **Aquisição** — data aquisição, valor aquisição (R$), fornecedor, número NF, centro de custo
3. **Tipo-específico** — rendered conditionally (see below)
4. **Fotos** — multi-photo drag-and-drop zone, max 5 photos, thumbnail preview with remove button

### AssetModal — Type-Specific Sections

| Type        | Fields                                                                                      |
| ----------- | ------------------------------------------------------------------------------------------- |
| MAQUINA     | Potência HP, tipo combustível, fabricante, modelo, ano, número série, horímetro atual       |
| VEICULO     | RENAVAM (mono), placa (mono), fabricante, modelo, ano, odômetro atual, tipo combustível     |
| IMPLEMENTO  | Fabricante, modelo, ano, número série, vinculo máquina pai (optional autocomplete)          |
| BENFEITORIA | Material construção, área m², capacidade, coordenadas (lat/lon inputs + mini-mapa opcional) |
| TERRA       | Matrícula (mono), área ha, código CAR, documentação cartorária                              |
| EQUIPAMENTO | Fabricante, modelo, ano, número série                                                       |

### AssetModal — Benfeitoria Map Picker

- Mini-map (240px height) rendered below coordinate inputs using existing Leaflet/FarmMap pattern
- Click-to-place marker sets lat/lon fields
- Manual lat/lon input always visible — map is additive, not required
- Map only renders when BENFEITORIA type is selected

### AssetImportModal — Steps

| Step           | Title                     | Content                                                                                                                                    |
| -------------- | ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| 1 — idle       | "Importar ativos"         | Drag-and-drop zone, CSV/XLSX accepted, template download link                                                                              |
| 2 — uploading  | "Processando arquivo..."  | Skeleton loading, cancel button                                                                                                            |
| 3 — mapping    | "Mapeamento de colunas"   | Column mapping form — required: nome, tipo, classificacao_cpc, fazenda; optional: tag, numero_serie, fabricante, modelo, ano, centro_custo |
| 4 — previewing | "Preview dos ativos"      | Table with valid/invalid rows, row count, validation errors per row                                                                        |
| 5 — confirming | "Importando..."           | Progress indicator                                                                                                                         |
| 6 — done       | "Resultado da importação" | Imported count, skipped count, failed count, error list                                                                                    |

### AssetsPage — Filter Bar

Filters displayed inline in a horizontal bar above the list:

- Fazenda (select)
- Tipo (multi-select pills: MAQUINA, VEICULO, IMPLEMENTO, BENFEITORIA, TERRA, EQUIPAMENTO)
- Status (select: ATIVO, INATIVO, EM_MANUTENCAO, ALIENADO, EM_ANDAMENTO)
- Faixa de valor (min/max inputs with R$ prefix)
- Busca (text input — searches name, tag, serial)
- Export buttons: CSV, Excel, PDF (icon + label, right-aligned)

### AssetsPage — List View

- Default: card list (mobile-first)
- Desktop (≥768px): switch to table view (view toggle button top-right)
- Table columns: Tag | Nome | Tipo | Fazenda | Status | Valor Aquisição | Ações
- Tag column uses JetBrains Mono 14px
- Status column: badge with color (ATIVO=success, INATIVO=info, EM_MANUTENCAO=warning, ALIENADO=neutral, EM_ANDAMENTO=warning) + icon — never color alone
- Row click navigates to `/assets/:id`
- Ações column: edit icon button + delete icon button (both 48x48px touch target)

### AssetDetailPage — Tabs

Tab bar pattern matches AnimalDetailPage:

- Horizontal tab list, `role="tablist"`, each tab `role="tab"`
- Active tab: bottom border 2px solid `--color-primary-600`, font weight 600
- Tab content: `role="tabpanel"`, lazy-loaded per tab

### AssetDetailPage — Header

- Breadcrumb: Início > Ativos > {asset name} (14px Source Sans 3, `--color-neutral-500`)
- Asset name as `<h1>` — DM Sans 700 24px
- Asset tag badge alongside name — JetBrains Mono 14px, neutral-200 background pill
- Status badge — color + icon (same as list view)
- Edit and Delete action buttons in top-right header area

## Accessibility Contract

Per CLAUDE.md WCAG 2.1 AA requirements:

- Semantic HTML: `<main>`, `<nav>`, `<section>`, `<button>`, `<table>` — never `<div onClick>`
- Focus outline: 2px solid `--color-primary-600` on all interactive elements — never removed
- Form fields: `<label>` associated via `for`/`id`, `aria-required="true"` on required fields
- Errors: `role="alert"` on inline validation messages, icon + text (never color alone)
- Icon-only buttons: `aria-label` required — edit="Editar ativo", delete="Excluir ativo", close="Fechar"
- Type selector buttons: `aria-pressed` to reflect selected state
- Tab list: `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected`, `aria-controls`
- Photo upload zone: keyboard accessible, `role="button"`, `aria-label="Adicionar fotos"`
- Status badges: icon + text always (never color alone)
- Asset tag in Mono font: `aria-label="Código do ativo: AT-{YYYY}/{NNNN}"`
- Map picker: coordinate inputs always present as accessible alternative
