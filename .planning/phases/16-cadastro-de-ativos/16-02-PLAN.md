**Fuel Records Module (`apps/backend/src/modules/fuel-records/`):**

1. `fuel-records.types.ts`:
   - `FuelRecordError` class (extends Error, statusCode)
   - `CreateFuelRecordInput`: assetId (required), farmId (required), fuelDate (required), liters (required), pricePerLiter (required), hourmeterAtFuel?, odometerAtFuel?, notes?
   - `ListFuelRecordsQuery`: assetId?, farmId?, page?, limit?, dateFrom?, dateTo?
   - `FuelStatsResult`: assetAvgLitersPerHour, fleetAvgLitersPerHour, assetCostPerHour, fleetCostPerHour, totalLiters, totalCost, recordCount

2. `fuel-records.service.ts`:
   - `createFuelRecord(ctx, input)`: Calculate totalCost = liters * pricePerLiter using Decimal. Create record with createdBy = ctx.userId. Return with asset name included.
   - `listFuelRecords(ctx, query)`: Paginated list with filters, include asset.name. Default sort by fuelDate desc.
   - `getFuelStats(ctx, assetId, periodStart?, periodEnd?)`:
     - Asset-level: aggregate SUM(liters), SUM(totalCost), COUNT for the asset. Calculate avgLitersPerHour = SUM(liters) / (maxHourmeter - minHourmeter) from records with hourmeterAtFuel not null.
     - Fleet-level: same aggregates for ALL assets in the org of same assetType (MAQUINA, VEICULO) in same period.
     - Return both for benchmarking comparison.
   - `deleteFuelRecord(ctx, id)`: Hard delete, return deleted record.

3. `fuel-records.routes.ts`:
   Express router prefix `/org/:orgId/fuel-records`:
   - `POST /` — createFuelRecord (permission: assets:create)
   - `GET /` — listFuelRecords (permission: assets:read)
   - `GET /stats/:assetId` — getFuelStats (permission: assets:read)
   - `DELETE /:id` — deleteFuelRecord (permission: assets:delete)
   All with authenticate + authorize middleware, RLS context, try/catch with FuelRecordError.

**Meter Readings Module (`apps/backend/src/modules/meter-readings/`):**

5. `meter-readings.types.ts`:
   - `MeterReadingError` class
   - `CreateMeterReadingInput`: assetId (required), readingDate (required), readingType (required, "HOURMETER" | "ODOMETER"), value (required)
   - `ListMeterReadingsQuery`: assetId?, readingType?, page?, limit?

6. `meter-readings.service.ts`:
   - `createMeterReading(ctx, input)`:
     - Wrap in `prisma.$transaction`:
       1. Find last reading: `tx.meterReading.findFirst({ where: { assetId, readingType }, orderBy: { readingDate: 'desc' } })`
       2. Validate anti-regression: if lastReading exists and input.value <= lastReading.value, throw MeterReadingError(`Leitura nao pode ser menor ou igual a ultima registrada (${lastReading.value} ${readingType === 'HOURMETER' ? 'h' : 'km'}).`, 400)
       3. Create MeterReading with previousValue = lastReading?.value ?? null
       4. Update Asset snapshot: `tx.asset.update({ where: { id: assetId }, data: { currentHourmeter: readingType === 'HOURMETER' ? value : undefined, currentOdometer: readingType === 'ODOMETER' ? value : undefined } })`
     - Return created reading
   - `listMeterReadings(ctx, query)`: Paginated list, default sort readingDate desc. Include asset.name.
   - `getLatestReadings(ctx, assetId)`: Return latest HOURMETER and latest ODOMETER reading for the asset.

7. `meter-readings.routes.ts`:
   Express router prefix `/org/:orgId/meter-readings`:
   - `POST /` — createMeterReading (permission: assets:update — operators can do this)
   - `GET /` — listMeterReadings (permission: assets:read)
   - `GET /latest/:assetId` — getLatestReadings (permission: assets:read)
   All with authenticate + authorize middleware.

8. Fill in `meter-readings.routes.spec.ts` (replace it.todo stubs with full tests, at least 6):

   - GET / returns paginated list
   - GET /latest/:assetId returns latest readings by type
