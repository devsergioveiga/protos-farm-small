**saving-analysis.service.ts** — All functions are read-only (no writes). Create:

1. `getSavingByQuotation(ctx: RlsContext, params: SavingQueryParams)`:
   - Query quotations with status='FECHADA' in date range (createdAt between startDate/endDate)
   - Include suppliers -> proposal -> items
   - Optional filter by category (via the quotation's RC items) and supplierId
   - For each quotation with >= 2 proposals:
     - For each item (matched by productId across proposals): find max unitPrice and winner unitPrice (winner = the proposal that was selected/approved)
     - saving per item = (maxPrice - winnerPrice) \* quantity
     - saving total = sum of all item savings
   - Filter out quotations with only 1 proposal (no saving calculable)
   - Return SavingSummary with totalSaving, quotationCount, avgSavingPercent (totalSaving / sum of all max totals \* 100)

2. `getPriceHistory(ctx, productId: string, params: { startDate, endDate })`:
   - Query PurchaseOrderItem where productId matches AND PurchaseOrder createdAt in range
   - Include PurchaseOrder for sequentialNumber and supplier name (via PO's supplierId -> Supplier.tradeName)
   - Return PriceHistoryResult with points sorted by date ascending
   - Each point: date = PO.createdAt, price = item.unitPrice, purchaseOrderNumber = PO.sequentialNumber, supplierName

3. `getCycleIndicators(ctx, params: SavingQueryParams)`:
   - Total OCs in period
   - % formal = count(OC where quotationId IS NOT NULL) / total \* 100
   - % emergency = count(OC where isEmergency = true) / total \* 100
   - Avg cycle days: for OCs that have a full chain (RC -> OC -> GR), calculate avg days from RC.createdAt to most recent GoodsReceipt.createdAt. Use the PO's purchaseRequestId to find RC, and GoodsReceipt where purchaseOrderId = PO.id.
   - Return CycleIndicators

4. `getTopProducts(ctx, params: SavingQueryParams, limit = 10)`:
   - Aggregate PurchaseOrderItem.totalPrice grouped by productId/productName where PO.createdAt in range
   - Order by totalSpent desc, take top 10
   - Return TopProductBySpend[]

5. `getTopSuppliers(ctx, params: SavingQueryParams, limit = 5)`:
   - Aggregate PurchaseOrder total (sum of items.totalPrice) grouped by supplierId where PO.createdAt in range
   - Join Supplier for name (tradeName or companyName)
   - Order by totalVolume desc, take top 5
   - Return TopSupplierByVolume[]

6. `getAnalyticsDashboard(ctx, params: SavingQueryParams)`:
   - Call getSavingByQuotation, getCycleIndicators, getTopProducts, getTopSuppliers in parallel (Promise.all)
   - Return AnalyticsDashboard combining all results
