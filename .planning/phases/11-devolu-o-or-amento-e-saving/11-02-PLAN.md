**goods-returns.service.ts** — Create with these exported functions:

1. `getNextDevSequentialNumber(tx, organizationId)` — same pattern as quotations: findFirst where sequentialNumber startsWith `DEV-${year}/`, orderBy desc, parse last number, return `DEV-${year}/${padStart(4,'0')}`.

2. `createGoodsReturn(ctx: RlsContext & { userId: string }, input: CreateGoodsReturnInput)`:
   - Validate goodsReceiptId exists and status is CONFIRMADO — throw 400 "Apenas recebimentos confirmados podem ser devolvidos"
   - Validate items: each item's returnQty must be <= receivedQty on the GoodsReceipt item (check by productId match) — throw 400 "Quantidade de devolucao excede quantidade recebida"
   - Calculate totalPrice for each item = returnQty \* unitPrice
   - Get supplierId from GoodsReceipt (via PurchaseOrder relation or direct field)
   - Create GoodsReturn with items in withRlsContext transaction
   - Assign sequential number via getNextDevSequentialNumber
   - Return GoodsReturnOutput

3. `listGoodsReturns(ctx, query: ListGoodsReturnsQuery)`:
   - Paginated list with filters (status, supplierId, date range, search on sequentialNumber/supplierName)
   - Include computed fields: totalValue = sum(items.totalPrice), itemCount = items.length
   - Return ListGoodsReturnsResult with statusLabel, reasonLabel, actionLabel

4. `getGoodsReturn(ctx, id)`:
   - Fetch with include: items, goodsReceipt (select sequentialNumber, purchaseOrderId)
   - Join supplier for supplierName
   - Return GoodsReturnOutput

5. `transitionGoodsReturn(ctx: RlsContext & { userId: string }, id: string, input: TransitionGrReturnInput)`:
   - Validate transition with canGrReturnTransition — throw 400 "Transicao invalida de {from} para {to}"
   - **On APROVADA**: Execute inside the SAME withRlsContext transaction:
     a. Create StockOutput with type='RETURN', status='CONFIRMED', items mapped from return items (productId, quantity=returnQty, unitPrice). Set goodsReturnId on the StockOutput (note: may need to use a different tracking approach if StockOutput model lacks goodsReturnId — store the stockOutputId on GoodsReturn instead). Deduct from StockBalance atomically.
     b. Execute financial side effect based on expectedAction:
     - **TROCA**: No financial action. Just update status.
     - **CREDITO**: Find original Payable via GoodsReceipt's payable relation (query payable where goodsReceiptId = goodsReturn.goodsReceiptId). Create new Payable with: totalAmount = -(sum of return item totalPrices), isCredit=true, goodsReturnId=returnId, description="Nota de credito — devolucao {sequentialNumber}", same farmId/supplierName/category as original, dueDate=now(). Store creditPayableId on GoodsReturn.
     - **ESTORNO**: Find original Payable. Reduce totalAmount by return value. Recalculate PENDING installments proportionally (divide remaining amount equally among PENDING installments). If full return (return value = original total), cancel all PENDING installments and set Payable status to CANCELADA.
       c. Store stockOutputId on GoodsReturn record.
       d. Create notification to the supplier's contact user (or to MANAGER role) about the return approval.
   - **On CONCLUIDA**: Update resolutionStatus to RESOLVED.
   - Update status and return updated GoodsReturnOutput.

6. `uploadReturnPhoto(ctx, returnId, itemId, file)`:
   - Validate return exists
   - Update GoodsReturnItem with photoUrl = `/uploads/goods-returns/${orgId}/${returnId}/${file.filename}`, photoFileName = file.originalname

7. `deleteGoodsReturn(ctx, id)`:
   - Only allow deletion if status is PENDENTE — throw 400 "Apenas devoluçoes pendentes podem ser excluidas"
   - Soft delete (set deletedAt)
