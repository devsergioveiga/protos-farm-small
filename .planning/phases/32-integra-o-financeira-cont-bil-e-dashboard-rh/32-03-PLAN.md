       ```typescript
       export interface HrDashboardQuery {
         farmId?: string;
         year: number;
         month: number;
       }
       export interface HrDashboardResponse {
         headcount: {
           total: number;
           byStatus: Record<string, number>;      // ATIVO, AFASTADO, FERIAS, DESLIGADO
           byContractType: Record<string, number>; // CLT_INDETERMINATE, SEASONAL, etc.
         };
         currentMonthCost: {
           gross: number;
           net: number;
           charges: number;
           avgPerEmployee: number;
           costPerHectare: number | null;
         };
         trend12Months: Array<{ yearMonth: string; gross: number; net: number; charges: number }>;
         composition: Array<{ label: string; amount: number; percentage: number }>;
         costByActivity: Array<{ activityType: string; totalCost: number }>;
         turnover: { last12MonthsRate: number; terminationsLast12: number; admissionsLast12: number };
         upcomingContractExpirations: Array<{
           days: number;
           count: number;
           employees: Array<{ id: string; name: string; endDate: string; contractType: string }>;
         }>;
         alerts: { overduePayablesPayroll: number; pendingTimesheets: number; expiredContracts: number };
       }
       ```

    2. Create hr-dashboard.service.ts with `getHrDashboard(rls, query)`:
       - All queries scoped by organizationId (from RLS context)
       - Optional farmId filter applied to all queries where applicable

       **headcount:** Query Employee.count grouped by status, then EmployeeContract.count grouped by contractType (where isActive=true). If farmId, filter employees through their active contract's farmId.

       **currentMonthCost:** Find PayrollRun for the given year/month with status COMPLETED. Use run.totalGross, totalNet, totalCharges. avgPerEmployee = totalGross / headcount.total. costPerHectare: if farmId, load Farm.totalAreaHa and divide; if no farmId, sum all Farm.totalAreaHa in org. Return null if no area data. Use Decimal arithmetic throughout.

       **trend12Months:** Find last 12 PayrollRun records (status=COMPLETED), ordered by referenceMonth desc. Use PayrollRun.totalGross/totalNet/totalCharges directly (per RESEARCH.md Pitfall 4 — do NOT drill into items).

       **composition:** For the current month run, break down by rubrica categories from PayrollRunItems:
         - "Salarios" = sum(grossSalary - overtimePay)
         - "Horas Extras" = sum(overtimePay)
         - "Encargos Patronais" = sum(inssPatronal + fgtsAmount)
         - "Beneficios" = sum(vtDeduction + mealAllowance) (approximate)
         - "Outros" = remainder
         Calculate percentage of each vs total. Use Decimal.

       **costByActivity:** Group TimeEntryActivity by operationType for the reference month, join with PayrollRunItem to estimate cost per activity based on minutes proportion.

       **turnover:** Count EmployeeStatusHistory transitions:
         - admissions = count where toStatus='ATIVO' and createdAt >= 12monthsAgo
         - terminations = count where toStatus='DESLIGADO' and createdAt >= 12monthsAgo
         - rate = ((admissions + terminations) / 2) / avgHeadcount * 100
         - avgHeadcount = average of monthly headcount counts (use Employee count)

       **upcomingContractExpirations:** Query EmployeeContract where isActive=true, contractType IN ['SEASONAL', 'CLT_DETERMINATE', 'TRIAL'], endDate between today and today+90. Group into 30/60/90 day buckets. Include employee name and endDate.

       **alerts:** 3 counts:
         - overduePayablesPayroll: Payable where category='PAYROLL', status='OVERDUE'
         - pendingTimesheets: Timesheet where status='PENDING_RH'
         - expiredContracts: EmployeeContract where isActive=true, endDate < today

    ANTI-PATTERNS:
    - Do NOT drill into PayrollRunItem for trend — use pre-computed totals (Pitfall 4)
    - Use Decimal from decimal.js for all monetary math, convert to number only at response boundary
