       ```
       enum AccountingEntryType {
         PAYROLL_SALARY
         PAYROLL_CHARGES
         VACATION_PROVISION
         THIRTEENTH_PROVISION
         TAX_LIABILITY
         SALARY_REVERSAL
       }

       model AccountingEntry {
         id                String               @id @default(uuid())
         organizationId    String
         organization      Organization         @relation(fields: [organizationId], references: [id])
         referenceMonth    DateTime             @db.Date
         entryType         AccountingEntryType
         debitAccount      String
         debitLabel        String
         creditAccount     String
         creditLabel       String
         amount            Decimal              @db.Decimal(14, 2)
         costCenterId      String?
         costCenter        CostCenter?          @relation(fields: [costCenterId], references: [id])
         farmId            String?
         farm              Farm?                @relation(fields: [farmId], references: [id])
         sourceType        String               // "PAYROLL_RUN" | "PAYROLL_PROVISION" | "PAYABLE_SETTLEMENT"
         sourceId          String
         reversedByEntryId String?
         notes             String?
         createdAt         DateTime             @default(now())

         @@index([organizationId, referenceMonth])
         @@index([sourceType, sourceId])
       }
       ```
       Add `accountingEntries AccountingEntry[]` relation on Organization, CostCenter, and Farm models.
       Run `cd apps/backend && npx prisma db push`.

    2. Create accounting-entries.types.ts with:

         export const ACCOUNT_CODES = {
           PAYROLL_SALARY: { debit: '6.1.01', debitLabel: 'Despesa com Salarios', credit: '2.1.01', creditLabel: 'Salarios a Pagar' },
           PAYROLL_CHARGES: { debit: '6.1.02', debitLabel: 'Despesa Encargos Sociais', credit: '2.1.02', creditLabel: 'Encargos a Recolher' },

           THIRTEENTH_PROVISION: { debit: '6.1.04', debitLabel: 'Desp. Prov. 13o', credit: '2.2.02', creditLabel: 'Provisao 13o a Pagar' },
           TAX_LIABILITY: { debit: '6.1.05', debitLabel: 'Despesa INSS/IRRF', credit: '2.1.03', creditLabel: 'INSS/IRRF a Recolher' },
           SALARY_REVERSAL: { debit: '2.1.01', debitLabel: 'Salarios a Pagar', credit: '1.1.01', creditLabel: 'Caixa/Bancos' },

       - AccountingEntryOutput interface
       - AccountingEntryListInput interface with optional filters (referenceMonth, farmId, entryType, page, limit)

    3. Create accounting-entries.service.ts with functions:

       **createPayrollEntries(organizationId, runId):**
       - Load PayrollRun with items (sum grossSalary, inssPatronal, fgtsAmount, inssAmount, irrfAmount)
       - Load PayrollProvisions for same referenceMonth (VACATION, THIRTEENTH types with accountingEntryJson)
       - Create 6 AccountingEntry records using ACCOUNT_CODES:
         - PAYROLL_SALARY: amount = sum(grossSalary), sourceType='PAYROLL_RUN', sourceId=runId
         - PAYROLL_CHARGES: amount = sum(inssPatronal + fgtsAmount), sourceType='PAYROLL_RUN', sourceId=runId
         - VACATION_PROVISION: from PayrollProvision accountingEntryJson where type=VACATION
         - THIRTEENTH_PROVISION: from PayrollProvision accountingEntryJson where type=THIRTEENTH
         - TAX_LIABILITY: amount = sum(inssAmount + irrfAmount), sourceType='PAYROLL_RUN', sourceId=runId
         - SALARY_REVERSAL: NOT created here — only on payment
       - Each entry gets costCenterId from run's farm default (or null), farmId from run
       - Use prisma.accountingEntry.createMany for batch insert

       **createReversalEntry(organizationId, payableId, amount, farmId):**
       - Creates SALARY_REVERSAL entry: debit 2.1.01, credit 1.1.01
       - sourceType='PAYABLE_SETTLEMENT', sourceId=payableId

       **revertPayrollEntries(organizationId, runId):**
       - Delete all AccountingEntry where sourceType='PAYROLL_RUN' and sourceId=runId

       **list(rls, filters):** paginated list with optional filters
       **exportCsv(rls, filters):** returns CSV string with all entries matching filters

    - Do NOT call createPayrollEntries inside the closeRun transaction — call AFTER (non-blocking try/catch)
    - Use Decimal from decimal.js for all amount arithmetic
