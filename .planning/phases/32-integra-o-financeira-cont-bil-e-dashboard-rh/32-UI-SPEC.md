## Screen-by-Screen Interaction Contract

### Screen 1: PayrollCpReviewModal (INTEGR-01)

**Trigger:** User clicks "Fechar Folha" on PayrollRunsPage when run is in CALCULATED state.

**Structure:**
- Modal with header / scrollable body / sticky footer
- Header: title "Revisão de Contas a Pagar — [Mês/Ano]" + PayrollRunStatusBadge
- Body: grouped table — sections: "Salários Líquidos", "Encargos Patronais", "Impostos a Recolher", "Outros"
  - Each section: collapsible `<details>` with `<summary>` showing section total
  - Table columns: Tipo | Colaborador (if per-employee) | Valor | Vencimento | Centro de Custo
  - Monetary values: JetBrains Mono, right-aligned
  - Column headers: ALL CAPS, 14px, weight 600
- Reconciliation row at body bottom: "Total CPs gerados: R$ X.XXX,XX — Total folha líquida: R$ X.XXX,XX" with green checkmark if difference < R$ 0,01, orange warning triangle if divergent
- Footer: two buttons — "Cancelar" (secondary, left) and "Confirmar Fechamento" (primary green, right, 1 per screen)
- Loading state: skeleton pulses in table cells while GET cp-preview fetches
- Error state: inline alert above table — red border-left card with AlertCircle icon + message

**Estorno:** Separate "Estornar Folha" button on PayrollRunDetailModal triggers `ConfirmModal` with `variant="danger"`, label "Estornar Fechamento", body copy as specified in Copywriting section.

### Screen 2: AccountingEntriesPage (INTEGR-02)

**Route:** `/accounting-entries`

**Structure:**
- Full page with breadcrumb: Dashboard > Contabilidade > Lançamentos
- Filter bar: month/year picker + fazenda select + entry type multi-select — all in a single row below breadcrumb
- Table: Competência | Tipo | Débito | Crédito | Valor | Centro de Custo | Origem
  - "Origem" is a hyperlink pill: "[FOLHA-2026-03]" linking to PayrollRunDetailModal
  - Tipo column: colored badge (6 colors — one per AccountingEntryType)
  - Table transforms to stacked cards below 768px (per CLAUDE.md mobile rule)
- Drill-down: clicking a row opens inline accordion expansion showing debitLabel + creditLabel + notes + reversedBy link
- Export: secondary "Exportar CSV" button in page header (right side)
- Empty state: as specified in Copywriting section
- No modal for creation — accounting entries are system-generated only, no manual creation form in v1.3

AccountingEntryType badge colors:

| Type | Badge Color |
|------|-------------|
| PAYROLL_SALARY | `--color-primary-100` bg / `--color-primary-700` text |
| PAYROLL_CHARGES | `--color-sky-100` bg / `--color-sky-500` text |
| VACATION_PROVISION | `--color-sun-100` bg / `--color-sun-500` text |
| THIRTEENTH_PROVISION | `--color-earth-100` bg / `--color-earth-500` text |
| TAX_LIABILITY | `--color-error-100` bg / `--color-error-500` text |
| SALARY_REVERSAL | `--color-neutral-100` bg / `--color-neutral-600` text |

### Screen 3: HrDashboardPage (INTEGR-03)

**Route:** `/hr-dashboard`

**Structure:**
- Page layout mirrors FinancialDashboardPage exactly: breadcrumb + filter bar + KPI card row + charts section + tables section
- Filter bar: fazenda select + year/month — auto-applies on change (no submit button needed)
- KPI card row (4 cards, responsive 2x2 on mobile → 4x1 on desktop): Total Colaboradores | Custo Bruto Folha | Custo por Hectare | Turnover 12m
- Each KPI card: big number in JetBrains Mono + label in Source Sans 3 + trend arrow vs previous month (green up/red down for cost, green down for turnover)
- Charts section (2-column grid, stacks on mobile):
  - Left: PayrollCostTrendChart — 12-month BarChart, 3 stacked series (Bruto/Líquido/Encargos), X-axis month labels abbreviated ("Jan", "Fev", etc.)
  - Right: PayrollCompositionChart — PieChart with legend list below, labels showing percentage + "R$ XXX,XX"
- Cost by activity table: Atividade | Custo Total | % do Total — sorted by cost descending, max 8 rows + "Ver mais" expand
- Upcoming expirations: 3-column card group (30 dias / 60 dias / 90 dias), each card shows count badge + employee list (name + endDate), styled with warning-500 for 30d, sun-500 for 60d, neutral-400 for 90d
- Alerts section: 3 compact alert rows — overdue payables (error), pending timesheets (warning), expired contracts (warning) — each with count badge, description, and "Ver" link
- Loading state: skeleton KPI cards (4 rectangles with pulse) + skeleton chart placeholders (grey rectangles) — never spinner
- Empty state (no payroll runs found for filter): as specified in Copywriting section

Charts are `React.lazy` + `Suspense` wrapped with skeleton fallback.

## Accessibility Contract

Source: CLAUDE.md accessibility rules (WCAG 2.1 AA).

- `<main id="main-content">` landmark on all three pages
- Breadcrumb nav: `<nav aria-label="Navegação estrutural">` with `aria-current="page"` on last item
- PayrollCpReviewModal: focus trapped inside while open; Escape key closes; `aria-modal="true"` on dialog element; `role="dialog"` with `aria-labelledby` pointing to modal title
- Collapsible table sections: use `<details>`/`<summary>` — keyboard-native, no custom ARIA needed
- Reconciliation status: `role="status"` on the reconciliation row (live region, non-intrusive)
- Error states in modal: `role="alert"` on error message — announced immediately to screen readers
- ConfirmModal for estorno: focus moves to "Manter Fechamento" (safe action) by default, not destructive button
- All chart containers: `aria-label` on the wrapping div describing the chart purpose; `role="img"` on SVG root; data table as `<details>` summary hidden visually for screen readers
- Empty state CTAs: `<button>` or `<a>` with descriptive accessible name (not just "Clique aqui")
- Focus visible: `outline: 2px solid var(--color-primary-500); outline-offset: 2px` — never removed
- Icon-only buttons: `aria-label` required on every instance
- Status badges: always include text label, never color only
