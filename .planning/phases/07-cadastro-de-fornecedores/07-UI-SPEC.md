## Spacing Scale

Declared values (multiples of 4):

| Token | Value               | Usage                                                         |
| ----- | ------------------- | ------------------------------------------------------------- |
| xs    | 4px (`--space-1`)   | Icon gaps, inline padding between badge items                 |
| sm    | 8px (`--space-2`)   | Compact element spacing, gap between form field icon and text |
| md    | 16px (`--space-4`)  | Default form field padding horizontal, gap between columns    |
| lg    | 24px (`--space-6`)  | Section padding inside modal, card internal padding           |
| xl    | 32px (`--space-8`)  | Gap between major sections (Top Fornecedores / table)         |
| 2xl   | 48px (`--space-12`) | Page-level top padding, empty state vertical spacing          |
| 3xl   | 64px (`--space-16`) | Empty state illustration margin                               |

Exceptions:

- Touch targets: minimum 48x48px for all interactive controls (design system rule)
- Input padding: 12px vertical × 16px horizontal (`--space-3` × `--space-4`)
- Modal header/footer padding: 24px (`--space-6`)
- Modal body padding: 24px (`--space-6`) with 16px between field rows

## Typography

| ----------------- | ------------- | -------------------- | ---------------------- | ------------------------ | ------------------------------------------------------------------------------------------------------ |
| Body              | Source Sans 3 | 16px (`--text-base`) | 400 (`--font-regular`) | 1.5 (`--leading-normal`) | Table cells, form field values, descriptions                                                           |

Rules:

- Maximum 2 font weights per font family on any single screen (DM Sans: 500+700; Source Sans 3: 400+600)
- CNPJ/CPF displayed with formatting mask (XX.XXX.XXX/XXXX-XX or XXX.XXX.XXX-XX) using JetBrains Mono at 14px (`--text-sm`)
- Status and category badge labels: ALL CAPS, `--text-sm` (14px), Source Sans 3 weight 600
- Star rating count: `--text-sm`, Source Sans 3, neutral-500, e.g., "(12 avaliações)"
- Average rating numeric: `--text-base`, DM Sans, weight 700
- Caption-style text (upload file name, timestamps, rating counts): use Label size (14px), not a separate caption size

## Color

| Role            | Token                 | Hex       | Usage                                                                               |
| --------------- | --------------------- | --------- | ----------------------------------------------------------------------------------- |
| Dominant (60%)  | `--color-neutral-0`   | `#FFFFFF` | Page background, modal background, form field background                            |
| Secondary (30%) | `--color-neutral-50`  | `#FAFAF8` | Page body background, Top Fornecedores section background, table row hover, sidebar |
| Accent (10%)    | `--color-primary-600` | `#2E7D32` | Primary CTA button only ("Novo Fornecedor", "Importar", "Avaliar", "Salvar")        |
| Destructive     | `--color-error-500`   | `#C62828` | Supplier delete action button, ConfirmModal danger variant, form validation errors  |

Accent reserved for:

1. "Novo Fornecedor" button (single primary CTA per page)
2. "Salvar" button inside SupplierModal footer
3. "Avaliar" primary button inside SupplierRatingModal
4. "Importar X fornecedores" confirm button inside SupplierImportModal

Secondary semantic colors (not accent, not destructive):

- Status badge ATIVO: `--color-success-100` background + `--color-success-500` text + icon
- Status badge INATIVO: `--color-neutral-100` background + `--color-neutral-600` text + icon
- Status badge BLOQUEADO: `--color-error-100` background + `--color-error-500` text + icon
- Rating badge low (<3): `--color-warning-100` background + `--color-warning-500` text + AlertTriangle icon
- Import preview row valid: `--color-success-100` background + CheckCircle2 icon
- Import preview row invalid: `--color-error-100` background + XCircle icon + error reason text
- Category badges: `--color-earth-100` background + `--color-earth-500` text (consistent across all categories)
- Top Fornecedores rank #1: `--color-sun-100` background border + `--color-sun-500` crown icon

Color rules enforced:

- Never use color as the sole status indicator — always pair with icon or text label (WCAG)
- Contrast ratio: 4.5:1 minimum for all text, 3:1 for large text (>18px bold)
- Green (`primary-600`) only on CTA buttons — never on text, icons, or decorative elements

## Interaction Contracts

### Page Load

1. Page renders with skeleton immediately (0ms)
2. Suppliers data fetches (`GET /api/org/suppliers`)
3. Top Fornecedores data fetches (`GET /api/org/suppliers/top3?category={default}`)
4. Skeleton replaced with real content or empty state
5. No spinner overlay — skeleton only

### CNPJ/CPF Validation Timing

- onBlur: validate format and check digits using document-validator.ts
- onBlur: if valid and field is not empty, check for duplicates (debounced, 400ms)
- Real-time formatting: apply mask as user types (auto-insert dots/slashes/dashes)

### Import Flow

1. User clicks "Importar" → SupplierImportModal opens
2. User drops/selects file → loading state (spinner inside modal, 200ms)
3. Preview returned from `POST /api/org/suppliers/import/preview` → Step 2 shown
4. User reviews rows → clicks "Importar X fornecedores"
5. Execute call (`POST /api/org/suppliers/import/execute`) → loading state on button
6. Success: modal closes + toast "X fornecedores importados" + list refetch
7. Error: error message shown in modal body, modal stays open

### Export Flow

- "Exportar CSV": immediate download, `Content-Disposition: attachment; filename="fornecedores.csv"`, no modal
- "Exportar PDF": immediate download, `Content-Disposition: attachment; filename="fornecedores.pdf"`, loading spinner on button during generation (300–1000ms)

### Rating Flow

1. User clicks Star icon in table row → SupplierRatingModal opens
2. User selects stars for each criterion, optionally adds comment
3. "Registrar Avaliação" → POST call, loading state on button
4. Success: modal closes + toast "Avaliação registrada com sucesso" + supplier row rating updates inline

### Destructive Delete Flow

1. User clicks Trash2 icon → ConfirmModal opens (variant="danger")
2. User clicks "Excluir fornecedor" → DELETE call, loading state on button
3. Success: modal closes + toast "Fornecedor excluído" + row removed from table
4. Error: toast "Não foi possível excluir. Tente novamente."

### Status Change to BLOQUEADO

1. Inline status selector in edit modal → user selects "BLOQUEADO"
2. ConfirmModal (variant="warning") appears before saving
3. User confirms "Bloquear fornecedor" → status saved

### Keyboard Navigation

- Tab order in SupplierModal: follows DOM order (section by section, top to bottom)
- Escape: closes any open modal or dropdown
- Enter on table row: opens edit modal for that supplier
- Focus trap: all modals trap focus within their container
- Focus visible: `outline: 2px solid var(--color-primary-500); outline-offset: 2px;`

## Accessibility Contracts

- All form inputs: `<label htmlFor>` associated, `aria-required="true"` on required fields
- Error messages: `role="alert"` + `aria-live="polite"`
- Icon-only buttons (table actions): `aria-label` required (e.g., `aria-label="Editar fornecedor {name}"`)
- Decorative icons: `aria-hidden="true"`
- Modal: `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing to modal H2
- Star rating inputs: `role="radiogroup"` with `aria-label="Prazo de Entrega"`, each star `role="radio"` + `aria-label="{N} estrelas"`
- Status badge: text label always present alongside icon (never icon-only badge)
- Import preview table: `<table>` with `<th scope="col">` headers, `<caption>` describing content
- Supplier table: `<table>` with `<th scope="col">` headers, `<caption>` "Lista de fornecedores"
- Focus visible: never suppressed
