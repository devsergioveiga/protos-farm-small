**Hook (`useEsocialEvents.ts`):**
Following useState+useCallback pattern:

```typescript
export function useEsocialEvents() {
  // State: events[], dashboard, loading, error, successMessage
  const fetchEvents = useCallback(async (params?: { referenceMonth?, eventGroup?, eventType?, status? }) => { ... }, []);
  const fetchDashboard = useCallback(async (referenceMonth: string) => {
    const data = await api.get<EsocialDashboard>(`/org/esocial-events/dashboard?referenceMonth=${referenceMonth}`);
    setDashboard(data);
  }, []);
  const generateBatch = useCallback(async (eventType: string, referenceMonth?: string) => {
    await api.post('/org/esocial-events/generate-batch', { eventType, referenceMonth });
    // ... refetch
  }, []);
  const downloadEvent = useCallback(async (id: string, eventType: string, version: number) => {
    const blob = await api.getBlob(`/org/esocial-events/${id}/download`);
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = `${eventType}_v${version}.xml`;
    a.click();
    setTimeout(() => URL.revokeObjectURL(url), 10000);
  }, []);
  const updateStatus = useCallback(async (id: string, input: UpdateEsocialStatusInput) => {
    await api.patch(`/org/esocial-events/${id}/status`, input);
    // ... refetch
  }, []);
  const reprocessEvent = useCallback(async (id: string) => {
    await api.post(`/org/esocial-events/${id}/reprocess`);
    // ... refetch
  }, []);
  return { events, dashboard, loading, error, successMessage, fetchEvents, fetchDashboard, generateBatch, downloadEvent, updateStatus, reprocessEvent };
}
```

**Backend Service (`income-statements.service.ts`):**

1. `generateStatements(orgId, input: GenerateIncomeStatementsInput, userId)`:
   - Query all COMPLETED PayrollRuns for the yearBase (Jan-Dec)
   - For each employee (or filtered employeeIds):
     a. Aggregate PayrollRunItems: sum grossSalary -> totalTaxable, sum inssAmount -> totalInss, sum irrfAmount -> totalIrrf
     b. totalExempt: sum items from lineItemsJson where rubrica type is 'EXEMPT' or 'FAMILY_SALARY' (salary-family, 13th first parcel)
     c. dependentDeduction: count employee dependents * 2275.08 (annual IRRF deduction per dependent for 2025/2026)
     d. Upsert IncomeStatement (unique on orgId+employeeId+yearBase)
   - Generate PDF for each using pdfkit (see PDF layout below)
   - Store PDF path in pdfKey field
   - Return array of IncomeStatementOutput

2. `listStatements(orgId, query: ListIncomeStatementsQuery)`:
   - Filter by yearBase, employeeId. Include employee name and CPF.
   - Paginated. Per D-16, no limit on history years.

3. `downloadStatement(orgId, statementId)`:
   - Fetch statement, generate or retrieve PDF
   - Return { buffer, filename: `informe-rendimentos-${employeeName}-${yearBase}.pdf` }

4. `sendStatements(orgId, input: SendIncomeStatementsInput, userId)`:
   - Fetch statements for yearBase (optionally filtered by employeeIds)
   - For each with employee.email: send PDF attachment via mail.service (same pattern as payslip send in Phase 28)
   - Update sentAt and sentTo on each sent record
   - Return { sent: number, skipped: number, errors: string[] }

5. `getRaisConsistency(orgId, yearBase)`:
   - Count distinct employees active during the year
   - Check EsocialEvent table: count employees with S-2200 events, S-1200 events for at least one month, S-2299 for terminated employees
   - Return RaisConsistencyOutput with isConsistent = (missingAdmissionEvents.length === 0 && missingRemunerationEvents.length === 0)
