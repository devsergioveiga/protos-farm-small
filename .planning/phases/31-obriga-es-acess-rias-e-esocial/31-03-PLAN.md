**esocial-validators.ts:**
Create one validate function per event type. Each returns `EsocialValidationError[]`:

```typescript
export function validateS2200Input(employee, contract, position): EsocialValidationError[] {
  const errors: EsocialValidationError[] = [];
  if (!employee.pisPassep) errors.push({ field: 'nisTrab', employeeName: employee.name, message: 'NIS/PIS nao informado' });
  if (!position.cbo) errors.push({ field: 'codCBO', employeeName: employee.name, message: 'CBO do cargo nao informado' });
  if (!contract.salary || Number(contract.salary) <= 0) errors.push({ field: 'vrSalFx', employeeName: employee.name, message: 'Salario invalido ou zero' });
  if (!employee.cpf) errors.push({ field: 'cpfTrab', employeeName: employee.name, message: 'CPF nao informado' });
  if (!employee.birthDate) errors.push({ field: 'dtNascimento', employeeName: employee.name, message: 'Data de nascimento nao informada' });
  return errors;
}

export function validateS1200Input(item, employee): EsocialValidationError[] {
  const errors: EsocialValidationError[] = [];
  if (!item.lineItemsJson) errors.push({ field: 'lineItemsJson', employeeName: employee.name, message: 'Itens de remuneracao ausentes' });
  // Check at least one rubrica has eSocialCode
  return errors;
}
```

**Service (`esocial-events.service.ts`):**

Create class `EsocialEventsService` with methods:

1. `generateEvent(orgId, input: GenerateEsocialEventInput, userId)`:
   - Determine event group from `EVENT_GROUP_MAP[input.eventType]`
   - Load source data based on `SOURCE_TYPE_MAP[input.eventType]`:
     - EMPLOYEE: load Employee with contracts, position
     - PAYROLL_RUN_ITEM: load PayrollRunItem with PayrollRun, Employee
     - PAYROLL_RUN: load PayrollRun
     - MEDICAL_EXAM: load MedicalExam with Employee
     - EMPLOYEE_ABSENCE: load EmployeeAbsence with Employee
     - EMPLOYEE_TERMINATION: load EmployeeTermination with Employee
     - ORGANIZATION: load Organization
     - FARM: load Farm
     - PAYROLL_RUBRICA: load PayrollRubrica
     - POSITION: load Position
     - CONTRACT_AMENDMENT: load ContractAmendment with Employee
     - EPI_DELIVERY: load EpiDelivery with Employee
   - Run validator: `getValidator(input.eventType)(data)` — if errors, return { validationErrors } without creating record
   - **S-1299 guard (per Pitfall 5):** If eventType is S-1299, check if ALL S-1200 and S-1210 events for the same referenceMonth + org are in EXPORTADO or ACEITO status. If any are PENDENTE, return error: "Exporte todos os eventos S-1200/S-1210 da competencia antes de gerar o S-1299."
   - Build XML: `getBuilder(input.eventType)(data)` — returns XML string
   - Create `EsocialEvent` record with status PENDENTE, xmlContent = generated XML, version = 1
   - Return EsocialEventOutput

2. `generateBatch(orgId, eventType, referenceMonth?, userId)`:
   - For periodic events (S-1200/S-1210): generate one event per PayrollRunItem in the run for that month
   - For S-1299: generate one closing event per run
   - For table events: generate based on current org/farm/rubrica/position state
   - Return array of results (including validation errors per employee)

3. `listEvents(orgId, query: ListEsocialEventsQuery)`:
   - Filter by eventGroup, eventType, status, referenceMonth
   - Paginated, ordered by createdAt DESC

4. `downloadEvent(orgId, eventId)`:
   - Fetch event, verify status is PENDENTE
   - Transition to EXPORTADO, set exportedAt = now
   - Return { xmlContent, filename: `${eventType}_v${version}_${sourceId}.xml` }

5. `downloadBatch(orgId, query: { referenceMonth, eventGroup?, eventType? })`:
   - Fetch all matching PENDENTE events, transition each to EXPORTADO
   - Return events as ZIP or concatenated (individual XMLs)

6. `updateStatus(orgId, eventId, input: UpdateEsocialStatusInput)`:
   - Validate transition using VALID_ESOCIAL_TRANSITIONS
   - If ACEITO: set acceptedAt = now
   - If REJEITADO: set rejectedAt = now, rejectionReason = input.rejectionReason (required)
   - Return updated event

7. `reprocessEvent(orgId, eventId, userId)`:
   - Verify current status is REJEITADO
   - Load source data again (may have been corrected)
   - Run validator again — if still errors, return errors
   - Build new XML with corrected data
   - Create NEW EsocialEvent record with version = old.version + 1, status PENDENTE, new xmlContent
   - Old record preserved (history per D-11)
   - Return new event

8. `getDashboard(orgId, referenceMonth)`:
   - Aggregate EsocialEvent counts by status and group for the given referenceMonth (and events with null referenceMonth for non-periodic/table events)
   - Return EsocialDashboardOutput
