<!-- Backend API endpoints from Plan 02 -->
Backend API (base: /api/org/:orgId/depreciation):
```
POST   /config                   → { id, assetId, method, fiscalAnnualRate, ... }  (201)
GET    /config/:assetId          → { id, assetId, method, ... } | null             (200)
PATCH  /config/:assetId          → { id, assetId, method, ... }                     (200)
DELETE /config/:assetId          →                                                   (204)
POST   /run                      → { id, periodYear, periodMonth, track, status, totalAssets, processedCount, skippedCount, totalAmount, ... } (202)
POST   /entries/:entryId/reverse → { id, depreciationAmount (negative), ... }       (200)
GET    /report?periodYear=&periodMonth=&track=&assetId=&page=&limit= → { entries: [...], total, page, limit } (200)
GET    /report/export?periodYear=&periodMonth=&track=&format=csv|xlsx → Binary file (200)
GET    /last-run?periodYear=&periodMonth=&track= → { id, status, ... } | null      (200)
```

**2. Create hooks:**

`useDepreciationConfig.ts`: Functions `getConfig(assetId)`, `createConfig(data)`, `updateConfig(assetId, data)`, `deleteConfig(assetId)`. Uses `api.get/post/patch/delete` following existing hook patterns. Returns `{ config, loading, error, create, update, remove, refetch }`.

`useDepreciationRun.ts`: Functions `triggerRun({ periodYear, periodMonth, track, force })`, `getLastRun(periodYear, periodMonth, track?)`. Returns `{ lastRun, loading, triggerRun, refetch }`.

`useDepreciationReport.ts`: Function `fetchReport(periodYear, periodMonth, track?, assetId?, page?, limit?)`. **Accepts optional `assetId` parameter** to filter entries for a specific asset (used by AssetDepreciationTab). Also includes `exportReport(periodYear, periodMonth, track?, format: 'csv' | 'xlsx')` which calls `GET /report/export` and triggers a browser file download using `URL.createObjectURL(blob)` + hidden `<a>` click pattern. Returns `{ data: DepreciationReportResponse, loading, error, refetch, exportReport }`.

All hooks use the same API base path pattern: `/api/org/${orgId}/depreciation/...` extracting orgId from auth context (same pattern as `useAssetDetail`).
