**1. Create `depreciation.service.ts`** — Config CRUD + report query.

Functions:

- `createConfig(rls: RlsContext, input: CreateDepreciationConfigInput)`: Validates asset exists and belongs to org. Validates asset.classification is DEPRECIABLE_CPC27 or BEARER_PLANT_CPC27 (reject NON_DEPRECIABLE_CPC27 and FAIR_VALUE_CPC29 with DepreciationError 400 "Ativo não depreciável"). Creates DepreciationConfig via `prisma.depreciationConfig.create()`. Returns config with asset included.

- `getConfig(rls: RlsContext, assetId: string)`: Returns DepreciationConfig or null. Include asset for type info.

- `updateConfig(rls: RlsContext, assetId: string, input: UpdateDepreciationConfigInput)`: Updates via `prisma.depreciationConfig.update({ where: { assetId } })`. Returns updated config.

- `deleteConfig(rls: RlsContext, assetId: string)`: Deletes via `prisma.depreciationConfig.delete({ where: { assetId } })`.

- `getReport(query: DepreciationReportQuery)`: Queries `depreciation_entries` for the period+track with joined asset (name, assetType, assetTag), farm (name), and ccItems (with costCenter name). Paginated (default 20/page). Returns `{ entries, total, page, limit }`.

- `getLastRun(organizationId: string, periodYear: number, periodMonth: number, track?: string)`: Returns latest DepreciationRun for the period.

**2. Create `depreciation-batch.service.ts`** — Batch processing + reversal.

Functions:

- `runDepreciationBatch(input: RunDepreciationInput)`:
  1. Check for existing completed run for this period+track. If exists and `force !== true`, throw DepreciationError 409 "Depreciação já executada para este período".
  2. Create DepreciationRun with status PENDING.
  3. Fetch eligible assets: `prisma.asset.findMany({ where: { organizationId, deletedAt: null, status: { not: 'EM_ANDAMENTO' }, classification: { in: ['DEPRECIABLE_CPC27', 'BEARER_PLANT_CPC27'] } }, include: { depreciationConfig: true, farm: { select: { id: true } } } })`.
  4. Update run with totalAssets count.
  5. For each asset, process in **individual** `prisma.$transaction()` (NOT one big transaction — avoids timeout per Research pitfall 2):
     a. Query latest DepreciationEntry for asset+track (`findFirst orderBy periodYear desc, periodMonth desc, where reversedAt is null`). If found, `openingBookValue = entry.closingBookValue`. If not found, `openingBookValue = asset.acquisitionValue`.
     b. If no depreciationConfig: mark as skipped with reason "Sem configuração de depreciação cadastrada", increment skippedCount.
     c. Call `computeDepreciation()` from engine service.
     d. If result.skipped: increment skippedCount, record reason.
     e. Try `prisma.depreciationEntry.create()`. Catch Prisma P2002 → mark as skipped (idempotent re-run).
     f. If asset.costCenterId: create `DepreciationEntryCCItem` with amount = full depreciationAmount, percentage = 100. Assert reconciliation: `sum(ccItems.amount) === depreciationAmount` before commit.
     g. Increment processedCount, add to totalAmount.
  6. Update DepreciationRun: status = COMPLETED (or PARTIAL if skippedCount > 0), completedAt, processedCount, skippedCount, totalAmount.
  7. Return the DepreciationRun with counts.

- `reverseEntry(rls: RlsContext, entryId: string)`:
  1. Find entry by id, verify org match.
  2. If already reversed (reversedAt not null): throw DepreciationError 400 "Lançamento já estornado".
  3. In a transaction:
     a. Create reversal entry with negative amounts: `depreciationAmount = -original.depreciationAmount`, `closingBookValue = original.openingBookValue`, `notes = "Estorno do lançamento {entryId}"`.
     b. Update original: `reversedAt = now()`, `reversalEntryId = reversal.id`.
     c. Delete associated ccItems of original entry.
  4. Return reversal entry.

All Decimal values from Prisma are wrapped with `new Decimal(value)` before arithmetic. Final storage values use `.toDecimalPlaces(2)`.

**2. Create `depreciation.cron.ts`** — Monthly cron with Redis distributed lock.

Copy the exact pattern from `digest.cron.ts`:

```typescript
import cron from 'node-cron';
import { redis } from '../../database/redis';
import { runDepreciationBatch } from '../../modules/depreciation/depreciation-batch.service';
import { logger } from '../utils/logger';

export function startDepreciationCron(): void {
  cron.schedule(
    '0 2 1 * *',
    async () => {
      const now = new Date();
      const prevMonth = now.getMonth() === 0 ? 12 : now.getMonth();
      const prevYear = now.getMonth() === 0 ? now.getFullYear() - 1 : now.getFullYear();
      const lockKey = `cron:depreciation:${prevYear}-${prevMonth}`;
      const locked = await redis.set(lockKey, '1', 'EX', 600, 'NX');
      if (!locked) {
        logger.info('Depreciation cron: another instance is running, skipping');
        return;
      }
      try {
        logger.info('Depreciation cron: starting batch');
        // Process FISCAL track for previous month
        await runDepreciationBatch({
          organizationId: '', // Cron processes ALL orgs — batch service must handle empty orgId by querying all orgs
          periodYear: prevYear,
          periodMonth: prevMonth,
          track: 'FISCAL',
          triggeredBy: 'cron',
        });
        // Process MANAGERIAL track separately
        await runDepreciationBatch({
          organizationId: '',
          periodYear: prevYear,
          periodMonth: prevMonth,
          track: 'MANAGERIAL',
          triggeredBy: 'cron',
        });
        logger.info('Depreciation cron: completed');
      } catch (err) {
        logger.error({ err }, 'Depreciation cron: failed');
      } finally {
        await redis.del(lockKey);
      }
    },
    { timezone: 'America/Sao_Paulo' },
  );
}
```
