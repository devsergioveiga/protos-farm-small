**2. Create `depreciation-batch.service.ts`** — Batch processing + reversal.

Functions:

- `runDepreciationBatch(input: RunDepreciationInput)`:
  1. **Handle empty organizationId (cron mode):** When `organizationId` is empty string or not provided, query `prisma.organization.findMany({ select: { id: true } })` to get all organization IDs. Loop over each org and call an internal `_processOrganization(orgId, input)` function. Return an aggregate DepreciationRun summary (sum of all org runs).
  2. **Single org processing (`_processOrganization`):** Check for existing completed run for this org+period+track. If exists and `force !== true`, throw DepreciationError 409 "Depreciacao ja executada para este periodo".
  3. Create DepreciationRun with status PENDING.
  4. Fetch eligible assets: `prisma.asset.findMany({ where: { organizationId, deletedAt: null, status: { not: 'EM_ANDAMENTO' }, classification: { in: ['DEPRECIABLE_CPC27', 'BEARER_PLANT_CPC27'] } }, include: { depreciationConfig: true, farm: { select: { id: true } } } })`.
  5. Update run with totalAssets count.
  6. For each asset, process in **individual** `prisma.$transaction()` (NOT one big transaction — avoids timeout per Research pitfall 2):
     a. Query latest DepreciationEntry for asset+track (`findFirst orderBy periodYear desc, periodMonth desc, where reversedAt is null`). If found, `openingBookValue = entry.closingBookValue`. If not found, `openingBookValue = asset.acquisitionValue`.
     b. If no depreciationConfig: mark as skipped with reason "Sem configuracao de depreciacao cadastrada", increment skippedCount.
     c. Call `computeDepreciation()` from engine service.
     d. If result.skipped: increment skippedCount, record reason.
     e. Try `prisma.depreciationEntry.create()`. Catch Prisma P2002 → mark as skipped (idempotent re-run).
     f. If asset.costCenterId: create `DepreciationEntryCCItem` with amount = full depreciationAmount, percentage = 100. Assert reconciliation: `sum(ccItems.amount) === depreciationAmount` before commit.
     g. Increment processedCount, add to totalAmount.
  7. Update DepreciationRun: status = COMPLETED (or PARTIAL if skippedCount > 0), completedAt, processedCount, skippedCount, totalAmount.
  8. Return the DepreciationRun with counts.

- `reverseEntry(rls: RlsContext, entryId: string)`:
  1. Find entry by id, verify org match.
  2. If already reversed (reversedAt not null): throw DepreciationError 400 "Lancamento ja estornado".
  3. In a transaction:
     a. Create reversal entry with negative amounts: `depreciationAmount = -original.depreciationAmount`, `closingBookValue = original.openingBookValue`, `notes = "Estorno do lancamento {entryId}"`.
     b. Update original: `reversedAt = now()`, `reversalEntryId = reversal.id`.
     c. Delete associated ccItems of original entry.
  4. Return reversal entry.

All Decimal values from Prisma are wrapped with `new Decimal(value)` before arithmetic. Final storage values use `.toDecimalPlaces(2)`.

**1. Create `depreciation.cron.ts`** — Monthly cron with Redis distributed lock.

Copy the exact pattern from `digest.cron.ts`. **CRITICAL FIX:** Do NOT pass empty `organizationId`. Instead, enumerate all organizations and call `runDepreciationBatch` once per org per track.

```typescript
import cron from 'node-cron';
import { redis } from '../../database/redis';
import { prisma } from '../../database/prisma';
import { runDepreciationBatch } from '../../modules/depreciation/depreciation-batch.service';
import { logger } from '../utils/logger';

export function startDepreciationCron(): void {
  cron.schedule(
    '0 2 1 * *',
    async () => {
      const now = new Date();
      const prevMonth = now.getMonth() === 0 ? 12 : now.getMonth();
      const prevYear = now.getMonth() === 0 ? now.getFullYear() - 1 : now.getFullYear();
      const lockKey = `cron:depreciation:${prevYear}-${prevMonth}`;
      const locked = await redis.set(lockKey, '1', 'EX', 600, 'NX');
      if (!locked) {
        logger.info('Depreciation cron: another instance is running, skipping');
        return;
      }
      try {
        logger.info('Depreciation cron: starting batch');

        // Enumerate all organizations
        const orgs = await prisma.organization.findMany({ select: { id: true } });
        logger.info(`Depreciation cron: processing ${orgs.length} organizations`);

        for (const org of orgs) {
          try {
            // Process FISCAL track
            await runDepreciationBatch({
              organizationId: org.id,
              periodYear: prevYear,
              periodMonth: prevMonth,
              track: 'FISCAL',
              triggeredBy: 'cron',
            });
            // Process MANAGERIAL track
            await runDepreciationBatch({
              organizationId: org.id,
              periodYear: prevYear,
              periodMonth: prevMonth,
              track: 'MANAGERIAL',
              triggeredBy: 'cron',
            });
          } catch (err) {
            // Log per-org failure but continue with other orgs
            logger.error({ err, orgId: org.id }, 'Depreciation cron: failed for org');
          }
        }

        logger.info('Depreciation cron: completed');
      } catch (err) {
        logger.error({ err }, 'Depreciation cron: failed');
      } finally {
        await redis.del(lockKey);
      }
    },
    { timezone: 'America/Sao_Paulo' },
  );
}
```
