## Components Inventory

Phase 17 introduces the following new frontend components. All follow the existing pattern established in Phase 16 assets components.

### New Page

**`DepreciationPage.tsx` + `DepreciationPage.css`**

Layout: standard page shell — breadcrumb + page title + header action bar + content area.

Header action bar (right-aligned):

- `DepreciationRunBadge` (last run status)
- Month/year selector (period picker — `<select>` for year, `<select>` for month)
- Track selector (`<select>`: "Fiscal (RFB)" | "Gerencial")
- "Executar Depreciação" button (primary, 1 per page)
- Export dropdown: CSV / XLSX / PDF (secondary button with chevron)

Content: `DepreciationReportTable` filling the remaining viewport.

Breadcrumb: Patrimônio > Depreciação

### New Components

**`DepreciationConfigModal.tsx` + `.css`**

Triggered from: AssetDrawer (new "Depreciação" tab) or inline from `DepreciationReportTable` warning row.

Modal size: medium (max-width 640px).

Form fields:

- Método de depreciação: `<select>` — Linear / Horas-uso / Produção / Acelerada
- Taxa anual fiscal (%): `<input type="number">` — pre-filled from `DEFAULT_RFB_RATES` by asset type
- Taxa anual gerencial (%): `<input type="number">` — optional, defaults to fiscal rate
- Vida útil (meses): `<input type="number">` — shown only when method = Linear, pre-filled from RFB defaults
- Valor residual (R$): `<input type="number">` — default 0
- Fator de aceleração: `<input type="number">` — shown only when method = Acelerada, default 2.0
- Total de horas: `<input type="number">` — shown only when method = Horas-uso
- Total de unidades: `<input type="number">` — shown only when method = Produção
- Centro de custo: read-only display from asset record (not editable here)

Conditional field visibility: fields toggle via controlled state, no layout shift — hidden fields use `display: none` not `visibility: hidden`.

RFB hint: inline info text below taxa anual fiscal showing "Taxa RFB padrão para {tipo}: X%" when a default exists — `var(--color-info-500)` with info icon.

Footer: "Cancelar" (tertiary) + "Salvar configuração" (primary, 1 per modal).

**`DepreciationReportTable.tsx`**

A `<table>` with columns:

| Ativo | Tipo | Fazenda | Valor Anterior | Depreciação | Valor Atual | Centro de Custo | Track |

- Numeric columns (Valor Anterior, Depreciação, Valor Atual): right-aligned, JetBrains Mono 14px
- "Depreciação" column: bold text when amount > 0; `var(--color-neutral-400)` when zero (fully depreciated or residual)
- Warning row state: asset rows with `skipped` reason show `var(--color-warning-100)` background + warning icon + reason text in a collapsed sub-row
- Pagination: standard prev/next at 20 rows/page (matching AssetsPage pattern)
- `<768px`: transform to stacked cards — each card shows asset name + depreciation amount prominently

**`DepreciationRunBadge.tsx`**

A compact status indicator rendered in the page header.

States:

- No run yet: gray dot + "Nenhuma execução" text
- PENDING: amber dot + "Em execução..." text + spinner (16px)
- COMPLETED: green dot (`var(--color-primary-600)`) + "Executado em DD/MM/AAAA HH:mm"
- PARTIAL: amber dot + "Parcial — N ativos ignorados" + link to skipped list
- FAILED: red dot + "Falhou — ver detalhes" + expand icon

**New AssetDrawer tab: "Depreciação"**

Added to existing `AssetDrawer.tsx` TABS array as `{ id: 'depreciacao', label: 'Depreciação' }`.

Tab content:

- If no `DepreciationConfig`: empty state with TrendingDown icon (48px) + "Sem configuração de depreciação" + "Configurar" CTA button
- If config exists: summary card (método, taxa, vida útil restante, valor contábil atual) + last 12 entries in a mini-table + "Editar configuração" button

## Interaction Contracts

### DepreciationPage load sequence

1. Page mounts → show skeleton rows (5 rows × 6 columns) in `DepreciationReportTable`
2. Fetch last `DepreciationRun` for current period + track → show `DepreciationRunBadge`
3. Fetch report data → replace skeletons with rows
4. If `skippedCount > 0`: show warning banner below header — "N ativos foram ignorados nesta execução. Ver detalhes."

### "Executar Depreciação" button interaction

1. User clicks → open confirmation modal: "Executar depreciação de {month}/{year} — track {Fiscal|Gerencial}?" with asset count and estimated total (if available from a dry-run preview endpoint)
2. Confirmation: "Cancelar" + "Executar" (primary)
3. On confirm: button enters loading state (spinner + "Executando..."), `DepreciationRunBadge` shows PENDING
4. On success: toast "Depreciação de {month}/{year} executada com sucesso. {N} ativos processados." + table refreshes
5. On partial success (skippedCount > 0): toast warning "Depreciação executada com {N} ativos ignorados. Verifique os alertas."
6. On failure: toast error (persistent) "Não foi possível executar a depreciação. Tente novamente ou verifique os logs."

Confirmation approach: `ConfirmModal` with `variant="warning"` (medium criticality — not destructive, but irreversible for the period).

### Reversal interaction

1. In `DepreciationReportTable`, each row has a "..." overflow menu with "Estornar lançamento"
2. Click → `ConfirmModal` with `variant="danger"`: "Estornar depreciação de {asset} — {month}/{year}? Esta ação não pode ser desfeita."
3. On confirm: POST reversal → row updates with "Estornado" badge + `var(--color-neutral-400)` text
4. Toast: "Lançamento de depreciação estornado com sucesso."

### DepreciationConfigModal — method change

When method `<select>` changes:

- Conditional fields appear/disappear with CSS `display: none` toggle (no animation — data fields, per CLAUDE.md "never animate layout shifts")
- RFB hint text updates immediately to reflect new method's default rate

### Empty state — no entries for period

`DepreciationReportTable` empty state:

- Icon: `TrendingDown` from lucide-react, 48px, `var(--color-neutral-400)`
- Heading: "Nenhuma depreciação neste período"
- Body: "Execute a depreciação mensal ou selecione outro período."
- CTA: "Executar Depreciação" button (primary)

### Assets tab "Depreciação" — no config empty state

- Icon: `Settings` from lucide-react, 48px, `var(--color-neutral-400)`
- Heading: "Ativo sem configuração de depreciação"
- Body: "Configure o método e as taxas para calcular o valor contábil deste ativo."
- CTA: "Configurar depreciação" button (primary, accent color)

## Accessibility Contract

- All table `<th>` elements: `scope="col"` attribute
- Numeric cells: `aria-label` with full description when value alone is ambiguous (e.g., `aria-label="Valor anterior: R$ 45.000,00"`)
- `DepreciationRunBadge` status dot: `role="status"` + `aria-live="polite"` for polling updates
- `DepreciationConfigModal`: `role="dialog"`, `aria-labelledby` pointing to modal title `<h2>`
- Conditional form fields: use `hidden` attribute (not CSS `visibility`) or `aria-hidden="true"` + `display: none` when not applicable
- Warning banners: `role="alert"` for skipped assets count
- Tab interface in `AssetDrawer` "Depreciação" tab: `role="tab"`, `aria-selected`, `aria-controls` (already established in AssetDrawer pattern)
- Period selectors: `<label>` associated via `htmlFor`, not placeholder-only
- Export button with dropdown: `aria-haspopup="menu"`, `aria-expanded` on trigger
- Focus trap in `DepreciationConfigModal`: standard modal pattern (already used in ConfirmModal)
- `prefers-reduced-motion`: no animations on table data changes; skeleton pulse respects the existing `@media (prefers-reduced-motion)` block in `tokens.css`

## Animation Contract

All timings use existing `tokens.css` duration vars:

| Interaction         | Duration | Easing               | CSS Var                                       |
| ------------------- | -------- | -------------------- | --------------------------------------------- |
| Modal open          | 300ms    | ease-out             | `var(--duration-normal)` + `var(--ease-out)`  |
| Modal close         | 200ms    | ease-in              | `var(--duration-fast)` + `var(--ease-in)`     |
| Table row hover     | 100ms    | ease-out             | `var(--duration-instant)` + `var(--ease-out)` |
| Badge status change | 200ms    | ease-out             | `var(--duration-fast)` + `var(--ease-out)`    |
| Skeleton pulse      | 1.5s     | ease-in-out infinite | `var(--ease-in-out)`                          |

No animation on:

- Conditional field show/hide in `DepreciationConfigModal` (layout shift — prohibited per CLAUDE.md)
- Table data refresh on run completion (data integrity — user must see new values clearly)
- Numeric value changes in report rows
