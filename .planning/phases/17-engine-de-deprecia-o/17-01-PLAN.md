**2. Create `depreciation-engine.service.ts`:**

Pure arithmetic functions — NO database access. All use `decimal.js` with `Decimal.ROUND_HALF_UP`.

Functions to implement:

- `daysInMonth(year: number, month: number): number` — uses `new Date(year, month, 0).getDate()` (month is 1-based)
- `getProRataDays(periodYear, periodMonth, acquisitionDate, disposalDate): { days: number; totalDays: number }` — returns full month days unless acquisition/disposal falls in the period
- `computeDepreciation(input: EngineInput): EngineOutput` — dispatches by method:
  - `STRAIGHT_LINE`: `depreciableValue * (annualRate / 12)` or `depreciableValue / usefulLifeMonths`. Uses rate from `config.track` (fiscal or managerial). Pro-rata adjustment: `monthlyAmount * days / totalDays`.
  - `ACCELERATED`: `openingBookValue * (annualRate * accelerationFactor / 12)`. Pro-rata applied.
  - `HOURS_OF_USE`: `depreciableValue / totalHours * periodicHours`. Returns `{ skipped: true, skipReason: 'HOURS_OF_USE requires periodicHours' }` if `periodicHours` is missing.
  - `UNITS_OF_PRODUCTION`: `depreciableValue / totalUnits * periodicUnits`. Returns `{ skipped: true, skipReason: 'UNITS_OF_PRODUCTION requires periodicUnits' }` if `periodicUnits` is missing.
  - Residual value clamping: `const remaining = openingBookValue.minus(residualValue); final = Decimal.min(proRated, remaining)`. If `remaining.isZero()` or negative, return `{ skipped: true, skipReason: 'Valor residual atingido' }`.
  - Final amount rounded to 2dp via `toDecimalPlaces(2, Decimal.ROUND_HALF_UP)`.

Set `Decimal.set({ rounding: Decimal.ROUND_HALF_UP })` at module level.
