## Page Structure: `/sped-ecd`

### Layout

Single page at route `/sped-ecd`. Follows the same structural pattern as TrialBalancePage and DrePage.

```
<main id="main-content" class="sped-ecd-page">
  <nav aria-label="Caminho da página">  ← Breadcrumb: Início / Contabilidade / SPED / Relatórios
  <header>                              ← Page title + icon
  <section aria-label="Período">        ← Fiscal year selector (shared between both tabs)
  <nav role="tablist">                  ← Tab rail: "SPED ECD" | "Relatório Integrado"
  <section role="tabpanel">             ← Active tab content
```

### Tab: SPED ECD

Content order (top to bottom):

1. **Validation panel** — shown immediately after fiscal year selection, auto-triggered (D-16).
   - Loading state: skeleton row (3 rows) while validation runs.
   - Success state: single row with CheckCircle (green) + "Pronto para geração. Nenhum problema encontrado."
   - Error/warning state: list of validation items, each as a row.
   - Each row: `[severity badge] [message] [link to fix]`
   - Severity badge "ERRO" uses `--color-error-500` background tint (`--color-error-100`), XCircle icon.
   - Severity badge "AVISO" uses `--color-warning-500` background tint (`--color-warning-100`), AlertTriangle icon.
   - Links point to `/chart-of-accounts`, `/fiscal-periods`, etc. — use `<Link>` component (React Router), not `<a>`.

2. **Action bar** — "Gerar SPED ECD" primary button.
   - Disabled state if any ERRO items exist (not just greyed color: `aria-disabled="true"` + `disabled` attribute + tooltip "Corrija os erros acima para gerar o arquivo").
   - Loading state during generation: spinner inside button + label "Gerando...".
   - On success: browser triggers file download (Content-Disposition: attachment), no modal needed.
   - File name displayed in toast: "Arquivo SPED_ECD_{CNPJ}_{ANO}.txt baixado com sucesso."

### Tab: Relatório Integrado

Content order (top to bottom):

1. **Farm selector** — optional dropdown to filter by farm/cost center (draws from same `/org/{id}/cost-centers` endpoint used by DrePage).

2. **Notas explicativas** — textarea section:
   - Label: "Notas explicativas adicionais"
   - Helper text (14px, muted): "O sistema gera automaticamente políticas contábeis, CPC 29 e contexto operacional. Use este campo para observações específicas do contador."
   - Textarea: min 4 rows, resizable vertically, plain text (no rich text editor).
   - Autosave: debounced 2s after last keystroke. Visual indicator: "Salvo" badge (muted text, checkmark icon) fades in after save, fades out after 3s.
   - No unsaved changes blocker — autosave is best-effort.

3. **Action bar** — "Gerar Relatório PDF" primary button.
   - Enabled as long as a fiscal year is selected (no blocking validations for PDF).
   - Loading state: spinner inside button + label "Gerando PDF...".
   - On success: browser triggers file download. Toast: "Relatório integrado gerado com sucesso."

## Fiscal Year Selector (Shared)

The `<select>` for fiscal year sits above the tab rail and applies to both tabs. This matches the pattern from DrePage.tsx and BalanceSheetPage.

```html
<section class="sped-ecd-page__filter-bar" aria-label="Período">
  <div class="sped-ecd-page__filter-group">
    <label for="sped-fy-select">Exercício Fiscal</label>
    <select id="sped-fy-select">
      <option value="">Selecione o exercício</option>
      …
    </select>
  </div>
</section>
```

- No month selector (D-03: full fiscal year, no monthly breakdown).
- On change: validation runs automatically (D-16). Show skeleton in validation panel immediately.
- Empty selection: validation panel is hidden, both action buttons disabled.

```

│  [heading h2, 16px/600] Pré-validação SPED                             │
│                                                                        │
│  ── Loading state ──────────────────────────────────────────────────   │
│  [skeleton row × 3]                                                    │
│                                                                        │
│  ── All-clear state ────────────────────────────────────────────────   │
│  [CheckCircle 20px green] Pronto para geração. Nenhum problema.        │
│                                                                        │
│  ── Issues state ───────────────────────────────────────────────────   │
│  [N erros • M avisos]  ← summary line, 14px/600                        │
│                                                                        │
│  [ERRO badge] [XCircle 16px] Contas sem mapeamento SPED: 3 contas      │
│               → Corrigir no Plano de Contas                            │
│                                                                        │
│  [ERRO badge] [XCircle 16px] Período 03/2025 não fechado               │
│               → Ir para Fechamento Mensal                              │
│                                                                        │
│  [AVISO badge] [AlertTriangle 16px] 12 contas sem movimento no período │
│                (não requer ação)                                       │
└────────────────────────────────────────────────────────────────────────┘
```

- Panel border: 1px `--color-neutral-200`, border-radius 8px.
- ERRO row background: `--color-error-100` (#FFEBEE).
- AVISO row background: `--color-warning-100` (#FFF8E1).
- All-clear row background: `--color-success-100` (#E8F5E9).
- Correction links: 14px, `--color-primary-600`, underline on hover.
- `role="alert"` on the panel wrapper so screen readers announce state changes.
- `aria-live="polite"` on the issues list.

## Interaction Contracts

### Validation trigger (D-16)
- Trigger: `onChange` on fiscal year `<select>`.
- Immediately show skeleton in validation panel.
- Call `GET /org/{id}/sped-ecd/validate?fiscalYearId={id}`.
- On response: replace skeleton with result list or all-clear state.
- Disable "Gerar SPED ECD" if response contains any item with `severity === 'ERROR'`.

### SPED download (D-01, D-12)
- Call `GET /org/{id}/sped-ecd/download?fiscalYearId={id}`.
- Backend responds with `Content-Disposition: attachment; filename="SPED_ECD_{CNPJ}_{ANO}.txt"`.
- Frontend creates `<a>` with `blob:` URL and clicks it, then revokes. No new tab.
- On error: show error toast.

### PDF download (D-18, D-21)
- Call `GET /org/{id}/sped-ecd/report?fiscalYearId={id}&costCenterId={optional}`.
- Same download pattern as SPED (blob URL).
- "Gerar Relatório PDF" enters loading state while request is in flight.

### Autosave notes (D-25)
- `onChange` on textarea → set `autosaveStatus = 'pending'`.
- Debounce 2000ms → call `PATCH /org/{id}/sped-ecd/notes` with `{ notesText: string }`.
- On success: set `autosaveStatus = 'saved'` → show "Notas salvas" badge → fade out after 3s.
- On error: show error toast "Não foi possível salvar as notas. Tente novamente."

### Tab switching
- State preserved: fiscal year selection and notes textarea content survive tab switches.
- Validation panel only visible in tab "SPED ECD".
- No network requests triggered by tab switch.

## Accessibility Checklist

- `<main id="main-content">` — skip-link target.
- Breadcrumb: `<nav aria-label="Caminho da página">`.
- Tab rail: `<nav role="tablist">`, each tab is `<button role="tab" aria-selected aria-controls>`.
- Tab panels: `<section role="tabpanel" aria-labelledby>`.
- Fiscal year `<select>`: `<label htmlFor>` always visible, never placeholder-only.
- Validation panel: `role="alert"` on wrapper, `aria-live="polite"` on list.
- "Gerar SPED ECD" button disabled state: `disabled` attribute + `aria-disabled="true"` + `title` for tooltip.
- Notes textarea: `<label htmlFor>` + `aria-describedby` pointing to helper text `<p>`.
- Autosave badge: `aria-live="polite"` region so screen readers announce "Notas salvas".
- Error icons (XCircle): `aria-hidden="true"` — severity communicated by badge text.
- All interactive elements: minimum 48x48px touch target.
- Focus outline: `2px solid var(--color-primary-500)`, `outline-offset: 2px` — never removed.
- `prefers-reduced-motion`: skeleton pulse and autosave fade-in respect `@media (prefers-reduced-motion: reduce)`.
- Correction links in validation panel: `<Link>` with descriptive text, not "clique aqui".
