| Token | CSS Var      | Value | Usage                                             |
| ----- | ------------ | ----- | ------------------------------------------------- |
| xs    | `--space-1`  | 4px   | Icon gaps, badge padding, chip inline padding     |
| sm    | `--space-2`  | 8px   | Compact row gaps, inline element spacing          |
| md    | `--space-4`  | 16px  | Default element spacing, input horizontal padding |
| lg    | `--space-6`  | 24px  | Card internal padding, section padding            |

- Touch targets (mobile): minimum 48x48px — all approval action buttons (Aprovar / Rejeitar / Devolver) must meet this regardless of icon-only or labeled presentation

- Notification bell dropdown item: 12px vertical, 16px horizontal (`--space-3` / `--space-4`)
- Timeline event connector line: 2px width, not a spacing value — use border-left on the timeline track

| ---------- | ------------- | --------------- | ------------- | ---------------------- | ------------------------ | ---------------------------------------------------------------- |
| Body       | `--text-sm`   | 14px (0.875rem) | Source Sans 3 | 400 (`--font-regular`) | 1.5 (`--leading-normal`) | Table cell content, notification body, modal field values        |
| Label      | `--text-base` | 16px (1rem)     | Source Sans 3 | 400 (`--font-regular`) | 1.5                      | Form labels, sidebar items, tab labels                           |
| Subheading | `--text-md`   | 18px (1.125rem) | DM Sans       | 700 (`--font-bold`)    | 1.3 (`--leading-snug`)   | Modal section headers (Dados Gerais, Itens, Anexos), card titles |
| Heading    | `--text-xl`   | 24px (1.5rem)   | DM Sans       | 700 (`--font-bold`)    | 1.2 (`--leading-tight`)  | Modal title (RC-2026/0001), page title ("Requisições de Compra") |

- ALL CAPS only on status badge labels: RASCUNHO, PENDENTE, APROVADA, REJEITADA, DEVOLVIDA, CANCELADA; urgency badges: NORMAL, URGENTE, EMERGENCIAL

| Role            | CSS Var               | Hex     | Usage                                                                                                  |
| --------------- | --------------------- | ------- | ------------------------------------------------------------------------------------------------------ |
| Dominant (60%)  | `--color-neutral-50`  | #FAFAF8 | Page background, modal background                                                                      |
| Secondary (30%) | `--color-neutral-100` | #F5F3EF | Cards, sidebar, table rows (even), notification dropdown background                                    |
| Accent (10%)    | `--color-primary-600` | #2E7D32 | Primary CTA buttons only — "Nova Requisição", "Enviar para Aprovação", "Salvar Rascunho" submit action |
| Destructive     | `--color-error-500`   | #C62828 | Rejeitar action button, cancel RC confirmation, error messages                                         |

Accent reserved for (explicit list — nothing else):

1. "Nova Requisição" primary CTA button on PurchaseRequestsPage
2. "Enviar para Aprovação" submit action in PurchaseRequestModal footer
3. "Salvar Rascunho" submit action in PurchaseRequestModal (secondary variant: outlined `--color-primary-600`)
4. "Aprovar" button in approval pending list (mobile and web)
5. Notification bell unread count badge background
6. Active sidebar item indicator

Secondary semantic colors (not accent, not destructive):

- Warning / Urgente: `--color-warning-500` (#F57F17) — urgency badge for URGENTE level
- Error / Emergencial: `--color-error-500` (#C62828) — urgency badge for EMERGENCIAL level; also used for SLA deadline exceeded indicator
- Normal urgency: `--color-neutral-500` (#7A7267) — NORMAL urgency badge (muted, no urgency)
- RASCUNHO badge: `--color-neutral-400` (#A8A196)
- PENDENTE badge: `--color-warning-500` (#F57F17) background-tint `--color-warning-100`
- APROVADA badge: `--color-success-500` (#2E7D32) background-tint `--color-success-100`
- REJEITADA badge: `--color-error-500` (#C62828) background-tint `--color-error-100`
- DEVOLVIDA badge: `--color-sky-500` (#0288D1) background-tint `--color-sky-100`
- CANCELADA badge: `--color-neutral-500` (#7A7267) background-tint `--color-neutral-100`

Contrast rule: WCAG AA — 4.5:1 for body text, 3:1 for large text. Never use color as the only indicator — always pair with icon or text label.

## Component Inventory

All components follow the established Phase 7 pattern: `Page + Modal + hook`.

### Web Components

#### PurchaseRequestsPage

- Layout: full-width table (same pattern as SuppliersPage)
- Table columns: Número (mono font) | Tipo | Fazenda | Urgência | Itens | Total Estimado | Status | Data | Ações
- Ações column: eye icon (ver detalhes) + edit icon (only if RASCUNHO or DEVOLVIDA) — 20px Lucide icons
- Filter bar above table: status filter (all / RASCUNHO / PENDENTE / APROVADA / REJEITADA / DEVOLVIDA / CANCELADA) + search by number or description
- "Nova Requisição" button: top-right, primary accent, ShoppingCart icon (20px) + label, max 1 per page
- Breadcrumb: Início > Compras > Requisições
- Skeleton loading: 3 table-row skeletons (same height as data rows) — no spinner
- Empty state (no RCs yet): ShoppingCart icon 48px (`--color-neutral-300`), heading "Nenhuma requisição ainda", body "Crie a primeira requisição de compra da sua organização.", CTA "Nova Requisição" (accent)
- Empty state (filtered, no results): Search icon 48px, heading "Nenhum resultado encontrado", body "Tente ajustar os filtros.", no CTA

#### PurchaseRequestModal (create / edit)

- Trigger: "Nova Requisição" or edit icon
- Size: large full-width modal — matches existing BulkImportModal pattern; max-width 900px, centered, scrollable body
- Header: title "Nova Requisição" (create) or "Editar Requisição RC-YYYY/NNNN" (edit); X close button (top-right)
- Three sections with dividers between them:
  1. **Dados Gerais**: Tipo* (select using SUPPLIER_CATEGORIES enum), Fazenda* (select), Urgência\* (radio or segmented: Normal / Urgente / Emergencial), Centro de Custo (select, optional), Data de Necessidade (date input, optional), Justificativa (textarea, required only when Emergencial)
  2. **Itens** (tabela editável): columns: Produto/Descrição* | Qtd* | Unidade\* | Preço Unit. Estimado | Observação | (remove row icon); "Adicionar Item" button below table; minimum 1 item enforced with inline validation
  3. **Anexos**: drag-and-drop upload zone with dashed border, list of uploaded files with remove option
- Autocomplete for product search: shows product name + unit from catalog; fallback "Usar descrição livre" option at bottom of results list
- Footer: "Cancelar" (ghost) + "Salvar Rascunho" (outlined accent) + "Enviar para Aprovação" (filled accent)
- Validation: onBlur inline per field; required fields marked with \*; error messages below field with `role="alert"` and AlertCircle icon 16px
- Emergencial urgency: reveals Justificativa field with smooth 200ms height transition (CSS max-height)
- Closing unsaved changes: ConfirmModal with `variant="warning"` — "Descartar alterações? As mudanças não serão salvas."

#### PurchaseRequestDetailModal (read-only + actions + timeline)

- Opens from eye icon or from notification link
- Sections: header (RC number, status badge, urgency badge, farm name, created-by, created-at) | items table (read-only) | attachments list | approval timeline | action bar (for approvers)
- Action bar (visible only to approver with pending step): three buttons — "Aprovar" (accent), "Devolver" (outlined warning), "Rejeitar" (outlined destructive); each opens a comment textarea inline below the action bar before confirming
- Aprovar: comment optional; confirm with "Confirmar Aprovação" button
- Devolver / Rejeitar: comment required; error if empty; confirm buttons "Confirmar Devolução" / "Confirmar Rejeição"
- Rejeitar confirmation: ConfirmModal with `variant="danger"` — "Rejeitar esta requisição? O solicitante será notificado com o motivo informado."
- Cancelar RC (status APROVADA, before Phase 9): ConfirmModal with `variant="danger"` — "Cancelar a requisição RC-YYYY/NNNN? Ela não poderá ser reativada."

#### Approval Timeline (dentro de PurchaseRequestDetailModal)

- Vertical left-aligned timeline (source: CONTEXT.md Claude's Discretion — this is the chosen format)
- Track: 2px border-left, `--color-neutral-200`, positioned at left edge of icon column
- Each event: circle icon (16px filled) on the track line + actor name (DM Sans 700, bold) + action label + relative time ("há 2 horas") + absolute datetime on hover tooltip
- Event icon colors by action type: criada (#7A7267 neutral) | enviada (#F57F17 warning) | aprovada (#2E7D32 success) | rejeitada (#C62828 error) | devolvida (#0288D1 sky) | cancelada (#7A7267 neutral)
- Comment (if present): indented block below the event row, italic, `--color-neutral-600`
- Delegation indicator: small badge "por delegação de [nome]" in `--color-neutral-400` below the actor name
- Skeleton: 3 event placeholder rows with pulsing opacity 0.4→0.7 at 1.5s infinite

#### ApprovalRulesPage

- Layout: vertically stacked rule cards (not a table — rules have complex nested structure)
- Each card: rule name | type filter (or "Todos os tipos") | value range (R$ X – R$ Y) | approver chain (1 or 2 avatars/names + step arrows) | priority chip | active/inactive toggle
- Cards ordered by priority (ascending) — priority number shown in top-left corner of card
- "Nova Regra de Alçada" button: top-right, accent, Settings2 icon + label
- Drag-and-drop reorder: drag handle (GripVertical icon, 20px) on left side of each card to reorder priority — visual drop zone indicator with `--color-primary-100` background
- Empty state: Settings2 icon 48px, heading "Nenhuma regra configurada", body "Configure as regras de alçada para aprovar requisições de compra.", CTA "Nova Regra de Alçada"

#### ApprovalRuleModal (create / edit)

- Size: medium modal, max-width 640px
- Fields: Nome da regra* | Tipo de RC (select or "Todos") | Valor mínimo (R$) | Valor máximo (R$, optional — "sem limite superior") | Aprovador 1* (user select) | Aprovação dupla? (checkbox) → reveals Aprovador 2* (user select) | Prioridade* (number input, 1–99)
- Validation: value range must be logically consistent (min < max); warning if ranges overlap with existing rules

#### NotificationBell (header component)

- Position: in `app-topbar__right`, before existing separator
- Icon: Bell (Lucide, 24px), `--color-neutral-600`
- Unread badge: red filled circle (`--color-error-500`) with white count number (12px badge exception), positioned top-right of bell icon, hidden when count is 0
- Click: opens dropdown panel (300ms ease-out scale + fade from top-right origin)
- Dropdown: max-width 380px, max-height 480px scrollable, `--shadow-xl`, `--radius-lg`
- Dropdown header: "Notificações" (heading) + "Marcar todas como lidas" link (right-aligned)
- Each notification item: clickable row with 12px vertical / 16px horizontal padding, `--color-neutral-0` background → `--color-neutral-50` on hover; unread items have 3px left border `--color-primary-600`; layout: icon (notification type, 20px) | title (DM Sans 700, bold, 14px) + body (14px, 2-line truncation) | time (relative, 12px, `--color-neutral-500`)
- Notification type icons: CheckCircle (RC_APPROVED, success color) | XCircle (RC_REJECTED, error color) | RotateCcw (RC_RETURNED, warning color) | Clock (SLA_REMINDER, warning color) | ShoppingCart (RC_PENDING, neutral)
- Empty state (no notifications): Bell icon 40px muted, "Nenhuma notificação" centered, no CTA
- Polling: 30-second interval — bell re-fetches `GET /org/notifications?unread=true&limit=20`; no loading indicator during poll (background update, only show new count when changed)
- Keyboard: Escape closes dropdown; focus trap inside dropdown when open

#### DelegationModal (within ApprovalRulesPage or user profile)

- Medium modal, max-width 540px
- Fields: Período — De* (date) + Até* (date) | Substituto\* (user select) | Observação (textarea, optional)
- Validation: start date must be today or future; end date must be after start date
- Active delegation shown as banner in ApprovalRulesPage when current user has an active delegation configured

### Sidebar Addition

- Group COMPRAS (existing): add two items below "Fornecedores"
  - `ShoppingCart` icon (20px) + "Requisições" → `/purchase-requests`
  - `Settings2` icon (20px) + "Alçadas" → `/approval-rules` (shown only if user has `purchases:manage` permission)

### Mobile Screens

#### purchase-request.tsx (Nova RC simplificada)

- Full-screen form, ScrollView
- Fields (top to bottom): Produto* (search input with autocomplete from cached catalog OR text input fallback) | Quantidade* (numeric input) | Unidade* (auto-filled from catalog or text) | Urgência* (segmented control: Normal / Urgente / Emergencial) | Foto (camera button, optional) | Observação (multiline text, optional)
- Geolocation: auto-captured silently on form load; no user-facing field
- Farm: auto-populated from logged-in user context; read-only display chip at top of form
- Center de custo: NOT shown (locked decision)
- Footer: sticky bottom bar with "Enviar Requisição" primary button and "Salvar Rascunho" secondary
- Offline mode: form submits normally; UI shows "Requisição salva offline. Será enviada quando houver conexão." toast (blue info) after submit; badge visible on the request in "Minhas Requisições" list

#### my-requests.tsx (Minhas Requisições)

- FlatList of RC cards
- Filter tabs at top: Todas | Pendente | Aprovada | Rejeitada (horizontal scroll, no ALL CAPS — sentence case for tab labels in mobile)
- Each card: RC number (mono, 14px) | Tipo | Data | Status badge | Urgência chip | Tap → detail screen (read-only)
- Offline badge: gray syncing indicator on unsent RCs ("Pendente envio")
- Empty state: ShoppingCart 48px, "Você não tem requisições ainda.", "Nova Requisição" CTA button

#### Pending approvals screen (mobile) — accessible via "Mais" tab

- List of RCs awaiting the current user's approval step
- Each item: RC number | Solicitante | Valor total estimado | Urgência chip | SLA countdown (only for URGENTE/EMERGENCIAL)
- Tap → PurchaseRequestDetailScreen with Aprovar / Rejeitar / Devolver action buttons at bottom (sticky footer, 48px tall buttons minimum)
- Rejeitar / Devolver: slide-up bottom sheet with comment text field and confirm button

Text is ALL CAPS at 12px (badge exception — not a scale step). Icon always present (never color alone).

### Urgency Chips

| Level       | Background            | Text                  | Additional                                                                                   |
| ----------- | --------------------- | --------------------- | -------------------------------------------------------------------------------------------- |
| NORMAL      | `--color-neutral-100` | `--color-neutral-500` | —                                                                                            |
| URGENTE     | `--color-warning-100` | `--color-warning-500` | AlertTriangle icon 12px                                                                      |
| EMERGENCIAL | `--color-error-100`   | `--color-error-500`   | AlertOctagon icon 12px + pulsing dot animation (2s, ease-in-out, reduced-motion: static dot) |

### SLA Indicator

- Shown on PENDENTE RCs with urgency URGENTE or EMERGENCIAL
- In table: small clock icon + "X h restantes" or "Prazo vencido" text
- "Prazo vencido": `--color-error-500` text + AlertTriangle icon
- "< 2h restantes": `--color-warning-500` text
- "> 2h restantes": `--color-neutral-500` text (muted)

### Form Validation States

- Field at rest: `--color-neutral-300` border, 1px
- Field focused: `--color-primary-600` border, 2px, outline 2px `--color-primary-500` offset 2px
- Field error: `--color-error-500` border, 1px; error message below with AlertCircle 16px + `role="alert"`
- Field valid (onBlur): `--color-neutral-300` border (no green — avoid visual noise for optional fields)

### Loading States

- Table loading: 5 skeleton rows, same row height as data rows, opacity 0.4→0.7 pulse at 1.5s infinite ease-in-out
- Modal opening: content skeleton — form field placeholders at 16px height and full width
- Notification bell: no loading state during background poll; only shows updated count

### Animation

- Modal enter: opacity 0→1 + translateY(8px→0), 300ms `--ease-out`
- Notification dropdown: opacity 0→1 + scaleY(0.95→1) from top-right origin, 200ms `--ease-out`
- Badge count change: scale 1.2→1.0, 100ms `--ease-spring`
- Urgency EMERGENCIAL pulsing dot: opacity 0.4→1.0, 2s ease-in-out infinite; `prefers-reduced-motion`: static dot, no animation
- Status badge appearance (after approval action): no animation — update via query invalidation and re-render

## Accessibility

All requirements sourced from CLAUDE.md + WCAG 2.1 AA:

- Semantic HTML: `<main>`, `<nav>`, `<section>`, `<button>` for all interactive elements — never `<div onClick>`
- Focus visible: `outline: 2px solid var(--color-primary-500); outline-offset: 2px` on all focusable elements — never `outline: none`
- Form fields: `<label htmlFor>` on every input, `aria-required="true"` on required fields, error messages with `role="alert"` + `aria-live="polite"`
- Icon-only buttons (edit, delete, view): `aria-label` required — examples: `aria-label="Ver detalhes da requisição RC-2026/0001"`, `aria-label="Editar requisição"`, `aria-label="Fechar notificações"`
- Bell badge: `aria-label="X notificações não lidas"` on the bell button, updating dynamically
- Status badges: include text inside badge — never icon-only for status
- Timeline: `<ol>` list with `<li>` per event; `<time datetime="ISO">` for all timestamps
- Modal: focus trap, Escape closes, focus returns to trigger on close
- Notification dropdown: focus trap, Escape closes, `role="menu"` + `role="menuitem"` or `role="listbox"` + `role="option"` on items
- Mobile: `accessibilityLabel` and `accessibilityRole` on all interactive elements; `accessibilityState={{ disabled }}` on disabled buttons
- `prefers-reduced-motion`: EMERGENCIAL pulsing dot becomes static; modal/dropdown animations disabled (transition: none)
