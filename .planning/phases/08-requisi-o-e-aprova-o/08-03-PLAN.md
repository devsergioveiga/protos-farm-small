Logic by action:

**SUBMIT** (RASCUNHO -> PENDENTE):

- Validate canTransition(rc.status, 'PENDENTE')
- Inside withRlsContext transaction:
  - Calculate total: sum of `items.estimatedUnitPrice * quantity` (where both exist)
  - Call `matchApprovalRule(tx, ctx.organizationId, rc.requestType, total)` from approval-rules.service
  - If no rule found, throw PurchaseRequestError('Nenhuma regra de alcada configurada para este tipo e valor', 400)
  - Resolve approver1: `resolveApprover(tx, rule.approver1Id, ctx.organizationId)`
  - Create ApprovalAction step 1: `{ purchaseRequestId: id, organizationId: ctx.organizationId, step: 1, assignedTo: resolved1, originalAssignee: resolved1 !== rule.approver1Id ? rule.approver1Id : null, status: 'PENDING' }`
  - If rule.approverCount === 2: resolve approver2, create ApprovalAction step 2 with status 'PENDING'
  - Compute slaDeadline: if SLA_HOURS[rc.urgency] is not null, set `slaDeadline = new Date(Date.now() + SLA_HOURS[rc.urgency] * 60 * 60 * 1000)`
  - Update RC: `status: 'PENDENTE', submittedAt: new Date(), slaDeadline`
  - Create notification for step 1 approver: type RC_PENDING, title "Nova requisicao para aprovar", body "RC-YYYY/NNNN aguarda sua aprovacao.", referenceId: id, referenceType: "purchase_request"
- After commit: `void dispatchPushNotification(notification).catch(...)`

**APPROVE** (PENDENTE -> APROVADA, or step 1 -> step 2):

- Find current user's pending ApprovalAction for this RC (assignedTo = ctx.userId, status = 'PENDING')
- If not found, throw PurchaseRequestError('Voce nao tem permissao para aprovar esta requisicao', 403)
- Inside transaction:
  - Update ApprovalAction: status = 'APPROVED', comment = input.comment, decidedAt = new Date()
  - Check if there's a next pending step (step 2 with status 'PENDING')
    - If yes: don't change RC status, notify step 2 approver with RC_PENDING
    - If no (all steps resolved): update RC to APROVADA, notify creator with RC_APPROVED

**REJECT** (PENDENTE -> REJEITADA):

- Validate comment required: if !input.comment, throw PurchaseRequestError('Motivo obrigatorio ao rejeitar', 400)
- Find current user's pending ApprovalAction
- Inside transaction:
  - Update ApprovalAction: status = 'REJECTED', comment, decidedAt
  - Update RC: status = 'REJEITADA'
  - Create notification for creator: type RC_REJECTED, body includes comment

**RETURN** (PENDENTE -> DEVOLVIDA):

- Validate comment required: if !input.comment, throw PurchaseRequestError('Motivo obrigatorio ao devolver', 400)
- Find current user's pending ApprovalAction
- Inside transaction:
  - Update ApprovalAction: status = 'RETURNED', comment, decidedAt
  - Update RC: status = 'DEVOLVIDA'
  - Create notification for creator: type RC_RETURNED, body includes comment

**CANCEL** (APROVADA -> CANCELADA):

- Validate canTransition(rc.status, 'CANCELADA')
- Update RC: status = 'CANCELADA', cancelledAt = new Date()
