Hook interface:

```typescript
interface UsePurchaseKanbanResult {
  cards: KanbanCardData[];
  cardsByColumn: Record<KanbanColumn, KanbanCardData[]>;
  isLoading: boolean;
  error: string | null;
  filters: KanbanFilters;
  setFilters: (filters: KanbanFilters) => void;
  transitionCard: (
    cardId: string,
    targetColumn: KanbanColumn,
  ) => Promise<{ success: boolean; error?: string }>;
  refresh: () => void;
}
```

Implementation:

- Fetch cards on mount and when filters change: `api.get<KanbanCardData[]>('/org/purchase-kanban', { params: filters })`
- Build URL with query params from filters (farmId, urgency, supplierId, startDate, endDate)
- Derive `cardsByColumn` by grouping cards by column field (computed from cards array, not separate state)
- `transitionCard`: POST to `/org/purchase-kanban/transition` with `{ cardId, targetColumn }`. On success, call refresh(). On error, return { success: false, error: err.message }.
- Skeleton loading pattern: isLoading true initially, false after first fetch

**1. Create `apps/frontend/src/pages/PurchaseKanbanPage.tsx`:**

Import DndContext, DragEndEvent, DragOverlay, DragStartEvent, PointerSensor, KeyboardSensor, useSensor, useSensors from '@dnd-kit/core'.

Page structure:

- Breadcrumb: "Compras > Kanban"
- Page title: `<h1>` "Kanban de Compras" (DM Sans 700)
- Filter bar below title: row of filter controls
  - Farm select (from useFarmContext or dedicated fetch)
  - Urgency select: options Normal, Urgente, Emergencial
  - Date range: startDate + endDate inputs (type="date")
  - Clear filters button
- Board area: horizontal scroll container with 7 columns
- ConfirmModal for drag transitions
- DragOverlay with KanbanCard preview

Implementation details:

- `const sensors = useSensors(useSensor(PointerSensor, { activationConstraint: { distance: 8 } }), useSensor(KeyboardSensor));`
- State: `activeCard: KanbanCardData | null` (for DragOverlay), `pendingTransition: { cardId: string; from: KanbanColumn; to: KanbanColumn } | null` (for ConfirmModal)
- `handleDragStart(event: DragStartEvent)`: set activeCard from event.active.data.current
- `handleDragEnd(event: DragEndEvent)`:
  - Extract `from` = active.data.current.column, `to` = over?.id (column id)
  - If from === to or !over, return (no-op)
  - Set pendingTransition = { cardId: active.id, from, to }
  - This opens ConfirmModal (LOCKED DECISION: never silent transitions)
- ConfirmModal onConfirm:
  - Call `transitionCard(pendingTransition.cardId, pendingTransition.to)`
  - On success: toast "Transicao realizada com sucesso", clear pendingTransition
  - On error: toast error message (the backend returns human-readable pt-BR errors like "Use a pagina de cotacoes para criar uma cotacao"), clear pendingTransition
- ConfirmModal props: `variant="warning"`, title="Confirmar transicao", description="Mover {card.number} de {fromLabel} para {toLabel}?"
