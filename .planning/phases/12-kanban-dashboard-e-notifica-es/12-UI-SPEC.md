## Spacing Scale

Declared values (all multiples of 4px — 4px base grid):

| Token | Value               | Usage                                                     |
| ----- | ------------------- | --------------------------------------------------------- |
| xs    | 4px (`--space-1`)   | Icon gaps, badge padding, inline tight spacing            |
| sm    | 8px (`--space-2`)   | Compact element spacing, card internal gaps               |
| md    | 16px (`--space-4`)  | Default element spacing, input padding horizontal         |
| lg    | 24px (`--space-6`)  | Section padding, card padding, column gaps                |
| xl    | 32px (`--space-8`)  | Layout gaps between KPI cards, board column width padding |
| 2xl   | 48px (`--space-12`) | Major section breaks, page-level vertical rhythm          |
| 3xl   | 64px (`--space-16`) | Page-level spacing, empty state icon size                 |

Exceptions:

- Kanban column horizontal scroll: columns have fixed width 280px with `--space-4` (16px) gap between them. This is a layout measure, not a spacing token.
- Touch targets: minimum 48×48px on all interactive elements (drag handles, toggle cells in preferences table).
- Drag handle area on KanbanCard: minimum 48px height to ensure easy grabbing.
- Input padding: minimum 12px vertical, 16px horizontal (per CLAUDE.md rule).

**Source:** `tokens.css` spacing scale confirmed. CLAUDE.md exceptions noted.

## Kanban Board Interaction Contract

### Layout

- Horizontal scroll container. 7 fixed-width columns (280px each), 16px gap.
- On screens < 1024px: columns visible via horizontal scroll with visible scrollbar (not hidden).
- Column header: heading text (20px DM Sans 500) + counter badge (pill, `--color-primary-600` bg when count > 0, `--color-neutral-300` when 0).
- Maximum visible cards per column: no limit. Column scrolls vertically within fixed viewport height.

### KanbanCard anatomy (exactly, top to bottom)

1. **Row 1:** sequential number (JetBrains Mono 14px `--color-neutral-800`) + urgency badge (right-aligned)
2. **Row 2:** entity type label (Source Sans 3 14px semibold `--color-neutral-600`) + farm name (Source Sans 3 14px regular `--color-neutral-500`)
3. **Row 3:** requester name (Source Sans 3 16px regular `--color-neutral-700`)
4. **Row 4:** total value (JetBrains Mono 16px `--color-neutral-800`) + days-in-stage (right-aligned, Source Sans 3 14px `--color-neutral-500`)

Overdue state: left border 3px solid `--color-error-500`. No background change. Icon `AlertCircle` (16px, `--color-error-500`, `aria-hidden`) appended after days-in-stage text.

Card hover: `translateY(-2px)` + `--shadow-lg`. Transition 200ms ease-out.
Card dragging (isDragging): opacity 0.4, no transform. DragOverlay clone is fully opaque with `--shadow-xl`.
Card minimum height: 80px (ensures 48px touch target for drag handle area).

### Urgency badges

| Value       | Label       | Background            | Text                                |
| ----------- | ----------- | --------------------- | ----------------------------------- |
| NORMAL      | Normal      | `--color-neutral-200` | `--color-neutral-700`               |
| URGENTE     | Urgente     | `--color-sun-100`     | `#7a5800` (sun-dark, contrast-safe) |
| EMERGENCIAL | Emergencial | `--color-error-100`   | `--color-error-500`                 |

### Drag interaction states

- **Valid drop target (during drag):** column highlighted with `--color-primary-100` background, 2px dashed border `--color-primary-500`.
- **Invalid drop target (during drag):** opacity 0.4, cursor `not-allowed`. No dashed border. `aria-disabled="true"` on droppable element.
- **Drag confirm modal:** When drag would execute a high-consequence transition (RC_PENDENTE → RC_APROVADA = approve request), show `ConfirmModal` with `variant="warning"`, title "Aprovar requisição?", confirm label "Aprovar".

### Column PAGO temporal limit

Cards shown: last 30 days only. Counter badge reflects total (not just visible). A "Ver todos" link below the column opens purchase-requests filtered by status=PAGO.

## Dashboard Interaction Contract

### Layout

- Full-width page. Max-width 1280px centered.
- Filter row (farm + period) pinned below page header. Period presets: "Mês", "Trimestre", "Safra", "Ano".
- 4 KPI cards in a responsive 2×2 grid (2 columns on < 1024px, 4 columns on ≥ 1024px).
- Charts section: 2-column grid on ≥ 1024px. Lazy-loaded with Suspense + skeleton placeholder.
- Alerts section: same pattern as FinancialDashboardPage alerts — distinct background `--color-warning-100`, icon + count + link.

### KPI cards

| KPI                         | Icon           | Overdue state                                |
| --------------------------- | -------------- | -------------------------------------------- |
| RCs pendentes de aprovação  | `ShoppingCart` | YoyBadge up = warning (more pending = worse) |
| OCs em atraso               | `AlertCircle`  | Value in `--color-error-500` when > 0        |
| Prazo médio do ciclo (dias) | `Clock`        | YoyBadge up = warning (slower = worse)       |
| Entregas atrasadas          | `Truck`        | Value in `--color-error-500` when > 0        |

KPI card click: navigate to `/purchasing-kanban?filter={preset}` via `useNavigate`. Only KPIs with overdue data are clickable. Non-clickable KPIs use `<div>` not `<button>`.

### Drill-down query params accepted by KanbanPage

| Param value        | Filter applied                                                            |
| ------------------ | ------------------------------------------------------------------------- |
| `overdue_po`       | Show only cards with `isOverdue: true` in OC_EMITIDA + AGUARDANDO_ENTREGA |
| `pending_approval` | Show only RC_PENDENTE column                                              |

## Notification Preferences Page Interaction Contract

### Layout

Matrix table: rows = notification event types, columns = Badge + Email.
Each cell: toggle (checkbox styled as toggle switch, 48px touch target minimum).
Opt-out model: absence of saved preference = enabled. Toggle ON = enabled (green), OFF = disabled (neutral gray).

| Column header  | Icon          |
| -------------- | ------------- |
| Badge (in-app) | `Bell` (20px) |
| Email          | `Mail` (20px) |

Row grouping by module:

1. Requisições (RC_APPROVED, RC_REJECTED, RC_RETURNED, RC_PENDING, SLA_REMINDER)
2. Cotações (QUOTATION_PENDING_APPROVAL, QUOTATION_APPROVED, QUOTATION_RECEIVED, QUOTATION_DEADLINE_NEAR)
3. Pedidos (PO_OVERDUE, PO_GOODS_RECEIVED)
4. Orçamento e Devoluções (BUDGET_EXCEEDED, RETURN_REGISTERED, RETURN_RESOLVED)
5. Resumo (DAILY_DIGEST — email only, badge column disabled/greyed)

Toggle save: auto-save on change (no submit button). Show inline success indicator (checkmark icon, 1s, then disappear). On error: revert toggle + show toast "Não foi possível salvar sua preferência. Tente novamente."

## Accessibility Contract

- `DndContext` from dnd-kit provides keyboard navigation out of the box (Space to pick up, Arrow keys to move, Space/Enter to drop, Escape to cancel). No additional implementation needed.
- KanbanCard: `<article>` element, `aria-label="{sequentialNumber} — {requesterName}, {totalValue}"`.
- KanbanColumn: `<section>`, `aria-label="Coluna {label}: {count} itens"`. When disabled during drag: `aria-disabled="true"`.
- Column counter badge: `aria-live="polite"` — updates when cards move.
- Toggle in preferences: `<button role="switch" aria-checked="{true|false}" aria-label="{eventType label} por {channel}"`.
- KPI cards (clickable): `<button>` with `aria-label="{kpi label}: {value}, ver detalhes"`. Non-clickable: `<div role="status">`.
- Charts: each chart `<figure>` with `<figcaption>` describing what the chart shows. Recharts provides `role="img"` on SVG.
- Form filters: all `<select>` elements have `<label>` (visible or `sr-only`). `aria-label` on search input.
- Focus visible: `outline: 2px solid var(--color-primary-500); outline-offset: 2px` — already in global CSS.
- Drag handles do NOT steal focus from card content — `{...listeners}` applied to full card, not a sub-element.
