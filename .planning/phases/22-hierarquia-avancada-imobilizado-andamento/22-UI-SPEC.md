## AssetHierarchyTab — Interaction Contract

**Trigger:** Tab visible only when `asset.parentAsset !== null` OR `asset.childAssets.length > 0`.

**Layout:** Vertical tree with indentation per level. Max 3 levels.

- Level 0 (root): no indent, bold label.
- Level 1 (child): 24px left indent.
- Level 2 (grandchild): 48px left indent.

**Each node shows:**
- Asset name + asset tag (mono font)
- Acquisition value in BRL (mono font, right-aligned)
- Status badge (reuse existing `StatusBadge` component from AssetDrawer)
- Clickable: clicking any node calls `onNavigate(assetId)` to open that asset in the drawer

**Parent node footer:** Shows totalized value of all active children (excludes ALIENADO + deletedAt != null) in a summary row: "Total filhos: R$ X.XXX,XX" in bold mono.

**Empty state (no hierarchy):** Not shown — tab is hidden when no hierarchy exists.

**Error state:** Inline `role="alert"` message: "Nao foi possivel carregar a hierarquia. Tente novamente."

## AssetRenovationModal — Interaction Contract

**Trigger:** "Registrar Reforma" button in AssetGeneralTab action menu. Button only shown when `asset.status === 'ATIVO'` or `asset.status === 'INATIVO'`. Never shown for `EM_ANDAMENTO` or `ALIENADO`.

**Modal size:** 560px max-width (consistent with AssetDisposalModal pattern).

**Form fields (all required unless marked optional):**

| Field | Type | Label | Validation |
|-------|------|-------|------------|
| Descricao | text input | "Descricao da reforma *" | Required, max 255 chars |
| Data da reforma | date input | "Data da reforma *" | Required, not future date |
| Valor total | currency input | "Valor total (R$) *" | Required, > 0, Decimal precision |
| Decisao contabil | radio group | "Classificacao contabil *" | Required — 2 options (see below) |
| Nova vida util (meses) | number input | "Nova vida util (meses)" | Optional, only enabled when CAPITALIZAR selected, integer > 0 |
| Observacoes | textarea | "Observacoes" | Optional, max 1000 chars |

**Decisao contabil radio options:**

- Option A — "Capitalizar": icon `TrendingUp`, description "Soma ao valor contabil do ativo. Use quando a reforma aumenta capacidade ou prolonga vida util."
- Option B — "Lancamento em despesa": icon `Receipt`, description "Registra como despesa no periodo. Use quando a reforma apenas restaura a condicao original."

Radio cards use the label-wrapping-hidden-input pattern (locked decision from Phase 19 in STATE.md).

**Field interaction:** "Nova vida util" field becomes enabled only when "Capitalizar" is selected. When "Lancamento em despesa" is selected, field is visually disabled (opacity 0.4) and not required.

**Submit label:** "Registrar Reforma"

**Success flow:** Modal closes → toast "Reforma registrada com sucesso" → AssetDrawer refetches asset detail (acquisitionValue + depreciationConfig update reflected).

**Error state:** Inline `role="alert"` below submit button: "Nao foi possivel registrar a reforma. Verifique os dados e tente novamente."

## AssetWipContributionsTab — Interaction Contract

**Trigger:** Tab visible only when `asset.status === 'EM_ANDAMENTO'`.

**Tab label:** "Andamento"

**Layout sections (top to bottom):**

1. **Budget summary card** (always shown)
   - Fields: Orcamento total, Total aportado, Saldo disponivel — all in BRL mono font
   - Progress bar: height 8px, rounded, track `--color-neutral-100`, fill color:
     - `--color-primary-600` when ratio < `wipBudgetAlertPct`
     - `--color-warning-500` when ratio >= `wipBudgetAlertPct` AND total <= budget
     - `--color-error-500` when total > budget
   - Budget alert banner (shown when `budgetAlert === true`): warning icon `AlertTriangle` (20px) + "Orcamento proximo do limite. Revise os aportes planejados." — background `--color-warning-100`, border `--color-warning-500`.
   - Budget exceeded banner (shown when `budgetExceeded === true`): error icon `AlertCircle` (20px) + "Orcamento ultrapassado. Total aportado supera o valor autorizado." — background `--color-error-100`, border `--color-error-500`.
   - When `wipBudget` is null: show "Sem orcamento definido" in neutral-400 text instead of progress bar.

2. **Etapas section** (shown when `stages.length > 0`, hidden otherwise)
   - Heading: "Etapas" (label size, semibold)
   - List of stage rows: name + target date + completion status chip (Pendente / Concluida)
   - Completion chip: green (`--color-success-500`) with `CheckCircle` icon when completed; neutral with `Clock` icon when pending.

3. **Aportes section**
   - Heading: "Aportes" + count badge
   - "Registrar Aporte" primary button — single primary CTA per screen (top-right of section)
   - Table/list of contributions: date | description | amount (BRL mono) | supplier (optional)
   - Table transforms to stacked cards on mobile (<768px).
   - Empty state (no contributions yet): `PlusCircle` icon (48px) + "Nenhum aporte registrado." + "Registrar Aporte" secondary link.

4. **Activation section** (fixed at bottom, separated by border-top)
   - "Ativar Ativo" button — `variant="secondary"` (not primary, since "Registrar Aporte" is already primary)
   - On click: open `ConfirmModal` with `variant="warning"`, title "Ativar imobilizado em andamento", message: "Esta acao e irreversivel. O ativo passara para status Ativo e comecara a depreciar no proximo ciclo. Total a ser capitalizado: R$ X.XXX,XX". Confirm label: "Confirmar ativacao". Cancel label: "Cancelar".
   - After confirm: POST activate endpoint → success toast "Ativo ativado com sucesso. Depreciacao iniciara no proximo ciclo." → drawer reloads asset (status now ATIVO, WIP tab disappears).
   - If `depreciationConfigMissing: true` in activation response: show additional info banner `--color-info-100` with `Info` icon: "Configure a depreciacao para que o calculo automatico funcione corretamente." + link "Configurar depreciacao" that switches drawer to tab 'depreciacao'.

**Add contribution inline form (triggered by "Registrar Aporte" button):** Opens as modal (not inline). `AssetWipContributionModal.tsx` + `.css`.

## Accessibility Contract

- All new modals: Escape key closes modal, focus trapped inside while open, focus returns to trigger element on close.
- All new tabs: `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected`, `aria-controls`.
- Budget alert banner: `role="alert"` — announced by screen readers on render.
- Budget exceeded banner: `role="alert"`.
- Depreciation config missing banner: `role="status"` (informational, not urgent).
- Activation ConfirmModal: uses existing `ConfirmModal` component which already handles `aria-modal`, `aria-labelledby`, `aria-describedby`.
- Progress bar: `role="progressbar"`, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax` set to `wipBudget` or `totalContributed` if no budget.
- Tree navigation: each node is a `<button>` with `aria-label="{assetName} — {BRL value}"`. Collapse/expand toggle: `aria-expanded`.
- Renovation radio cards: `<fieldset>` + `<legend>` wrapping the two options. Each card uses `<label>` wrapping `<input type="radio">`.
- Form labels: all visible, `htmlFor` matching input `id`. Required fields marked with `*` in label + `aria-required="true"` on input.
- Error messages: `role="alert"` on inline error spans.
- Icons decorative (accompany text): `aria-hidden="true"`. Icons functional (standalone): `aria-label` required.
