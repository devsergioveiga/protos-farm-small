**4 steps using display:none/block pattern:**

Step 1 — "Tipo de Ativo": Radio card grid (2 columns on mobile, 3 on desktop). Each card shows Lucide icon (24px) + label + description. Clicking selects and auto-populates code prefix in step 3. Cards use `border: 2px solid var(--color-neutral-200)` default, `border-color: var(--color-primary-600)` when selected.

Step 2 — "Sugestao de Codigo": Read-only preview showing:
- Code prefix in a styled badge (JetBrains Mono, 18px)
- "Exemplos:" followed by 2-3 example names in a list
- Subtitle: "Sugestao baseada no tipo selecionado"
- No user input on this step

Step 3 — "Configuracao": Form with:
- Codigo* — text input, pre-filled with `{codePrefix}-` from selected type. Label always visible (never placeholder-only). `aria-required="true"`.
- Nome* — text input, empty. Label always visible. `aria-required="true"`.
- Descricao — textarea, optional.
- Fazenda — if FarmContext has selectedFarm, show read-only farm name. If not, show farm selector dropdown.
- Validation: onBlur for required fields. Error messages with `role="alert"`.

Step 4 — "Confirmacao": Summary of all selections:
- Tipo: {label}
- Codigo: {code value}
- Nome: {name value}
- Descricao: {description or "---"}
- Fazenda: {farm name}
- Primary CTA button: "Criar Centro de Custo" (green, `--color-primary-600`)

**Navigation:**
- Header: stepper with 4 dots. Inactive: `--color-neutral-300`. Active: `--color-primary-600`. Completed: checkmark icon + `--color-primary-600`.
- Footer: "Cancelar" (left), "Voltar" (center, hidden on step 1), "Proximo" / "Criar Centro de Custo" (right).
- "Proximo" disabled if required fields not filled on current step.
- On "Criar Centro de Custo" click: POST to `/api/orgs/${orgId}/farms/${farmId}/cost-centers` with `{ code, name, description }`. Show loading state on button. On success: call `onSuccess()` then `onClose()`. On error: show error banner with `role="alert"`.

**Accessibility:**
- Modal: `aria-modal="true"`, `role="dialog"`, `aria-labelledby` pointing to wizard title.
- Focus trap inside modal (use existing focus-trap pattern or simple keydown handler).
- Escape key: if form has input (step 3), show confirm before closing. If no input yet, close directly.
- All form inputs have associated `<label>` elements.
- Radio cards: use `<label>` wrapping hidden `<input type="radio">` pattern (same as Phase 19 payment type cards).
