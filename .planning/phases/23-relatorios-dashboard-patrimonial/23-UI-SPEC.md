## Spacing Scale

Declared values (all multiples of 4). Source: `tokens.css` `--space-*` scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-1`) | Icon gaps, inline label spacing |
| sm | 8px (`--space-2`) | Compact row spacing, badge padding |
| md | 16px (`--space-4`) | Default element spacing, input padding horizontal |
| lg | 24px (`--space-6`) | Section padding, card padding, page-level gutters |
| xl | 32px (`--space-8`) | Gap between major sections (KPI row → chart row) |

- Tab bar items: `padding: 12px 16px` (`--space-3` / `--space-4`) per AssetDrawer tab pattern.

- Wizard modal header/footer: `padding: 16px 24px` (`--space-4` / `--space-6`) per WorkOrderCloseWizard pattern.
- Input fields: minimum 12px vertical, 16px horizontal padding per CLAUDE.md rule.

## Page Layout Contract

### AssetReportsPage (`/asset-reports`)

Structure:
```
<main>
  breadcrumb: Início > Patrimônio > Relatórios
  <header>
    icon + title "Relatórios Patrimoniais"
    [filter bar: farm selector, date range]
  </header>
  <nav role="tablist">
    Tab: Inventário | Tab: Depreciação | Tab: TCO
  </nav>
  <section role="tabpanel">
    [active tab content]
  </section>
</main>
```

Tab bar pattern: horizontal scrollable, `border-bottom: 2px solid transparent` default, `var(--color-primary-600)` on active. Exactly matches `AssetDrawer` tab pattern. Tab items: min-height 48px, padding `12px 16px`.

### Tab: Inventário Patrimonial

- KPI row (4 cards, 2-column grid on mobile, 4-column on ≥1024px): Valor Bruto Total, Depreciação Acumulada, Valor Líquido, Qtd Ativos
- Table: Classification / Qtd / Valor Bruto / Depr. Acumulada / Valor Líquido. Desktop table; mobile transforms to stacked cards.
- Export toolbar above table: buttons [PDF] [Excel] [CSV] aligned right.

### Tab: Depreciação e Projeção

- Horizon selector: radio group or segmented control — 12 meses / 36 meses / 60 meses. Default: 12 meses.
- `DepreciationProjectionChart`: Recharts `LineChart` with 2 series — "Depreciação Mensal Projetada" (area fill, primary-600) and "Valor Líquido Acumulado" (line only, sky-500). X axis: month/year labels. Y axis: BRL formatted with `Intl.NumberFormat`.
- Note badge: if any asset uses HOURS_OF_USE or UNITS_OF_PRODUCTION method, show info banner "Projeção estimada para ativos com método por horas/produção (dados históricos insuficientes)."
- Period filter: `dateFrom` / `dateTo` for historical view.

### Tab: TCO — Custo Total de Propriedade

- Summary cards row (3 cards): Custo Médio/Hora da Frota, Custo Total de Manutenção, Custo Total de Combustível.
- `TCOFleetView` table: Asset name / Tipo / Valor Aquisição / Depr. Acumulada / Manutenção / Combustível / TCO Total / Alerta.
- Alert column uses status badge:
  - `OK`: neutral background (`--color-neutral-100`), text `--color-neutral-600`
  - `MONITOR`: warning background (`--color-warning-100`), text `--color-warning-500`, icon `AlertTriangle` size 16px
  - `REPLACE`: error background (`--color-error-100`), text `--color-error-500`, icon `AlertCircle` size 16px
  - `NO_DATA`: muted text `--color-neutral-400`, "—"
- Fleet group view: collapsible `<details>` rows grouped by AssetType.
- Mobile: table transforms to cards with the alert badge prominent.

### CostCenterWizardModal

Pattern: `WorkOrderCloseWizard` (4 steps, `display: none/block` transitions, no CSS animation between steps per STATE.md locked decision).

| Step | Content |
|------|---------|
| 1 | Asset type selector: radio cards for MAQUINA / VEICULO / IMPLEMENTO / BENFEITORIA / TERRA with icon and label |
| 2 | Template suggestion: code prefix, 2–3 example names. Read-only preview. Subtitle: "Sugestão baseada no tipo selecionado" |
| 3 | Form: Código* (pre-filled with prefix), Nome*, Descrição (optional), Fazenda (from FarmContext or selector if null) |
| 4 | Confirmation summary + primary CTA |

Stepper in header: 4 dots/steps — inactive (`--color-neutral-300`), active (`--color-primary-600`), completed (checkmark icon, `--color-primary-600`).

## Interaction States

### Skeleton Loading

All KPI cards and charts use skeleton pulse animation (opacity 0.4→0.7, 1.5s infinite) matching `PatrimonyDashboardPage` pattern. Never a full-page spinner.

```css
@keyframes skeleton-pulse {
  0%, 100% { opacity: 0.4; }
  50% { opacity: 0.7; }
}
@media (prefers-reduced-motion: reduce) {
  .asset-reports__skeleton-* { animation: none; }
}
```

Skeleton shapes:
- KPI card skeleton: height 100px, `--color-neutral-100`, `--radius-md`
- Chart skeleton: height 300px, `--color-neutral-100`, `--radius-md`
- Table skeleton: 5 rows, each height 48px with 8px gap

### Export Button States

- Default: outlined button, border `--color-neutral-300`, text `--color-neutral-700`, icon `Download` size 16px
- Loading: button disabled, icon replaced with animated spinner (CSS border-top rotation, 200ms)
- Success: brief 1s success state not required (file download is immediate)
- Error: inline error banner below export toolbar

### Repair-vs-Replace Alert Badge

Never use color as sole indicator. Each badge state combines: background color + text label + icon (16px).

## Accessibility

- `<nav role="tablist">` with `role="tab"` + `aria-selected` on each tab button.
- `<section role="tabpanel" aria-labelledby="{tab-id}">` on each panel.
- Focus: `outline: 2px solid var(--color-primary-600); outline-offset: 2px` on `:focus-visible`. Never removed.
- Skeleton containers: `role="status" aria-label="Carregando relatório"`.
- Error banners: `role="alert" aria-live="polite"`.
- Alert badges: never rely on color alone — always include text label + icon with `aria-hidden="true"` + `aria-label` on the badge container if icon-only.
- Export buttons: `aria-label="Exportar relatório em PDF"` etc.
- Wizard: `aria-modal="true"`, focus trap inside modal, `Escape` closes to current step (not abort — confirm cancel first if form has input).
- Recharts: wrap charts in `<figure><figcaption>` with descriptive caption. Recharts default tooltips are keyboard-accessible via mouse focus; no custom keyboard override required.
- Numeric cells in tables: `<td class="...--mono">` with `tabular-nums` via JetBrains Mono. No special aria — data is in DOM.
