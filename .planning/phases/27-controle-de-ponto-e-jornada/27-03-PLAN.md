**time-entries.service.ts** — Create with these exported functions using `withRlsContext` pattern:

1. `createTimeEntry(ctx: RlsContext, orgId: string, employeeId: string, input: CreateTimeEntryInput)`:
   - Validate employee belongs to org and is ATIVO status
   - If `input.latitude` and `input.longitude` provided: run PostGIS `ST_Contains` query against `farms.boundary` where `id = input.farmId` — set `outOfRange = !result[0]?.inside ?? false`. If farm has no boundary (`boundary IS NULL`), set `noBoundary = true`, `outOfRange = false`
   - Check if month is locked: `SELECT 1 FROM timesheets WHERE employeeId = X AND referenceMonth = firstDayOfMonth(input.date) AND payrollRunId IS NOT NULL`. Return 409 if found.
   - If `input.source === 'MANAGER'`, validate `managerNote` has >= 10 chars. Return 400 if not.
   - Calculate `workedMinutes` if `clockIn` and `clockOut` both present (differenceInMinutes minus break duration).
   - Calculate `nightMinutes` by computing overlap of [clockIn, clockOut] with [21:00, 05:00] rural night window.
   - Create `TimeEntry` via `tx.timeEntry.create()`.
   - Return formatted `TimeEntryOutput` with employee name and farm name via select joins.

2. `listTimeEntries(ctx: RlsContext, orgId: string, query: TimeEntryListQuery)`:
   - Paginated with default limit=50, max 200
   - Where clause from query params: farmId, employeeId, dateFrom/dateTo range, source
   - Include employee.name, farm.name in select
   - Order by date DESC, clockIn DESC

3. `getTimeEntry(ctx: RlsContext, orgId: string, id: string)`:
   - Include activities, employee.name, farm.name

4. `updateTimeEntry(ctx: RlsContext, orgId: string, id: string, input: UpdateTimeEntryInput)`:
   - Check not LOCKED (same as create)
   - Recalculate workedMinutes and nightMinutes if times changed

5. `addActivity(ctx: RlsContext, orgId: string, timeEntryId: string, input: CreateTimeEntryActivityInput)`:
   - Look up employee's hourly rate from EmployeeSalaryHistory (latest entry) or EmployeeContract.salary / monthly hours
   - `hourlyRate = salary / (scheduledHoursPerMonth)` — use WorkSchedule to derive monthly hours
   - `costAmount = hourlyRate * (minutes / 60)`
   - Create TimeEntryActivity, link to costCenterId if provided

6. `addTeamActivity(ctx: RlsContext, orgId: string, teamId: string, input: { date: string; operationType: string; fieldOperationId?: string; fieldPlotId?: string; farmLocationId?: string; costCenterId?: string; minutes: number; notes?: string })` **(PONTO-02 modo rapido por equipe)**:
   - Fetch FieldTeam with members (where `leftAt IS NULL`) via `prisma.fieldTeam.findUnique({ where: { id: teamId }, include: { members: { where: { leftAt: null }, include: { employee: true } } } })`
   - For each member with a non-null `employeeId`: find their TimeEntry for the given date. If no TimeEntry exists for that date, skip (they didn't punch in).
   - For each matching TimeEntry: call `addActivity()` internally to create TimeEntryActivity with the same operation/plot/cost-center/minutes
   - Use team's costCenterId as fallback if individual costCenterId not provided
   - Return `{ created: number; skipped: number; details: Array<{ employeeId, employeeName, status: 'created' | 'skipped_no_entry' }> }`

**overtime-bank.service.ts:**

1. `getOvertimeBankSummary(ctx: RlsContext, orgId: string, employeeId: string)`:
   - Query all OvertimeBankEntry for employee

**timesheets.service.ts** — Create with these exported functions:

1. `createTimesheet(ctx: RlsContext, orgId: string, input: CreateTimesheetInput)`:
   - Validate employee belongs to org
   - Check unique constraint: no existing timesheet for same employeeId + referenceMonth (@@unique in schema)
   - Create with status=DRAFT, all totals=0
   - Return TimesheetOutput

2. `calculateTimesheet(ctx: RlsContext, orgId: string, timesheetId: string)`:
   - Fetch all TimeEntry for employee + referenceMonth
   - For each day: build DailyWorkInput from time entry data
   - Get WorkSchedule from employee's current contract to determine scheduledMinutes
   - Call `calcDailyWork()` for each day, `calcMonthlyTotals()` for aggregation
   - Detect inconsistencies: MISSING_CLOCK_OUT (clockOut is null), OUT_OF_RANGE, NO_BOUNDARY, INTERJORNADA_VIOLATION (calcInterjornada)
   - Update timesheet totals: totalWorked, totalOvertime50, totalOvertime100, totalNightMinutes, totalAbsences
   - Link time entries to timesheet via timesheetId
   - Create OvertimeBankEntry CREDIT if overtime50 > 0 (set expiresAt = referenceMonth + 6 months)
   - Transition status from DRAFT -> PENDING_MANAGER

3. `approveTimesheet(ctx: RlsContext, orgId: string, timesheetId: string, userId: string, input: TimesheetApprovalInput)`:
   - State machine transitions (matching RESEARCH.md Pattern 5):
     - `APPROVE_MANAGER`: PENDING_MANAGER -> MANAGER_APPROVED -> auto-transition to PENDING_RH. Set managerApprovedBy, managerApprovedAt.
     - `APPROVE_RH`: PENDING_RH -> APPROVED. Set rhApprovedBy, rhApprovedAt.
     - `REJECT`: Any approvable state -> REJECTED -> auto-transition to DRAFT. Requires justification >= 20 chars.
     - `EMPLOYEE_ACCEPT`: APPROVED -> APPROVED (set employeeAcceptedAt).
     - `EMPLOYEE_DISPUTE`: APPROVED -> APPROVED (set employeeDisputeNote). Requires justification.
   - Return 400 for invalid transition with message describing valid transitions
   - Return 400 if REJECT/EMPLOYEE_DISPUTE without justification >= 20 chars

4. `correctTimeEntry(ctx: RlsContext, orgId: string, timesheetId: string, input: TimesheetCorrectionInput)`:
   - Fetch current TimeEntry state as `beforeJson`
   - Apply corrections (only clockIn, breakStart, breakEnd, clockOut)
   - Recalculate workedMinutes and nightMinutes
   - Create TimesheetCorrection with beforeJson, afterJson, justification, correctedBy
   - Return updated TimesheetOutput

5. `getTimesheet(ctx: RlsContext, orgId: string, id: string)`:
   - Include timeEntries (ordered by date ASC), corrections, employee.name
   - Calculate inconsistencies on-the-fly from time entries data

6. `listTimesheets(ctx: RlsContext, orgId: string, query: TimesheetListQuery)`:
   - Paginated, filter by farmId (via employee.employeeFarms), employeeId, referenceMonth, status
   - Include employee.name, count of inconsistencies

7. `generateTimesheetPdf(ctx: RlsContext, orgId: string, id: string)`:
   - Fetch timesheet with all time entries and employee data
   - Use pdfkit (dynamic import pattern): `const PDFDocument = (await import('pdfkit')).default;`
   - A4 portrait, margin 50
   - Header: Organization name, "Espelho de Ponto", employee name, referenceMonth
   - Table: Date | Entrada | Int.Inicio | Int.Fim | Saida | Horas | HE 50% | HE 100% | Noturno | Obs
   - Footer: totals row, approval signatures section
   - Return `{ buffer: Buffer, filename: string }`
