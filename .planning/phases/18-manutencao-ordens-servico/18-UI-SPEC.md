## Spacing Scale

Declared values (multiples of 4px via `--space-*` tokens):

| Token | Value               | Usage                                             |
| ----- | ------------------- | ------------------------------------------------- |
| xs    | 4px (`--space-1`)   | Icon gaps, inline padding, badge internal padding |
| sm    | 8px (`--space-2`)   | Compact element spacing, input addon gaps         |
| md    | 16px (`--space-4`)  | Default element spacing, form field gaps          |
| lg    | 24px (`--space-6`)  | Section padding, card internal padding            |
| xl    | 32px (`--space-8`)  | Layout gaps between sections                      |
| 2xl   | 48px (`--space-12`) | Major section breaks, page top padding            |
| 3xl   | 64px (`--space-16`) | Empty state icon size, page-level spacing         |

Exceptions:

- Touch targets: minimum 48x48px for all interactive elements (CLAUDE.md requirement — applies to kanban cards' drag handles, close wizard buttons, maintenance plan action buttons)
- Input padding: minimum 12px vertical (`--space-3`), 16px horizontal (`--space-4`) per CLAUDE.md
- Kanban card internal padding: 16px all sides (`--space-4`) — matches existing `KanbanCard.css` pattern

**Source:** `apps/frontend/src/styles/tokens.css` (pre-existing scale), `CLAUDE.md` (touch target rule)

## Typography

| Role                         | Size                 | Weight                  | Line Height              | Font          | Token            |
| ---------------------------- | -------------------- | ----------------------- | ------------------------ | ------------- | ---------------- |
| Body                         | 16px (`--text-base`) | 400 (`--font-regular`)  | 1.5 (`--leading-normal`) | Source Sans 3 | `--font-body`    |
| Label / UI text              | 14px (`--text-sm`)   | 600 (`--font-semibold`) | 1.5 (`--leading-normal`) | Source Sans 3 | `--font-body`    |
| Heading (page, modal title)  | 20px (`--text-lg`)   | 700 (`--font-bold`)     | 1.2 (`--leading-tight`)  | DM Sans       | `--font-display` |

- Font: `JetBrains Mono` (`--font-mono`), 14px, weight 400

- Minimum 12px — nothing smaller anywhere in this phase
- ALL CAPS only for short status labels (e.g., "ABERTA", "EM ANDAMENTO") — never in sentences
- Never center-align text blocks longer than 2 lines

## Screen-by-Screen Interaction Contract

### MaintenancePlansPage

**Layout:** Full-width page with header breadcrumb, filter bar, data table below.

**Header:**

- Breadcrumb: "Patrimônio / Planos de Manutenção"
- Primary CTA (1 per page): button with `Plus` icon (20px) + label "Novo Plano" → opens `MaintenancePlanModal`
- Page title: "Planos de Manutenção" at `--text-lg` / `--font-bold` / DM Sans

**Filter bar:**

- Filters: ativo/inativo toggle, tipo de gatilho (Horímetro / Km / Calendário), fazenda selector
- All filters inline, horizontal scroll on mobile

**Table columns:** Nome do plano | Ativo | Gatilho | Intervalo | Próxima execução | Último executado | Ações

- "Próxima execução" column: if overdue (past date), show date in `--color-error-500` with `AlertCircle` (16px) icon + `aria-label="Vencido"`
- "Ações" column: "Ver OS" link + edit icon button + toggle ativo/inativo

**Empty state:** `Wrench` icon 48px `--color-neutral-400` + "Nenhum plano de manutenção ainda." + "Crie o primeiro plano preventivo para este ativo." + "Novo Plano" CTA button

### WorkOrdersPage

**Layout:** Full-width page with header, tabs (Lista / Kanban), filter bar, content area.

**Header:**

- Breadcrumb: "Patrimônio / Ordens de Serviço"
- Primary CTA: "Abrir OS" button with `Plus` icon

**Tab: Lista** — standard table

- Columns: OS # | Tipo | Ativo | Status badge | Responsável | Custo total | Abertura | Ações
- OS # displayed in `JetBrains Mono` 14px
- Status badges: colored pill per status color mapping above, always icon + text

**Tab: Kanban** — wraps `MaintenanceKanban`

- 3 columns: ABERTA | EM ANDAMENTO | AGUARDANDO PEÇA
- Each card: OS number (mono), asset name, OS type badge, age in days
- Column count badge on column header

**Empty state (Lista tab):** `ClipboardList` icon 48px + "Nenhuma OS aberta." + "Abra a primeira ordem de serviço." + "Abrir OS"

### MaintenanceDashboardPage

**Layout:** KPI row (4 cards) + section "OS Abertas" (kanban) + section "Custo por Ativo" (table) + section "Alertas" (list)

**KPI cards (4):**

1. Disponibilidade Mecânica — percentage, large `--text-xl` DM Sans 500
2. MTBF — "X horas" in JetBrains Mono, or "N/D" with tooltip when no corrective OS
3. MTTR — "X horas" in JetBrains Mono, or "N/D"
4. Custo YTD — BRL currency in JetBrains Mono

**Overdue alerts section:** `AlertTriangle` icon 20px `--color-error-500` per item, list of plans past due date.

**Source:** REQUIREMENTS.md MANU-05, RESEARCH.md Pattern 5

### WorkOrderModal (create/edit OS)

**Pattern:** Modal (max-width 720px), header / scrollable body / sticky footer.

**Sections in body:**

1. Ativo vinculado (required selector — searchable dropdown)
2. Tipo: PREVENTIVA / CORRETIVA / SOLICITAÇÃO (radio group, always visible)
3. Título + Descrição (text inputs)
4. Peças consumidas (inline table with `+` row button — product selector + qty + unit cost)
5. Mão de obra (hours input + cost/hour input → auto-computed total displayed in mono)
6. Custo externo (BRL input + fornecedor text)
7. Fotos (file upload area, max 5 photos, thumbnail previews 80x80px)
8. Centro de custo (inherited from asset by default, override allowed)

**Footer:** "Salvar" primary button + "Cancelar" secondary button

**Cost summary row** (bottom of body, above footer): "Total estimado: R$ X.XXX,XX" in JetBrains Mono 16px, updates live as parts/labor/external cost change.

### WorkOrderCloseWizard (3-step modal)

**Trigger:** "Encerrar OS" button on `WorkOrderModal` or from OS detail. Fires for ALL OS regardless of cost — backend enforces `accountingTreatment` requirement.

**Step 1 — Resumo de Custos:**

- Read-only breakdown: Peças + Mão de obra + Custo externo = Total
- All values in JetBrains Mono
- Below threshold (< R$ 1.000,00): info banner "Classificação automática: Despesa imediata" with option to override
- CTA: "Continuar" → Step 2

**Step 2 — Classificação Contábil:**

- 3 radio card options, each with label + 2-line description (Source Sans 3 14px):
  - "Despesa imediata" — neutral card border
  - "Capitalização" — `--color-sky-100` card background when selected
  - "Diferimento" — `--color-earth-100` card background when selected
- If DIFERIMENTO selected: numeric input "Distribuir em X meses" appears inline (display:none → display:block, no animation per CLAUDE.md)
- CTA: "Confirmar" → Step 3

**Step 3 — Confirmar Encerramento:**

- Summary: OS number, asset, classification chosen, total cost
- Warning banner (if CAPITALIZACAO): "O valor contábil do ativo será atualizado. Revise a configuração de depreciação." — `--color-warning-100` background, `--color-warning-500` border-left 4px, `AlertTriangle` 20px icon
- CTA (destructive-adjacent): "Encerrar OS" button — `--color-primary-600` (standard primary, not red — closing is not destructive)
- Secondary: "Voltar"

**Stepper header:** 3 dots (Step 1 / Step 2 / Step 3), active dot `--color-primary-600`, completed dot `--color-success-500`, inactive dot `--color-neutral-300`

**Source:** REQUIREMENTS.md MANU-06, RESEARCH.md Pattern code + Frontend WorkOrder Close Wizard, STATE.md locked decision

### MaintenancePlanModal

**Pattern:** Single-step modal, max-width 560px.

**Fields:**

1. Nome do plano (required text)
2. Ativo vinculado (required searchable selector — pre-filled when opened from AssetMaintenanceTab)
3. Tipo de gatilho: Horímetro / Km / Calendário (radio group)
4. Intervalo (numeric input, label changes per trigger: "A cada X horas" / "A cada X km" / "A cada X dias")
5. Alertar com antecedência de (numeric input, same unit as interval)
6. Descrição (optional textarea)

**Footer:** "Salvar Plano" primary + "Cancelar"

### AssetMaintenanceTab (replace stub)

**Content (within AssetDrawer tab panel):**

- "Nova OS" button (`--color-primary-600`, secondary variant to not compete with page-level primary) + "Novo Plano" secondary button — top right of tab
- "Planos de Manutenção" sub-section: compact table (Nome | Gatilho | Próxima execução | Status)
- "Histórico de OS" sub-section: list of last 10 work orders (OS # in mono + status badge + custo + data de abertura)
- Empty when no plans: single empty state "Nenhum plano cadastrado. Configure o primeiro plano preventivo." with small "Novo Plano" link-style button

## Interaction States

### Status Badge Interaction

Each status badge in lists and kanban cards must include:

- Colored pill with `role="status"` (or within a `<td>` with appropriate header context)
- Icon left of text: `Circle` (filled) for ABERTA, `Loader2` for EM_ANDAMENTO, `PackageX` for AGUARDANDO_PECA, `CheckCircle2` for ENCERRADA, `XCircle` for CANCELADA
- Icon size: 14px, `aria-hidden="true"` (status text provides the accessible label)

### Form Validation

- Inline validation on blur per field (not on submit)
- Required fields marked with `*` in label
- Error messages with `role="alert"` + `AlertCircle` 16px icon left of text
- Cost fields (decimal inputs): validate Brazilian decimal format (comma as decimal separator) and display formatted value on blur

### Kanban Drag-and-Drop

- Reuse `@dnd-kit/core` + `KanbanBoard` / `KanbanColumn` / `KanbanCard` components unchanged
- Valid drop targets highlighted with `--color-primary-100` background + 2px `--color-primary-500` border
- Invalid drop target (ENCERRADA/CANCELADA are not kanban columns — no drop target): not displayed in kanban
- Drag overlay: card with 4° rotation + `box-shadow: var(--shadow-xl)`
- After drop: optimistic update + API call; on error: revert card + error toast

### Photo Upload Area

- Drag-and-drop zone + click to browse
- Border: 2px dashed `--color-neutral-300`; hover/drag-over: 2px dashed `--color-primary-500` background `--color-primary-50` (= `--color-neutral-50` tinted — use `--color-primary-50` which maps to `#E8F5E9`)
- Thumbnails: 80x80px grid, remove button (×) top-right of thumbnail, `aria-label="Remover foto [N]"`
- Max 5 photos, counter displayed "X/5 fotos"
- File type validation: only JPEG/PNG/WEBP; error "Apenas imagens JPG, PNG ou WEBP são aceitas."

### WorkOrderCloseWizard — Accounting Treatment Radio Cards

- Unselected: white background, `--color-neutral-200` border, `--radius-md` border-radius
- Selected: treatment-specific background (see Color section), `--color-primary-600` border 2px, `--radius-md`
- Hover (unselected): `--color-neutral-100` background
- Focus: 2px `--color-primary-500` outline per accessibility rules
- Radio input visually hidden; entire card is the click target with `cursor: pointer`

## Accessibility Contract

All components in Phase 18 must satisfy:

| Rule              | Requirement                                                                                                    | Source         |
| ----------------- | -------------------------------------------------------------------------------------------------------------- | -------------- |
| Touch targets     | 48x48px minimum for all buttons, drag handles, close buttons                                                   | CLAUDE.md      |
| Focus visible     | `outline: 2px solid var(--color-primary-500); outline-offset: 2px` — never removed                             | CLAUDE.md      |
| Form labels       | `<label htmlFor>` for every input — placeholder is supplementary only                                          | CLAUDE.md      |
| Error messages    | `role="alert"` + icon + text — never color alone                                                               | CLAUDE.md      |
| Icon-only buttons | `aria-label` mandatory (e.g., edit plan icon, delete icon, close modal X)                                      | CLAUDE.md      |
| Modal focus trap  | Focus must stay inside open modal; Escape closes; focus returns to trigger on close                            | CLAUDE.md      |
| Status badges     | Color + icon + text — never color alone (WCAG 1.4.1)                                                           | CLAUDE.md      |
| Kanban columns    | `aria-label="Coluna [status]: X itens"` per existing `KanbanColumn` pattern                                    | codebase       |
| Kanban cards      | `aria-label` on `<article>` describing OS content (OS #, asset name, age)                                      | codebase       |
| Data tables       | `<th scope="col">` + `<caption>` for all tables                                                                | CLAUDE.md      |
| Reduced motion    | All transitions must respect `prefers-reduced-motion: reduce` via existing tokens.css rule                     | tokens.css     |
| WCAG AA contrast  | `--color-primary-600` on white: verified 7.2:1 — passes; `--color-error-500` on white: verified 5.4:1 — passes | CLAUDE.md spec |
