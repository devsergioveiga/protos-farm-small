**work-orders.service.ts:**

Import Decimal from 'decimal.js'. Set `Decimal.set({ rounding: Decimal.ROUND_HALF_UP })` at module top.

1. `getNextSequentialNumber(tx, organizationId)`:
   - Follow purchase-requests.service.ts pattern exactly
   - `findFirst({ where: { organizationId }, orderBy: { sequentialNumber: 'desc' }, select: { sequentialNumber: true } })`
   - Return `(last?.sequentialNumber ?? 0) + 1`

2. `createWorkOrder(ctx: RlsContext, input: CreateWorkOrderInput)`:
   - Use `prisma.$transaction` wrapping ALL operations:
     a. Verify asset belongs to org
     b. Get sequential number via `getNextSequentialNumber(tx, ctx.organizationId)`
     c. Create WorkOrder record with status ABERTA, sequentialNumber, all input fields
     d. **Atomically** update asset status to EM_MANUTENCAO: `tx.asset.update({ where: { id: input.assetId }, data: { status: 'EM_MANUTENCAO' } })`
   - Return work order with includes

3. `listWorkOrders(ctx: RlsContext, query: ListWorkOrdersQuery)`:
   - Filter by org, optional status/assetId/type/farmId (via asset.farmId)/dateRange
   - Include: asset { id, name, assetTag }, parts count
   - Paginate, return { data, total, page, limit }

4. `getWorkOrder(ctx: RlsContext, id: string)`:
   - Include parts (with product { name, unit }), ccItems, asset, maintenancePlan
   - Throw WorkOrderError('OS nao encontrada', 404)

5. `updateWorkOrder(ctx: RlsContext, id: string, input: UpdateWorkOrderInput)`:
   - Only allow status changes: ABERTA -> EM_ANDAMENTO, ABERTA -> AGUARDANDO_PECA, EM_ANDAMENTO -> AGUARDANDO_PECA, AGUARDANDO_PECA -> EM_ANDAMENTO
   - Throw WorkOrderError('Transicao de status invalida', 400) for invalid transitions
   - If status changes to EM_ANDAMENTO and startedAt is null, set startedAt = now()

6. `addWorkOrderPart(ctx: RlsContext, workOrderId: string, input: AddWorkOrderPartInput)`:
   - Verify OS belongs to org and status is not ENCERRADA/CANCELADA
   - Create WorkOrderPart with `totalCost = new Decimal(input.quantity).times(input.unitCost).toDecimalPlaces(2)`
   - Recalculate totalPartsCost on WorkOrder: `sum(parts.totalCost)`

7. `removeWorkOrderPart(ctx: RlsContext, workOrderId: string, partId: string)`:
   - Delete part, recalculate totalPartsCost

8. `closeWorkOrder(ctx: RlsContext, id: string, input: CloseWorkOrderInput)`:
   **This is the critical function. Follow RESEARCH.md Pattern 2 exactly:**

   ```
   if (!input.accountingTreatment) {
     throw new WorkOrderError('Classificacao contabil obrigatoria para encerrar OS', 400);
   }
   if (input.accountingTreatment === 'DIFERIMENTO' && !input.deferralMonths) {
     throw new WorkOrderError('Numero de meses obrigatorio para diferimento', 400);
   }
   ```

   Inside `prisma.$transaction`:
   a. Fetch WO with parts and asset (with costCenter)
   b. Verify status not ENCERRADA/CANCELADA
   c. **Stock deduction**: if parts.length > 0, call `createConsumptionOutput(tx, { organizationId, items: parts.map(p => ({ productId: p.productId, quantity: Number(p.quantity) })), fieldOperationRef: 'work-order:${id}', outputDate: new Date(), responsibleName: input.closedBy, notes: 'OS #${wo.sequentialNumber} — ${wo.title}' })`
   d. **Compute costs**: partsCost = sum(parts.totalCost), laborCost = laborHours \* laborCostPerHour (or 0), externalCost (or 0), totalCost = partsCost + laborCost + externalCost — ALL using Decimal
   e. **CAPITALIZACAO**: `tx.asset.update({ where: { id: wo.assetId }, data: { acquisitionValue: { increment: totalCost.toNumber() } } })`
   f. **DIFERIMENTO**: compute monthly = totalCost / deferralMonths (Decimal), compute endMonth/endYear from startMonth + deferralMonths, create DeferredMaintenance record
   g. **CC appropriation**: costCenterId = input.costCenterId ?? wo.asset.costCenterId. If present, create WorkOrderCCItem with amount=totalCost, percentage=100
   h. **Reset asset**: `tx.asset.update({ where: { id: wo.assetId }, data: { status: 'ATIVO' } })`
   i. **Update linked maintenance plan**: if wo.maintenancePlanId, update plan's lastExecutedAt = now(), lastMeterValue = hourmeterAtOpen (or odometerAtOpen), recalculate nextDue using computeNextDue from maintenance-plans.service.ts
   j. Update WorkOrder with all computed values, status ENCERRADA, closedAt

9. `cancelWorkOrder(ctx: RlsContext, id: string)`:
   - In transaction: set status CANCELADA, reset asset status to ATIVO

10. `getMaintenanceDashboard(ctx: RlsContext, query: { year?: number })`:
    - **MTBF**: total operating hours / count of CORRETIVA OS (only CORRETIVA type counts as failure). Return null if count is 0.
    - **MTTR**: sum of (closedAt - openedAt) in hours for CORRETIVA OS / count. Return null if count is 0.
    - **Availability**: uptime / (uptime + downtime) \* 100. Downtime = sum of (closedAt - openedAt) for all OS. Uptime = total hours in period - downtime.
    - **totalCostYTD**: sum of totalCost for ENCERRADA OS in current year
    - **openOrdersCount**: count where status in (ABERTA, EM_ANDAMENTO, AGUARDANDO_PECA)
    - **overdueMaintenancesCount**: count of active MaintenancePlans where nextDueAt < now
    - **byStatus**: groupBy status, count
    - **costByAsset**: groupBy assetId, sum totalCost, top 10
    - **recentOrders**: last 10 OS ordered by openedAt desc
