**useMaintenancePlans.ts:**
Follow useAssets.ts pattern:

- State: plans, loading, error, total, page
- `fetchPlans(query)` — GET `/api/org/maintenance-plans` with query params
- `createPlan(input)` — POST, toast "Plano de manutencao criado com sucesso.", call onSuccess
- `updatePlan(id, input)` — PUT, toast "Plano atualizado."
- `deletePlan(id)` — DELETE, toast "Plano excluido."
- `toggleActive(id, isActive)` — PUT { isActive }
- Return { plans, loading, error, total, page, setPage, fetchPlans, createPlan, updatePlan, deletePlan, toggleActive }

**useWorkOrders.ts:**
Follow useAssets.ts pattern:

- State: workOrders, loading, error, total, page
- `fetchWorkOrders(query)` — GET `/api/org/work-orders`
- `createWorkOrder(input)` — POST, toast "OS aberta. [Nome do ativo] em manutencao."
- `updateWorkOrder(id, input)` — PATCH
- `addPart(workOrderId, input)` — POST `/api/org/work-orders/${id}/parts`
- `removePart(workOrderId, partId)` — DELETE
- `closeWorkOrder(id, input)` — PATCH `/api/org/work-orders/${id}/close`, toast "OS #NNNN encerrada. [Nome do ativo] voltou ao status Ativo."
- `cancelWorkOrder(id)` — PATCH `/api/org/work-orders/${id}/cancel`
- `fetchDashboard()` — GET `/api/org/work-orders/dashboard`
- Return all state and handlers
