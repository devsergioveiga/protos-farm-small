Create `goods-receipts.service.ts` with these exported functions:

**1. `getNextGrSequentialNumber(tx, organizationId)`** — Query last REC-YYYY/NNNN, increment, pad to 4 digits. Pattern identical to PO sequential number.

**2. `createGoodsReceipt(ctx: RlsContext & { userId: string }, input: CreateGoodsReceiptInput)`** — Inside withRlsContext:

- If receivingType === 'EMERGENCIAL': require emergencyJustification, purchaseOrderId must be null
- If purchaseOrderId provided: verify PO exists, belongs to org, is in status CONFIRMADA or EM_TRANSITO
- Generate sequential number REC-YYYY/NNNN
- For each item: calculate totalPrice = receivedQty \* unitPrice; calculate divergencePct = Math.abs(receivedQty - orderedQty) / orderedQty (guard orderedQty === 0); set hasDivergence = divergencePct > 0.05
- Set isProvisional = true if receivingType === 'NF_ANTECIPADA'
- Create GoodsReceipt with items and divergences in single create (nested writes)
- Return formatted GoodsReceiptOutput

**3. `listGoodsReceipts(ctx, query: ListGoodsReceiptsQuery)`** — Paginated list with filters:

- Filter by status, receivingType, supplierId, purchaseOrderId
- Search by sequentialNumber or supplier name (contains, insensitive)
- Include supplier (select id, name, tradeName), purchaseOrder (select sequentialNumber), \_count (items, divergences)
- Exclude deletedAt != null
- Order by createdAt desc
- Return ListGoodsReceiptsResult with formatted items

**4. `getGoodsReceiptById(ctx, id)`** — Full detail with items, divergences, supplier, purchaseOrder, creator. Return GoodsReceiptOutput.

**5. `transitionGoodsReceipt(ctx: RlsContext & { userId: string }, id: string, input: TransitionGrInput)`** — Inside withRlsContext:

- Load current GR, validate canGrTransition(current.status, input.status)
- If transitioning to EM_CONFERENCIA: set receivedAt = now()
- If transitioning to CONFERIDO: set conferredAt = now()
- If transitioning to REJEITADO: require rejectionReason, set rejectedAt = now()
- NOTE: CONFIRMADO transition is NOT handled here — Plan 03 adds confirmGoodsReceipt
- Update status + timestamp
- Return updated GoodsReceiptOutput

**6. `listPendingDeliveries(ctx)`** — Query PurchaseOrders in status CONFIRMADA or EM_TRANSITO where not all items are fully received:

- For each PO: count items, count items where receivedQuantity < quantity
- Include supplier, expectedDeliveryDate
- Calculate isOverdue = expectedDeliveryDate < now()
- Return PendingDelivery[]

**7. `updateGoodsReceiptDivergencePhoto(ctx, goodsReceiptId, divergenceId, photoUrl, photoFileName)`** — Update divergence record with photo path.

Helper function `formatGoodsReceipt(gr)` and `formatGoodsReceiptListItem(gr)` to map Prisma Decimal fields to numbers and add label strings.

Use `Number()` cast for all Decimal fields (per Phase 8 pattern with Prisma Decimal).
