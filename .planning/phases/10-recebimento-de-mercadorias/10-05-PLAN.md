**GoodsReceiptModal.tsx** — 4-step wizard modal.

Props: `{ isOpen: boolean; onClose: () => void; onSuccess: () => void; preselectedPurchaseOrderId?: string }`

State:

- `step`: 1 | 2 | 3 | 4
- `formData`: Partial&lt;CreateGoodsReceiptInput&gt; accumulated across steps
- `selectedPO`: PurchaseOrder | null (loaded in step 1)
- `isSubmitting`: boolean
- `showConfirmModal`: boolean (for step 4 confirm action)

**Modal structure:**

- Header: step indicator (4 circles with labels: "1. Pedido", "2. Nota Fiscal", "3. Conferencia", "4. Resumo"), active step highlighted with primary color
- Body: scrollable content area, renders current step component
- Footer: "Voltar" button (disabled on step 1), "Proximo" button (steps 1-3), "Registrar Recebimento" button (step 4, primary)

**Step 1 — Selecionar Pedido:**

- Search input to find POs by number (api.get('/org/purchase-orders?search=X&status=CONFIRMADA'))
- List of matching POs as selectable cards (sequentialNumber, supplier.name, item count, expectedDeliveryDate)
- Selected PO highlighted with border
- Toggle: "Recebimento emergencial (sem pedido)" — shows supplierId select + emergencyJustification textarea + receivingType=EMERGENCIAL
- receivingType select: dropdown with 6 options (only show when PO is selected, default STANDARD)
- If preselectedPurchaseOrderId: auto-load that PO
- On "Proximo": if PO selected, load PO details (items), store purchaseOrderId + supplierId + items mapping in formData

**Step 2 — Dados da Nota Fiscal:**

- Fields (all in one section, max-width 800px):
  - invoiceNumber (text input, label "Numero da NF", required for STANDARD/PARCIAL/NF_FRACIONADA)
  - invoiceSerie (text input, label "Serie", optional)
  - invoiceCfop (text input, label "CFOP", optional)
  - invoiceDate (date input, label "Data de Emissao", required when invoiceNumber filled)
  - invoiceTotal (number input, label "Valor Total da NF (R$)", required when invoiceNumber filled, step=0.01)
  - invoiceKey (text input, label "Chave de Acesso", optional, maxLength=44, pattern hint "44 digitos")
  - storageFarmId (select from farms, label "Fazenda de Armazenamento")
  - notes (textarea, label "Observacoes", optional)
- For MERCADORIA_ANTECIPADA: show info banner "Mercadoria sem NF — entrada provisoria no estoque. NF sera registrada posteriormente."
- For NF_ANTECIPADA: show info banner "NF registrada antecipadamente. Aguardando chegada da mercadoria."
- Validation: invoiceKey length must be exactly 44 if provided; invoiceTotal must be > 0 if provided

**Step 3 — Conferencia de Itens:**

- Table with columns: Produto (productName), Unidade (unitName), Qty Pedida (orderedQty, readonly), Qty NF (invoiceQty, editable number input), Qty Recebida (receivedQty, editable number input), Preco Unit (unitPrice, readonly), Visual OK (checkbox), Lote (text input), Validade (date input), Obs (text input)
- Auto-calculate per row:
  - totalPrice = receivedQty \* unitPrice
  - divergencePct = orderedQty > 0 ? Math.abs(receivedQty - orderedQty) / orderedQty \* 100 : 0
  - hasDivergence = divergencePct > 5
- If hasDivergence: show yellow AlertTriangle icon + badge with divergencePct formatted as "X.X%"
- If ANY item has divergence: show yellow banner at top "Atencion: X item(ns) com divergencia > 5%"
- Divergence registration row (expandable): when item has divergence, show expandable section below with:
  - divergenceType select (5 options from DIVERGENCE_TYPE_LABELS)
  - action select (3 options from DIVERGENCE_ACTION_LABELS)
  - observation textarea
  - (Photo upload deferred to after creation — upload endpoint exists but wizard creates first, then user can add photos)
- For EMERGENCIAL (no PO items): show "Adicionar item" button to add free-text rows (productName, unitName, receivedQty, unitPrice inputs)
- Table scrolls horizontally on mobile (overflow-x: auto)

**Step 4 — Resumo e Confirmacao:**

- Readonly summary of all data:
  - Pedido: PO sequential number or "Emergencial"
  - Fornecedor: supplier name
  - Tipo: receivingType label
  - NF: invoiceNumber, invoiceSerie, invoiceDate, invoiceTotal (formatted BRL)
  - Items summary table: productName, orderedQty, receivedQty, totalPrice
  - Divergences summary: count + list
  - Total value: sum of item totalPrices
- Warnings section:
  - If MERCADORIA_ANTECIPADA: "Entrada no estoque sera criada em modo RASCUNHO (bloqueada para consumo ate NF chegar)"
  - If NF_ANTECIPADA: "Conta a pagar sera criada. Estoque sera atualizado quando mercadoria chegar."
  - If divergences exist: "X divergencia(s) registrada(s). Revise antes de confirmar."
- "Registrar Recebimento" primary button: calls createGoodsReceiptApi(formData) -> on success, automatically transitions to EM_CONFERENCIA -> CONFERIDO -> shows toast "Recebimento REC-XXXX registrado" + asks "Deseja confirmar agora?" via ConfirmModal
- If user confirms: calls confirmGoodsReceiptApi(id) -> toast "Recebimento confirmado. Estoque e conta a pagar gerados." + onSuccess() + onClose()
- If user skips: toast "Recebimento registrado como conferido." + onSuccess() + onClose()

**Key-remount pattern:** Outer wrapper manages isOpen/step/formData. If the modal is reopened, reset all state via key prop.

**Detail view mode:** When GoodsReceiptModal receives an existing receipt ID (via `existingId` prop), switch to read-only detail mode:

**Import GoodsReceiptModal** from '@/components/goods-receipts/GoodsReceiptModal'.

**State additions:**

- `showCreateModal: boolean` (for new receipt wizard)
- `selectedReceiptId: string | null` (for detail view)
- `preselectedPOId: string | null` (for "Registrar Recebimento" from Pendencias tab)

**Wiring:**

1. "Novo Recebimento" button in header: `onClick={() => setShowCreateModal(true)}`
2. Eye icon action in Recebimentos table: `onClick={() => setSelectedReceiptId(item.id)}`
3. "Registrar Recebimento" button in Pendencias tab: `onClick={() => { setPreselectedPOId(delivery.purchaseOrderId); setShowCreateModal(true); }}`

**Modal rendering (at bottom of component, before closing tags):**

```tsx
{
  showCreateModal && (
    <GoodsReceiptModal
      isOpen={showCreateModal}
      onClose={() => {
        setShowCreateModal(false);
        setPreselectedPOId(null);
      }}
      onSuccess={() => {
        refetch();
        setShowCreateModal(false);
        setPreselectedPOId(null);
      }}
      preselectedPurchaseOrderId={preselectedPOId ?? undefined}
    />
  );
}
{
  selectedReceiptId && (
    <GoodsReceiptModal
      isOpen={!!selectedReceiptId}
      onClose={() => setSelectedReceiptId(null)}
      onSuccess={() => {
        refetch();
        setSelectedReceiptId(null);
      }}
      existingId={selectedReceiptId}
    />
  );
}
```
