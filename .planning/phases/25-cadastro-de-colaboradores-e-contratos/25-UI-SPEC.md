## Spacing Scale

Declared values (multiples of 4px — `--space-*` tokens from `tokens.css`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-1`) | Icon gaps, badge padding, inline separators |
| sm | 8px (`--space-2`) | Compact row spacing, gap entre chips de status |
| md | 16px (`--space-4`) | Default element spacing, input internal padding |
| lg | 24px (`--space-6`) | Section padding, gap entre cards |
| xl | 32px (`--space-8`) | Layout gaps, tab panel padding |
| 2xl | 48px (`--space-12`) | Major section breaks no EmployeeDetailPage |
| 3xl | 64px (`--space-16`) | Empty state icon margin |

Exceptions:
- Touch targets mínimos: 48x48px — botões de ação da ficha (editar, mudar status)
- Input padding: 12px vertical (`--space-3`) × 16px horizontal (`--space-4`) — obrigatório por CLAUDE.md
- Avatar do colaborador no cabeçalho da ficha: 80x80px (não escala de 4px, herda padrão AnimalDetailPage)

## Typography

| Role | Family | Size | Weight | Line Height |
|------|--------|------|--------|-------------|
| Body | Source Sans 3 (`--font-body`) | 16px (`--text-base`) | 400 (`--font-regular`) | 1.5 (`--leading-normal`) |
| Label / UI | Source Sans 3 (`--font-body`) | 14px (`--text-sm`) | 700 (`--font-bold`) | 1.3 (`--leading-snug`) |
| Heading (página/seção) | DM Sans (`--font-display`) | 20px (`--text-lg`) | 700 (`--font-bold`) | 1.2 (`--leading-tight`) |
| Display (nome na ficha) | DM Sans (`--font-display`) | 24px (`--text-xl`) | 700 (`--font-bold`) | 1.2 (`--leading-tight`) |
| Dados monetários / CPF / PIS | JetBrains Mono (`--font-mono`) | 14px (`--text-sm`) | 400 (`--font-regular`) | 1.5 (`--leading-normal`) |

Weights in use: **400** (`--font-regular`) and **700** (`--font-bold`) only. Weights 500 (`--font-medium`) and 600 (`--font-semibold`) are not used in this phase.

Rules:
- Mínimo absoluto: 14px — nunca abaixo (legibilidade ao ar livre)
- ALL CAPS apenas em labels curtas: "STATUS", "CPF", "CBO", "CLT" — nunca em frases
- Nunca centralizar blocos de texto com mais de 2 linhas
- Valores salariais, CPF, PIS/PASEP, CTPS sempre em `--font-mono`

> Source: `tokens.css` (linhas 72–93) + CLAUDE.md tipografia

## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-neutral-50` | `#FAFAF8` | Fundo de página, fundo de modal |
| Secondary (30%) | `--color-neutral-100` | `#F5F3EF` | Cards de colaborador, sidebar, fundo de tabs, linhas alternadas de tabela |
| Accent (10%) | `--color-primary-600` | `#2E7D32` | Ver lista abaixo |
| Destructive | `--color-error-500` | `#C62828` | Ação "Desligar colaborador", remoção de documento, cancelamento de contrato |
| Warning | `--color-warning-500` | `#F57F17` | Alerta de vencimento de contrato (experiência, safra), badge AFASTADO |
| Info | `--color-info-500` | `#0277BD` | Badge de tipo de contrato (informativo), tooltip do gráfico |

Accent (`#2E7D32`) reserved for:
1. Botão primário "Cadastrar colaborador" (máximo 1 por tela)
2. Botão primário "Salvar" dentro de modals de criação/edição
3. Tab ativa no EmployeeDetailPage (underline indicator)
4. Link "Ver ficha completa" na listagem de colaboradores
5. Status badge ATIVO (`--color-success-500` = `#2E7D32` — mesmo valor)

Color rules:
- Nunca usar cor como único indicador de estado — sempre acompanhar com ícone Lucide + texto
- Status badges: ATIVO = verde + ícone `CheckCircle`, AFASTADO = amarelo + ícone `Clock`, FÉRIAS = azul + ícone `Umbrella`, DESLIGADO = cinza + ícone `XCircle`
- Erros de validação de CPF: `--color-error-500` + ícone `AlertCircle` + mensagem textual
- PIS/PASEP warning (permite salvar): `--color-warning-500` + ícone `AlertTriangle` + mensagem textual

> Source: CLAUDE.md cores + tokens.css semânticas + CONTEXT.md D-03 (state machine de status)

## Page-Level Interaction Contracts

### EmployeesPage (listagem)

- Filtros: status (multiselect chips), tipo de contrato (dropdown), fazenda (dropdown se multi-fazenda), cargo (autocomplete)
- Busca: full-text por nome, CPF (máscara automática ao digitar), matrícula
- Ordenação: nome A-Z (padrão), data admissão, cargo
- Tabela <768px: transformar em cards empilhados com nome, cargo, status badge e avatar
- Skeleton: 8 linhas de skeleton ao carregar, nunca spinner full-page
- Paginação: 20 por página, cursor-based

### EmployeeDetailPage (ficha)

- Cabeçalho fixo: avatar 80x80px (placeholder `UserRound` 48px se sem foto), nome em Display, cargo atual em Label, status badge, botões Editar e Mudar Status
- Tabs: Dados Pessoais | Contrato | Evolução | Documentos | Histórico
- Tab ativa: underline 2px `--color-primary-600`, não fundo colorido
- Breadcrumb: Colaboradores > [Nome do colaborador]
- Back button: `← Colaboradores` no canto superior esquerdo em mobile

### CreateEmployeeModal (multi-step)

- Step 1 — Dados Pessoais: nome completo, CPF (validação ao sair do campo — onBlur), data nascimento, sexo, PIS/PASEP (warning se inválido), RG, CTPS
- Step 2 — Contrato: tipo de contrato (radio cards com ícone), data admissão, cargo (autocomplete), salário, jornada
- Step 3 — Dados Bancários e Saúde: banco, agência, conta, tipo sanguíneo, plano de saúde
- Step 4 — Revisão: cards de resumo, botão "Cadastrar colaborador"
- Stepper no header do modal com número + label do step
- Campos obrigatórios marcados com `*`
- Validação inline onBlur, não apenas no submit

### EmployeeBulkImportModal (upload → preview → confirmar)

- Step 1 — Upload: drag-and-drop + botão alternativo, aceita .xlsx/.csv, template Excel downloadável
- Step 2 — Mapeamento: igual ao `AnimalBulkImportModal` — tabela de colunas detectadas vs campos esperados
- Step 3 — Preview: tabela com erros inline (CPF inválido = linha vermelha + ícone `AlertCircle`, PIS inválido = linha amarela + ícone `AlertTriangle`)
- Step 4 — Relatório: contagem criados/erros, lista de linhas com erro para download CSV

- Loading state: skeleton retangular 100% × 240px com pulse
- Estado vazio (sem movimentos): placeholder com ícone `TrendingUp` 48px + copy de empty state

## Accessibility Contract

- Todos os status badges incluem texto visível — nunca cor isolada
- Formulário multi-step: `aria-current="step"` no step ativo do stepper
- CPF/PIS com erro: `role="alert"` na mensagem de erro, `aria-describedby` no input
- Tabs do `EmployeeDetailPage`: padrão WAI-ARIA Tabs — `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected`, `aria-controls`
- Avatar sem foto: `aria-label="Foto não disponível"` na imagem placeholder
- Ícones decorativos (acompanham texto): `aria-hidden="true"`
- Ícones funcionais (ação isolada): `aria-label` obrigatório
- Botão "Desligar colaborador": `aria-label="Desligar colaborador [Nome]"` para distinguir múltiplos
- Upload de documento: `<input type="file" accept=".pdf,.jpg,.jpeg,.png">` com label associado
- Focus visible: `outline: 2px solid var(--color-primary-500); outline-offset: 2px` — nunca remover
- Escape fecha modals e dropdowns
- Tab order: breadcrumb → tabs → conteúdo da tab ativa → ações no rodapé
