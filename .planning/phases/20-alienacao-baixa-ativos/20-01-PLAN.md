**`createDisposal(ctx: RlsContext, assetId: string, input: CreateDisposalInput, userId: string): Promise<DisposalOutput>`**

- CRITICAL: Use `prisma.$transaction` directly (NOT withRlsContext) per STATE.md decision — avoids nested RLS deadlocks
- Inside transaction:
  a. Guard: find asset by id + organizationId + deletedAt null. 404 if not found. 409 if status === 'ALIENADO'.
  b. Validate: if VENDA, require saleValue > 0, buyerName, dueDate. If DESCARTE/SINISTRO/OBSOLESCENCIA, require motivation.
  c. Compute netBookValue: query `tx.depreciationEntry.findFirst({ where: { assetId, reversedAt: null }, orderBy: [{ periodYear: 'desc' }, { periodMonth: 'desc' }], select: { closingBookValue: true } })`. If found, use `new Decimal(closingBookValue)`. Else, use `new Decimal(asset.acquisitionValue ?? 0)`.
  d. Compute gainLoss: if VENDA, `new Decimal(input.saleValue).minus(netBookValue)`. Else, `netBookValue.negated()` (full loss).
  e. Bulk-cancel depreciation entries: `tx.depreciationEntry.updateMany({ where: { assetId, reversedAt: null }, data: { reversedAt: new Date(), notes: 'Cancelado por alienacao em ' + input.disposalDate } })` — store `count` as `cancelledDepreciationCount`.
  f. Update asset: `tx.asset.update({ where: { id: assetId }, data: { status: 'ALIENADO', disposalDate: new Date(input.disposalDate) } })`.
  g. Create AssetDisposal record with all fields.
  h. If VENDA and saleValue > 0 and dueDate present: generate installments using `generateInstallments(Money(saleValue), installmentCount ?? 1, new Date(firstDueDate ?? dueDate))` from `@protos-farm/shared`. Then `tx.receivable.create({ data: { organizationId: ctx.organizationId, farmId: asset.farmId, clientName: input.buyerName, category: 'ASSET_SALE', description: 'Venda ' + asset.assetTag + ' — ' + asset.name, totalAmount: new Decimal(saleValue).toDecimalPlaces(2), dueDate: installments[0].dueDate, installmentCount: installments.length, originType: 'ASSET_DISPOSAL', originId: disposal.id } })`.
