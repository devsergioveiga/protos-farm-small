2. Add to `financial-dashboard.service.ts` a new exported function:
   ```typescript
   export async function getPatrimonyDashboard(
     ctx: RlsContext,
     query: PatrimonyDashboardQuery,
   ): Promise<PatrimonyDashboardOutput>
   ```
   Implementation:
   - Compute monthStart and monthEnd from query.year/month (same pattern as getFinancialDashboard).
   - Build base where clause: `{ organizationId: ctx.organizationId, deletedAt: null }` + farmId filter if provided.
   - `totalActiveValue`: `prisma.asset.aggregate({ where: { ...base, status: { not: 'ALIENADO' } }, _sum: { acquisitionValue: true } })` — convert null to 0.
   - `accumulatedDepreciation`: `prisma.depreciationEntry.aggregate({ where: { organizationId: ctx.organizationId, reversedAt: null, ...(farmId ? through asset relation) }, _sum: { depreciationAmount: true } })`. If farmId filter needed, use raw query or join through asset. Simpler approach: query through asset relation with `asset: { farmId }`.
   - `netBookValue`: totalActiveValue - accumulatedDepreciation.
   - `acquisitionsInPeriod`: `prisma.asset.aggregate({ where: { ...base, acquisitionDate: { gte: monthStart, lt: monthEnd } }, _sum: { acquisitionValue: true }, _count: true })`.
   - `disposalsInPeriod`: `prisma.assetDisposal.aggregate({ where: { organizationId: ctx.organizationId, disposalDate: { gte: monthStart, lt: monthEnd }, ...(farmId ? { asset: { farmId } } : {}) }, _sum: { saleValue: true, gainLoss: true }, _count: true })`.
   - `assetCountByType`: `prisma.asset.groupBy({ by: ['assetType'], where: { ...base, status: { not: 'ALIENADO' } }, _count: true })`.
   - `assetCountByStatus`: `prisma.asset.groupBy({ by: ['status'], where: { ...base }, _count: true })`.
