    **Service** (`opening-balance.service.ts`):

    1. `getOpeningBalancePreview(organizationId, fiscalYearId)`:
       - Query 5 sources in parallel using `Promise.all`:
         a. Bank: `prisma.bankAccountBalance.findMany({ where: { bankAccount: { organizationId } }, include: { bankAccount: true } })` — filter `currentBalance > 0`, map to DEBIT lines. Find matching COA account by code pattern `1.1.01` or name containing "Caixa" or "Bancos".
         b. Payables: `prisma.payable.groupBy({ by: ['category'], where: { organizationId, status: { in: ['PENDING', 'PARTIAL', 'OVERDUE'] } }, _sum: { totalAmount: true } })` — map to CREDIT lines under Passivo accounts.
         c. Receivables: `prisma.receivable.groupBy({ by: ['category'], where: { organizationId, status: { in: ['PENDING', 'PARTIAL', 'OVERDUE'] } }, _sum: { totalAmount: true } })` — map to DEBIT lines under Ativo accounts.
         d. Assets: `prisma.asset.findMany({ where: { organizationId, status: 'ACTIVE' }, select: { acquisitionCost: true, accumulatedDepreciation: true } })` — compute `netBookValue = acquisitionCost - accumulatedDepreciation`, aggregate total, map to DEBIT line under 1.2.x Imobilizado.
         e. Provisions: `prisma.payrollProvision.findMany({ where: { organizationId }, select: { vacationProvision: true, thirteenthProvision: true } })` — sum each, map to CREDIT lines under 2.2.01 and 2.2.02 accounts.

       - For each source, look up the matching COA account by code prefix using `prisma.chartOfAccount.findFirst({ where: { organizationId, code: { startsWith: prefix }, isSynthetic: false, isActive: true } })`.
       - If no matching COA account found for a source, skip that line (don't error — wizard shows what's available).
       - Return array of `OpeningBalanceLinePreview`.

    2. `postOpeningBalance(organizationId, input, createdBy)`:
       - Guard: check no existing `OPENING_BALANCE` entry for this fiscal year:
         ```typescript
         const existing = await prisma.journalEntry.findFirst({
           where: { organizationId, entryType: 'OPENING_BALANCE' as const,
                    period: { fiscalYearId: input.fiscalYearId } },
         });
         if (existing) throw new OpeningBalanceError('Ja existe saldo de abertura para este exercicio fiscal', 'ALREADY_EXISTS', 409);
         ```
       - Compute contra-entry: sum all DEBIT amounts, sum all CREDIT amounts, difference goes to PL "Lucros e Prejuizos Acumulados" account (find by code starting with `3.` and name containing "Lucros" or "Prejuizos").
       - Build full lines array: user lines + contra-entry line.
       - **Create the JournalEntry directly via Prisma** (bypassing `createJournalEntryDraft` to set `entryType: 'OPENING_BALANCE'` directly):
         ```typescript
         const entry = await prisma.journalEntry.create({
           data: {
             organizationId,
             entryDate: periodStartDate,
             periodId: input.periodId,
             description: `Saldo de Abertura — Exercicio ${year}`,
             entryType: 'OPENING_BALANCE' as const,
             status: 'DRAFT' as const,
             entryNumber: 0,
             createdBy,
             lines: { create: allLines.map((l, i) => ({ ...l, lineOrder: i + 1 })) },
           },
         });
         ```
       - Then call `postJournalEntry(organizationId, entry.id, createdBy)` to assign number, update AccountBalance, and set status POSTED.
       - Return the posted entry.
