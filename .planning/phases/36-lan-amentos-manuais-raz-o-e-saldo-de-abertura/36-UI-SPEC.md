Declared values (multiples of 4px — base 4px scale per CLAUDE.md):

| Token | CSS Var | Value | Usage |
|-------|---------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, inline badge padding |
| sm | `--space-2` | 8px | Between form label and input, table cell padding-y |
| md | `--space-4` | 16px | Default element spacing, card padding-x, modal body padding |
| lg | `--space-6` | 24px | Section padding, modal header padding |
| xl | `--space-8` | 32px | Layout gaps between page sections |

- Journal entry line rows: 12px vertical padding (`--space-3`) to keep multi-line table compact without sacrificing readability

## Interaction Contracts

### JournalEntryModal — Line Table

- Minimum 2 lines on open (1 debit placeholder + 1 credit placeholder)
- Maximum no enforced limit — scroll within modal body
- "Adicionar linha" button appends a new empty row at the bottom
- DEBIT/CREDIT side toggle: segmented control (not dropdown) — two buttons side by side, 48px minimum touch target each
- Account picker: searchable combobox filtered to analytic accounts (`allowManualEntry = true`) — COA tree search by code or name
- Amount field: `JetBrains Mono` 14px, right-aligned, 2 decimal places, Brazilian numeric format (comma decimal separator)
- `BalanceIndicator` updates live on every amount/side change (no submit required)
- "Lançar" (primary CTA) disabled until `BalanceIndicator` shows balanced state
- Modal has header (heading + entry number/date) + scrollable body (line table) + sticky footer (balance summary + actions)
- Modal width: 800px on desktop, full-width on mobile (<768px)

### Opening Balance Wizard

- Step indicator in modal header: Step 1 of 2 — "Revisão" | Step 2 of 2 — "Contrapartida"
- Step 1: editable table of pre-populated lines (account + debit/credit amount); user can adjust amounts or add/remove lines
- Step 2: read-only summary of net difference + which PL account absorbs the contra-entry
- "Próximo" CTA on step 1 only enabled when step 1 lines are non-empty
- "Postar Saldo de Abertura" CTA on step 2 — primary, disabled on processing
- Back button on step 2 returns to step 1 without losing edits

### Ledger Page

- Account selector is required before any data loads — shows empty state prompt until selection
- Period selector defaults to current open fiscal period
- Running balance column: right-aligned, `JetBrains Mono`, positive in `--color-success-500` (or `--color-neutral-700` for neutral), negative in `--color-error-500`
- Drill-down: clicking any entry number row opens a read-only `JournalEntryDetailModal` (inline view of all lines)
- "Saldo Anterior" row at top of table, visually distinct (neutral-200 background, bold weight `--font-bold`)
- Export CSV and Export PDF are secondary buttons (no primary treatment)

### Trial Balance Page

- Period selector with fiscal year context
- Table: Account code | Account name | Saldo Anterior (D/C) | Movimento Débito | Movimento Crédito | Saldo Atual (D/C)
- Group rows (synthetic accounts) have bold text (`--font-bold`) and light neutral-100 background
- Total rows use DM Sans `--font-bold` with top border `--color-neutral-300`
- "Comparativo com período anterior" toggle: adds a second set of columns; off by default
- Balance validation bar: full-width strip above table, green if balanced, red if not — always visible

### Reversal Flow

- "Estornar" button visible only on POSTADO entries
- Opens `ReversalModal` (not inline)
- Reason textarea: required, min 10 chars, validation on blur
- "Confirmar Estorno" inside `ConfirmModal` variant=`danger` — two-click confirmation
- After confirmation: optimistic loading state on button, success toast on resolution

### Status Badge

- Non-interactive pill: 6px radius, font `--text-sm` `--font-bold`, ALL CAPS, icon 16px left of text
- Touch/click on badge does nothing (it is informational only)

## Accessibility

- All modals: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading ID
- Focus trap inside open modal; Escape closes modal (returns focus to trigger)
- Journal entry line table: `<table>` with `<th scope="col">` — never div-based layout
- Amount inputs: `aria-label="Valor linha {n}"`, type=`"text"` with inputmode=`"decimal"`
- DEBIT/CREDIT toggle: `role="group"` with `aria-label="Tipo"`, active button `aria-pressed="true"`
- BalanceIndicator: `role="status"` for live updates without focus interruption
- Status badges: `aria-label="Status: {status}"` on the containing span
- Account picker combobox: standard `aria-autocomplete="list"`, `aria-expanded` on trigger
- Error messages: `role="alert"` for form validation errors
- Skip to content link present in page layout (existing pattern)
- Focus visible: `outline: 2px solid var(--color-primary-500); outline-offset: 2px` — never removed
