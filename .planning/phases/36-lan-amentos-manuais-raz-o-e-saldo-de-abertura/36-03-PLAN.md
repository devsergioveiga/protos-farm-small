Running balance SQL (from 36-RESEARCH.md):
```sql
SELECT je.id, je.entry_number, je."entryDate", je.description,
  jel.side, jel.amount,
  SUM(CASE WHEN coa.nature = 'DEVEDORA'
    THEN CASE jel.side WHEN 'DEBIT' THEN jel.amount ELSE -jel.amount END
    ELSE CASE jel.side WHEN 'CREDIT' THEN jel.amount ELSE -jel.amount END
  END) OVER (ORDER BY je."entryDate", je.entry_number) AS running_balance
FROM journal_entry_lines jel
JOIN journal_entries je ON je.id = jel."journalEntryId"
JOIN chart_of_accounts coa ON coa.id = jel."accountId"
WHERE je."organizationId" = $1 AND jel."accountId" = $2
  AND je.status = 'POSTED' AND je."entryDate" BETWEEN $3 AND $4
ORDER BY je."entryDate", je.entry_number, jel."lineOrder";
```

    **Service** (`ledger.service.ts`):

    1. `getLedger(organizationId, filters: LedgerFilters)`:
       - Fetch account info: `prisma.chartOfAccount.findFirst({ where: { id: filters.accountId, organizationId } })`
       - Compute previousBalance: query `AccountBalance` for the month immediately before the start date, get `closingBalance`. If none exists, use `0`.
       - Execute raw SQL with window function (from RESEARCH.md Pattern 5):
         ```typescript
         const lines = await prisma.$queryRaw<LedgerLineRaw[]>`
           SELECT je.id AS "entryId", je."entry_number" AS "entryNumber",
             je."entry_date" AS "entryDate", je.description,
             jel.side, jel.amount::text,
             (${previousBalanceDecimal} + SUM(
               CASE WHEN ${accountNature} = 'DEVEDORA'
                 THEN CASE jel.side WHEN 'DEBIT' THEN jel.amount ELSE -jel.amount END
                 ELSE CASE jel.side WHEN 'CREDIT' THEN jel.amount ELSE -jel.amount END
               END
             ) OVER (ORDER BY je."entry_date", je."entry_number"))::text AS "runningBalance"
           FROM journal_entry_lines jel
           JOIN journal_entries je ON je.id = jel."journalEntryId"
           WHERE je."organization_id" = ${organizationId}
             AND jel."account_id" = ${filters.accountId}
             AND je.status = 'POSTED'
             AND je."entry_date" BETWEEN ${startDate}::date AND ${endDate}::date
           ORDER BY je."entry_date", je."entry_number", jel."line_order"
         `;
         ```
       - Return `LedgerOutput` with previousBalance, lines, and finalBalance (last line's runningBalance or previousBalance if no lines).

    2. `getTrialBalance(organizationId, filters: TrialBalanceFilters)`:
       - Fetch all active COA accounts for the org (both synthetic and analytic).
       - Fetch AccountBalance rows for the given fiscalYearId + month.
       - For each analytic account: previousBalance = openingBalance, debitMovement = debitTotal, creditMovement = creditTotal, currentBalance = closingBalance.
       - For synthetic accounts: aggregate children values recursively.
       - Sort by account code.
       - Compute grandTotals: sum all analytic-only accounts (avoid double-counting synthetics).
       - isBalanced: `grandTotals.movementDebit === grandTotals.movementCredit` (using Decimal comparison).
       - Return `TrialBalanceOutput`.

    3. `getDailyBook(organizationId, filters: DailyBookFilters)`:
       - Query: `prisma.journalEntry.findMany({ where: { organizationId, status: 'POSTED', entryDate: { gte, lte }, ...(entryType filter) }, include: { lines: { include: { account: { select: { code, name } } }, orderBy: { lineOrder: 'asc' } } }, orderBy: [{ entryDate: 'asc' }, { entryNumber: 'asc' }] })`.
       - Optional amount filter: if minAmount/maxAmount set, filter by `lines.some(l => amount >= min && amount <= max)` in service layer.
       - Return `DailyBookOutput`.

    4. Export functions (follow established patterns):
       - `exportLedgerCsv(organizationId, filters)`: Build CSV string with BOM, semicolon separator (Brazilian standard), headers "Data;Numero;Historico;Debito;Credito;Saldo". Return string.
       - `exportLedgerPdf(organizationId, filters, res)`: Use pdfkit, pipe to `res`. Header with company name + account code/name + period. Table rows with columns.
       - `exportTrialBalancePdf(organizationId, filters, res)`: pdfkit with 3-column trial balance table.
       - `exportTrialBalanceXlsx(organizationId, filters, res)`: ExcelJS workbook, single sheet "Balancete", header row bold, number format on decimal columns, auto-width.
       - `exportDailyBookPdf(organizationId, filters, res)`: pdfkit with "Termo de Abertura" header, entries grouped by date, "Termo de Encerramento" footer.
