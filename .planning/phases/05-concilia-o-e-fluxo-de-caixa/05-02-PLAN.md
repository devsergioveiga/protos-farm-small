<!-- Scoring algorithm from RESEARCH.md -->
Score calculation:
- Value exact match = 50 pts, within 1% = 30 pts, within 5% = 10 pts, else 0
- Date ±1 day = 40 pts, ±5 days = 20 pts, else 0
- Description substring match (first 8 chars) = 10 pts
- Total >= 95 → EXATO, >= 70 → PROVAVEL, else SEM_MATCH

Confidence levels per CONTEXT.md:

- EXATO (>= 95): valor identico + data +-1 dia
- PROVAVEL (70-94): valor proximo + data +-5 dias
- SEM_MATCH (< 70)

N:N reconciliation per CONTEXT.md:

- Sum of linked amounts must equal statement line amount exactly
- Use Money() for sum validation to avoid floating point errors

Add the following functions to `reconciliation.service.ts`:

1. `scoreMatch(statementLine: {amount, date, memo}, candidate: {amount, date, description}): number` — pure function implementing the scoring algorithm:
   - valueDiff = Math.abs(statementLine.amount - candidate.amount)
   - relDiff = candidate.amount !== 0 ? valueDiff / Math.abs(candidate.amount) : (valueDiff === 0 ? 0 : 1)
   - valueScore: exact=50, <1%=30, <5%=10, else=0
   - dayDiff = Math.abs((statementLine.date.getTime() - candidate.date.getTime()) / 86400000)
   - dateScore: <=1=40, <=5=20, else=0
   - descScore: first 8 chars substring match (case insensitive) = 10, else 0
   - Return valueScore + dateScore + descScore
     Export `toConfidence(score: number): ConfidenceLevel` — >=95 EXATO, >=70 PROVAVEL, else SEM_MATCH

2. `getImportLinesWithMatches(ctx, importId, statusFilter?): Promise<StatementLineWithMatches[]>` — for each PENDING line in the import:
   - Query open PayableInstallments (status PENDING) within date +-10 days and amount +-20% range
   - Query open ReceivableInstallments (status PENDING) within same window
   - Query Transfers within same window
   - Score each candidate, keep top 3 by score, attach confidence level
   - Return lines grouped: EXATO first, then PROVAVEL, then SEM_MATCH
   - Lines with status filter applied if provided (PENDING, RECONCILED, IGNORED)

3. `confirmReconciliation(ctx, statementLineId, reconciliationId): Promise<void>` — within withRlsContext:
   - Update Reconciliation: set confirmedBy=ctx.userId, confirmedAt=now()
   - Update BankStatementLine: set status=RECONCILED
   - If referenceType is PAYABLE: update Payable where id=referenceId set reconciled=true, reconciledAt=now()
   - If referenceType is RECEIVABLE: update Receivable where id=referenceId set reconciled=true, reconciledAt=now()
   - logAudit('reconciliation_confirmed', ...)

4. `rejectMatch(ctx, statementLineId, reconciliationId): Promise<void>` — delete Reconciliation record, keep line PENDING. logAudit.

5. `manualLink(ctx, statementLineId, links: Array<{referenceType, referenceId, amount}>): Promise<void>` — within withRlsContext:
   - Validate sum: `links.reduce((sum, l) => sum.add(Money(l.amount)), Money(0))` must equal `Money.fromPrismaDecimal(statementLine.amount)`. If not, throw ReconciliationError("Soma selecionada nao coincide com o valor do extrato", 400).
   - Create Reconciliation record for each link with score=100, confidence=EXATO, confirmedBy=ctx.userId, confirmedAt=now()
   - Update BankStatementLine status=RECONCILED
   - Update Payable/Receivable reconciled=true for each linked reference
   - logAudit

6. `ignoreStatementLine(ctx, statementLineId): Promise<void>` — update status=IGNORED, logAudit.

7. `searchCandidates(ctx, query: {search, bankAccountId}): Promise<MatchCandidate[]>` — search PayableInstallments + ReceivableInstallments + Transfers by description ILIKE or amount match. Return top 20 results with type, referenceId, description, amount, date. No scoring needed — these are manual search results.

8. `getReconciliationReport(ctx, importId): Promise<{summary, csvBuffer}>` — count lines by status (PENDING, RECONCILED, IGNORED), total amounts per status. Generate CSV with columns: Data, Tipo, Valor, Descricao, Status, Match. Return summary object + Buffer.
