1. Create `cashflow.service.ts`:

`getProjection(ctx, query: CashflowQuery): Promise<CashflowProjection>`:

- Build date range: today to today + 365 days
- Get current consolidated balance: within withRlsContext, aggregate BankAccountBalance.currentBalance (sum all accounts, or filter by farmId via BankAccount.farms junction if farmId provided). Use Money(0) as seed.
- Query open PayableInstallments: status PENDING, dueDate between today and endDate. Include parent Payable for category. Filter by farmId if provided.
- Query open ReceivableInstallments: status PENDING, dueDate between today and endDate. Include parent Receivable for category. Filter by farmId if provided.
- Query Checks with status A_COMPENSAR: expectedCompensationDate between today and endDate. Filter by farmId if provided.
- Also handle recurring CP/CR: if parent Payable/Receivable has recurrenceFrequency and recurrenceEndDate beyond last existing installment, project future amounts programmatically (generate virtual installments at recurrence interval up to 12 months or recurrenceEndDate, whichever comes first). Do NOT write these to DB.
- Build monthly buckets (Map<string, {inflows: Money, outflows: Money, checksPending: Money}>): key is YYYY-MM format
- For each PayableInstallment: add to outflows bucket at dueDate month. Map category to DFC class via PAYABLE_DFC_MAP.
- For each ReceivableInstallment: add to inflows bucket at dueDate month. Map category to DFC class via RECEIVABLE_DFC_MAP.
- For each Check EMITIDO A_COMPENSAR: add to outflows at expectedCompensationDate month.
- For each Check RECEBIDO A_COMPENSAR: add to inflows at expectedCompensationDate month.
- Compute running balance across 12 months: start from currentBalance.
- For each month: realistic = previous + inflows - outflows. Optimistic = previous + inflows*1.10 - outflows*0.95. Pessimistic = previous + inflows*0.90 - outflows*1.15. Use Money for all arithmetic.
- Build ProjectionPoint[] with label = month abbreviation in pt-BR ("Jan", "Fev", "Mar", etc.) + year suffix ("26")
- Detect negativeBalanceDate: first month where balanceRealistic < 0
- Build DfcSummary: aggregate by category and DFC class, compute monthly arrays and totals.
- Return CashflowProjection object.

`getNegativeBalanceAlert(ctx, farmId?): Promise<{date: string, amount: number} | null>`:

- Call getProjection internally, check negativeBalanceDate. Return {date, amount} or null.
