## Spacing Scale

Declared values (multiples of 4 only — project rule from CLAUDE.md):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip spacing |
| sm | 8px | Compact element spacing, table cell padding |
| md | 16px | Default element spacing, form field gap, card padding |
| lg | 24px | Section padding, page content padding |
| xl | 32px | Layout gaps, modal padding |
| 2xl | 48px | Major section breaks, empty state vertical spacing |

Exceptions:
- Touch targets: minimum 48x48px for all interactive elements (buttons, tabs, row actions)

## Pages and Layout

Four new pages follow the established HR payroll pattern (PayrollRunsPage pattern):

### VacationSchedulesPage

- URL: `/vacation-schedules`
- Breadcrumb: RH > Férias
- Primary visual anchor: the "Agendar Férias" CTA button in the page header
- Layout: two tabs — "Períodos Aquisitivos" | "Agendamentos"
- Tab "Períodos Aquisitivos": table with columns — Colaborador, Início Aquisição, Fim Aquisição, Dias Ganhos, Dias Gozados, Saldo, Status, Ações
- Tab "Agendamentos": table with columns — Colaborador, Período, Data Início, Data Fim, Dias, Abono, Valor Líquido, Pagamento Até, Status, Ações
- Primary CTA: "Agendar Férias" — opens VacationScheduleModal
- Vacation period status badges: ACCRUING (neutral), AVAILABLE (info-blue), SCHEDULED (warning-amber), EXPIRED (error-red)
- Schedule status badges: SCHEDULED (info-blue), PAID (success-green), CANCELLED (neutral-gray)
- Alert banner: yellow warning for periods within 60 days of doubling expiry — icon TriangleAlert + text "Férias vencerão em X dias. Programe antes de [data]."
- Calendar view (secondary): month calendar showing scheduled vacation periods per employee to avoid safra conflicts — accessible as a second view toggle within the tab

### EmployeeAbsencesPage

- URL: `/employee-absences`
- Breadcrumb: RH > Afastamentos
- Primary visual anchor: the "Registrar Afastamento" CTA button in the page header
- Layout: single list with filter bar (type, status, date range, employee search)
- Table columns: Colaborador, Tipo, Data Início, Data Fim, Dias, CAT, Impacto Folha, Status Retorno, Ações
- Primary CTA: "Registrar Afastamento" — opens EmployeeAbsenceModal
- Absence type labels (pt-BR): Atestado Médico, Afastamento INSS, Acidente de Trabalho, Licença-Maternidade, Licença-Paternidade, Casamento, Falecimento, Serviço Militar, Outro
- Info banner for WORK_ACCIDENT with CAT: icon AlertTriangle + "CAT registrada — estabilidade provisória até [data]" in info-blue
- Info banner for INSS_LEAVE: "A partir do 16º dia o INSS assume o pagamento. Notifique o colaborador."
- ASO required indicator: badge "ASO Obrigatório" in warning-amber beside return date when asoRequired=true

### EmployeeTerminationsPage

- URL: `/employee-terminations`
- Breadcrumb: RH > Rescisões
- Primary visual anchor: the "Iniciar Rescisão" CTA button in the page header
- Layout: single list — terminated employees, filterable by type and status
- Table columns: Colaborador, Tipo Rescisão, Data Rescisão, Aviso Prévio, Saldo Salário, Total Bruto, Total Líquido, Prazo Pagamento, Status, Ações
- Primary CTA: "Iniciar Rescisão" — opens EmployeeTerminationModal
- Termination type labels (pt-BR): Sem Justa Causa, Justa Causa, Pedido de Demissão, Fim de Safra, Acordo Mútuo
- Status badges: DRAFT (neutral), PROCESSED (info-blue), PAID (success-green)
- Payment deadline alert: warning badge "Pagar até [data]" in warning-amber when within 3 days; error-red when past deadline
- Destructive action: rescisão confirmation uses ConfirmModal with variant="danger" (medium criticality — not full ConfirmDeleteModal with name typing, since it is a process start, not an irreversible delete)
- PDF download actions: TRCT PDF + GRRF guia — secondary buttons with FileText icon + Download icon (20px)

### PayrollProvisionsPage

- URL: `/payroll-provisions`
- Breadcrumb: RH > Provisões
- Primary visual anchor: the "Calcular Provisões" CTA button in the page header
- Layout: two tabs — "Provisões do Mês" | "Relatório de Posição"
- Tab "Provisões do Mês": table grouped by employee, columns — Colaborador, Mês Referência, Tipo, Salário Base, Provisão, Encargos, Total, Centro de Custo, Ações
- Tab "Relatório de Posição": summary totals by cost center and provision type — totals table + breakdown
- Primary CTA: "Calcular Provisões" — triggers batch calculation for selected month
- Provision type labels: Férias, Décimo Terceiro
- Provision reversal (FERIAS-04): secondary button "Estornar" with ConfirmModal variant="warning" (medium criticality — reversible but significant)
- Monetary columns: always right-aligned, JetBrains Mono, formatted in BRL (pt-BR locale)

## Component Inventory

### New Modals (all follow existing pattern: isOpen, onClose, onSuccess props)

| Modal | Trigger | Criticality | Confirmation Pattern |
|-------|---------|-------------|---------------------|
| VacationScheduleModal | "Agendar Férias" button | Low | Standard submit button |
| EmployeeAbsenceModal | "Registrar Afastamento" button | Low | Standard submit button |
| EmployeeTerminationModal | "Iniciar Rescisão" button | Medium | ConfirmModal variant="danger" before submitting |
| ProvisionReversalConfirm | "Estornar" row action | Medium | ConfirmModal variant="warning" |

### VacationScheduleModal — Step Flow (multi-step, ≤3 steps)

Step 1 — Selecionar Período: employee selector, acquisitive period selector (shows available periods only), fraction selector (up to 3 fractions)
Step 2 — Definir Datas: start date, total days (min 5), abono pecuniário toggle (sells 10 days, adds to payment)
Step 3 — Conferir Cálculo: read-only breakdown — salário base, 1/3 constitucional, abono, INSS, IRRF, líquido, FGTS, data limite de pagamento

### EmployeeTerminationModal — Step Flow

Step 1 — Dados da Rescisão: employee (only ACTIVE status), termination type selector, termination date, notice period type
Step 2 — Conferir Cálculo: read-only breakdown — saldo salário, aviso prévio, 13º proporcional, férias vencidas, férias proporcionais, 1/3 férias, multa FGTS, INSS, IRRF, total líquido
Step 3 — Confirmar e Gerar TRCT: ConfirmModal danger inline + generate TRCT PDF action

### Status Badges — Unified Specification

```
ACCRUING    → neutral gray    bg:--color-neutral-200  text:--color-neutral-700  label:"AGUARDANDO"
AVAILABLE   → info-blue       bg:--color-info-100     text:--color-info-500     label:"DISPONÍVEL"
SCHEDULED   → warning-amber   bg:--color-warning-100  text:--color-warning-500  label:"AGENDADO"
EXPIRED     → error-red       bg:--color-error-100    text:--color-error-500    label:"VENCIDO"
PAID        → success-green   bg:--color-success-100  text:--color-success-500  label:"PAGO"
CANCELLED   → neutral gray    bg:--color-neutral-200  text:--color-neutral-600  label:"CANCELADO"
DRAFT       → neutral gray    bg:--color-neutral-200  text:--color-neutral-700  label:"RASCUNHO"
PROCESSED   → info-blue       bg:--color-info-100     text:--color-info-500     label:"PROCESSADO"
AFASTADO    → warning-amber   bg:--color-warning-100  text:--color-warning-500  label:"AFASTADO"
```

Badge anatomy: 4px border-radius, 4px horizontal padding, 12px font, 600 weight, ALL CAPS, always paired with context text in row.

### Skeleton Loading Pattern

All four pages use skeleton rows during data fetch — 5 skeleton rows each, matching the existing PayrollRunsPage SkeletonRow component pattern. Never show spinner full-page.

## Interaction Contracts

### Monetary Display (all pages)

- All monetary values: `toLocaleString('pt-BR', { style: 'currency', currency: 'BRL' })` — matches existing formatCurrency in PayrollRunsPage
- Font: JetBrains Mono (--font-mono), 14px, weight 400
- Alignment: right-aligned in table columns
- Null/missing value: render em dash "—" (not zero, not empty)

### Date Display

- All dates: `toLocaleDateString('pt-BR')` — DD/MM/YYYY format
- Reference months: "Mar/2026" abbreviation format (matches getMonthLabel in PayrollRunsPage)
- Payment deadlines: always show full date (DD/MM/YYYY), not relative

### Vacation Calendar Visual

- Month grid view within VacationSchedulesPage agendamentos tab
- Toggle button "Ver Calendário" / "Ver Lista" — secondary button, no primary style
- Calendar cells: colored band per employee when on vacation — color derived from employee index (neutral palette, not primary green — avoids false CTA signal)
- Safra conflict detection: cells with planned safra operations show striped pattern in earth-tone (#8D6E63)
- No drag-and-drop on calendar (out of scope for this phase)

### Termination Calculation Preview (Step 2 of EmployeeTerminationModal)

- Read-only breakdown table: two columns (Rubrica | Valor)
- Rows: Saldo de Salário, Aviso Prévio, 13º Proporcional, Férias Vencidas + 1/3, Férias Proporcionais + 1/3, Multa FGTS 40% (or 20% or 0%), Total Bruto, INSS, IRRF, Total Líquido
- Highlighted row for Total Líquido: bold, slightly larger (16px), border-top separator
- FGTS penalty label adapts to termination type: "Multa FGTS 40%" / "Multa FGTS 20%" / "Sem multa FGTS"
- All amounts in JetBrains Mono

### Absence Payroll Impact Badge

- Compact inline badge beside employee name in list: "–3 dias" in error-red for deducted days, "INSS paga" in info-blue when responsibility transfers to INSS
- Badge uses icon + text — never color alone

### Filter Bar (all four pages)

- Employee search: text input, min 2 chars before triggering filter, debounce 300ms
- Date range: two date inputs (De / Até) — 48px minimum height
- Status filter: select dropdown
- Type filter (absences/terminations): select dropdown
- Month selector (provisions): month/year selector — select with year + month options
- All filters apply instantly (no submit button needed)
- Active filter count badge on "Filtros" label when any filter is non-default

## Accessibility Contract

- All tables: `<table>` with `<th scope="col">` and `<caption>` (visually hidden captions acceptable)
- Modal focus: focus traps within modal when open, returns to trigger element on close
- Escape key: closes all modals and dropdowns
- Status badges: `aria-label` with full description, e.g. `aria-label="Status: Período disponível para agendamento"`
- PDF download buttons: `aria-label="Baixar TRCT de [nome do colaborador]"`
- Calendar cells: `aria-label="[data]: [nome do colaborador] em férias"`
- Alert banners: `role="alert"` for time-sensitive warnings (expiry within 60 days)
- Form errors: `role="alert"` + `aria-live="polite"` on error containers
- Required fields: `aria-required="true"` + asterisk (*) in visible label
- Focus visible: 2px outline using `var(--color-primary-500)` — never removed
