# Phase 3 — UI Design Contract: Dashboard Financeiro

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property          | Value                                                                           |
| ----------------- | ------------------------------------------------------------------------------- |
| Tool              | none (CSS modules + CSS custom properties)                                      |
| Preset            | not applicable                                                                  |
| Component library | none (project-native components)                                                |
| Icon library      | lucide-react (existing project standard)                                        |
| Font              | DM Sans (headings) + Source Sans 3 (body/UI) + JetBrains Mono (monetary values) |

**Rationale:** No shadcn or Tailwind in this project. Styling uses CSS modules (one `{Page}.css` per page) with CSS custom properties from `apps/frontend/src/styles/tokens.css`. This matches the existing pattern for all financial pages (PayablesPage.css, BankAccountsPage.css, MilkDashboardPage.css).

---

## Visual Focal Point

**Primary visual anchor:** The 4-card KPI grid at the top of the page — specifically the KPI values in JetBrains Mono 28px bold — is the primary entry point for the user's eye and conveys the most critical financial position at a glance.

---

## Spacing Scale

Declared values (multiples of 4 only — from `--space-*` tokens in tokens.css):

| Token | Value               | Usage in This Phase                                            |
| ----- | ------------------- | -------------------------------------------------------------- |
| xs    | 4px (`--space-1`)   | Icon gap within KPI card, YoY badge padding                    |
| sm    | 8px (`--space-2`)   | Gap between rank badge and label in Top 5 rows, alert icon gap |
| md    | 16px (`--space-4`)  | Default gap between chart header and chart, filter bar gap     |
| lg    | 24px (`--space-6`)  | Page padding, section margin-bottom, KPI card internal padding |
| xl    | 32px (`--space-8`)  | Gap between KPI grid and charts section                        |
| 2xl   | 48px (`--space-12`) | Major section breaks (KPIs → charts → top-5 → alerts)          |
| 3xl   | 64px (`--space-16`) | Not used in this phase                                         |

Exceptions:

- Filter bar gap: 12px (between farm dropdown and period dropdown) — matches MilkDashboardPage established pattern
- KPI grid gap: 16px on mobile, 24px on desktop ≥768px
- Progress bar height in Top 5 cards: 4px (decorative, not spacing)

Touch targets: all interactive elements (dropdowns, buttons) minimum 48×48px.
Input padding: 12px vertical / 16px horizontal on filter selects.

---

## Typography

All values derived from `tokens.css`. Fonts: DM Sans (headings), Source Sans 3 (body/UI), JetBrains Mono (monetary data).

Declared scale: **4 sizes** × **2 weights**.

| Role            | Size                | Weight                 | Line Height              | Font                                                  | Usage                                                                                                          |
| --------------- | ------------------- | ---------------------- | ------------------------ | ----------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| KPI display     | 28px (`--text-2xl`) | 700 (`--font-bold`)    | 1.2 (`--leading-tight`)  | JetBrains Mono                                        | Monetary KPI amount (R$ 1.234,56); page `<h1>` merged here                                                     |
| Section heading | 18px (`--text-md`)  | 700 (`--font-bold`)    | 1.3 (`--leading-snug`)   | DM Sans                                               | Chart titles ("Receitas vs Despesas", "Top 5 Categorias")                                                      |
| Body / labels   | 14px (`--text-sm`)  | 400 (`--font-regular`) | 1.5 (`--leading-normal`) | Source Sans 3                                         | KPI labels, alert text, filter labels, page subtitle, button/CTA, rank numbers, Top 5 rows, YoY badges, legend |
| Chart axis      | 12px (`--text-xs`)  | 400 (`--font-regular`) | —                        | Source Sans 3 (XAxis) / JetBrains Mono (YAxis values) | Recharts tick labels                                                                                           |

Weight mapping decisions:

- 500 → 400 (section heading weight upgraded to 700 for contrast; other 500 usages collapsed to 400)
- 600 → 700 (KPI labels, YoY badges, rank numbers)

Minimum size enforced: 12px (tokens.css `--text-xs`). Nothing smaller.
ALL CAPS usage: only short labels like "SALDO REAL" / "SALDO CONTÁBIL" badges on the Saldo KPI card — never full sentences.

## Component Inventory

### KPI Cards Grid

4 cards in a 2×2 grid on mobile, 4-column row on desktop (≥768px).

| KPI                  | Icon                    | Label                                                        | Value Format                                 | YoY Badge             |
| -------------------- | ----------------------- | ------------------------------------------------------------ | -------------------------------------------- | --------------------- |
| Saldo bancário total | `Building2` (lucide)    | "Saldo bancário real"                                        | `R$ 1.234.567,89` (JetBrains Mono)           | ↑/↓/— vs ano anterior |
| CP próximos 30 dias  | `TrendingDown` (lucide) | "A pagar — próximos 30 dias"                                 | `R$ 45.230,00`                               | ↑/↓/—                 |
| CR próximos 30 dias  | `TrendingUp` (lucide)   | "A receber — próximos 30 dias"                               | `R$ 78.900,00`                               | ↑/↓/—                 |
| Resultado do mês     | `BarChart2` (lucide)    | "Resultado do mês" (or "do trimestre" when quarter selected) | `R$ 33.670,00` positive green / negative red | ↑/↓/—                 |

KPI card anatomy:

- Icon in 32×32 circle with `--color-primary-50` background, icon `--color-primary-600`, 20px icon size
- Label: Source Sans 3 14px regular (400), `--color-neutral-500`
- Value: JetBrains Mono 28px bold (700), `--color-neutral-800`; if negative use `--color-error-500`
- YoY badge below value: Source Sans 3 12px regular (400), color per table above
- Card: white bg, 1px `--color-neutral-200` border, `--radius-lg` (12px), `--shadow-sm`, 24px padding

**Saldo card label distinction:** Two micro-labels below the main value:

- `SALDO REAL` badge (var(--color-primary-100) bg, var(--color-primary-700) text, 10px uppercase)
- Rationale: success criterion 5 mandates never mixing saldo bancário real with saldo contábil.

### Filters Bar

Positioned between page header and KPI grid. Left-aligned row.

| Filter  | Type       | Default             | Width     |
| ------- | ---------- | ------------------- | --------- |
| Fazenda | `<select>` | "Todas as fazendas" | 200px min |
| Período | `<select>` | "Mês atual"         | 180px min |

Period options:

```
Mês atual
Mês anterior
Último trimestre
```

Both selects: `<select>` native element (not custom dropdown) — consistent with MilkDashboardPage. Style: Source Sans 3 14px regular (400), 12px vertical / 16px horizontal padding, 1px `--color-neutral-200` border, `--radius-md` (8px), `--color-neutral-0` background, min-height 48px.

### BarChart — Receitas vs Despesas

Container: white card, 24px padding, `--shadow-sm`, `--radius-lg`.
Recharts `ResponsiveContainer` width="100%" height={240} on desktop, height={200} on mobile (<768px).
Two bars per month: Receitas (`--color-primary-600`) + Despesas (`--color-neutral-400`).
XAxis: month name abbreviation (e.g. "Jan", "Fev"), Source Sans 3 12px regular (400).
YAxis: BRL compact (e.g. "50k", "100k"), JetBrains Mono 12px regular (400).
Tooltip: Source Sans 3 14px regular (400), formatted as `R$ 1.234,56`.
Legend: below chart, Source Sans 3 12px regular (400).
No onClick handler — read-only.
`aria-label` on the container div: "Gráfico de barras: receitas e despesas por mês".
Lazy import via inner component `RevenueExpenseChart.tsx` (do not import recharts at page level).

### PieChart — Top 5 Categorias de Despesa

Container: white card, 24px padding, same style as BarChart card.
Recharts `PieChart` width/height: 200px on desktop, 160px on mobile.
5 slices with palette: primary-400, sun-500, sky-500, earth-500, neutral-400 (then gray if <5 categories).
Labels: percentage inside slice if ≥10%, otherwise in legend.
Legend: below chart on both desktop and mobile (never beside on mobile).
`aria-label` on container: "Gráfico de pizza: top 5 categorias de despesa".
No onClick — read-only.
Lazy import via inner component `TopCategoriesChart.tsx`.

### Top 5 Ranked Lists

Two side-by-side panels on desktop (≥768px), stacked on mobile:

- Left: "Top 5 despesas por categoria"
- Right: "Top 5 receitas por cliente"

Each row:

- Rank badge: DM Sans 14px bold (700), `--color-neutral-700`, min-width 28px
- Label: Source Sans 3 14px regular (400), `--color-neutral-700`
- Value: JetBrains Mono 14px regular (400), `--color-neutral-800`, right-aligned
- Progress bar: full width, height 4px, `--color-neutral-100` bg + `--color-primary-400` fill at `relativePercent`%
- Row gap: 12px vertical

No interactions — read-only list.

### Alerts Panel

Single card below the Top 5 section.

Three states:

1. **No alerts:** CheckCircle icon (`--color-success-500`, 24px) + "Nenhum alerta financeiro no momento" (Source Sans 3 14px regular, `--color-neutral-600`)

2. **Overdue CP:** AlertCircle icon (`--color-error-500`, 20px) + text pattern:
   "N conta(s) a pagar vencida(s), totalizando R$ X,XX" — Source Sans 3 14px, `--color-neutral-700`; N in bold.

3. **Projected negative balance:** AlertTriangle icon (`--color-warning-500`, 20px) + text:
   "O saldo bancário pode ficar negativo nos próximos 30 dias." — Source Sans 3 14px, `--color-neutral-700`

Multiple alerts stack vertically with 8px gap between rows. Each alert row has a 4px left border matching the icon color.

### Loading State

Skeleton screens — no full-page spinner.

| Element            | Skeleton                        |
| ------------------ | ------------------------------- |
| KPI card           | Rect 100%×96px, `--radius-lg`   |
| BarChart container | Rect 100%×288px, `--radius-lg`  |
| PieChart container | Rect 100%×248px, `--radius-lg`  |
| Top 5 panel        | 5 rects 100%×32px with 12px gap |

Skeleton pulse: opacity 0.4 → 0.7, 1.5s infinite, `ease-in-out`. Honor `prefers-reduced-motion: reduce` (freeze at 0.6).

### Error State

Full-width card with:

- `AlertCircle` icon 48px, `--color-error-500`
- Heading: "Não foi possível carregar o dashboard" — DM Sans 18px bold (700)
- Body: "Verifique sua conexão e tente novamente." — Source Sans 3 14px regular (400), `--color-neutral-600`
- Button: "Tentar novamente" — secondary style (neutral border, 48px height)

### Endividamento Placeholder

Not a 5th KPI card. A small info banner below the KPI grid:

- `Info` icon (`--color-info-500`, 16px) + "Endividamento: crédito rural disponível na Fase 6" — Source Sans 3 12px regular (400), `--color-neutral-500`
- Subtle appearance: `--color-info-100` background, 8px padding, `--radius-md`

## Layout Structure

```
<main class="fin-dashboard">

  <!-- Header -->
  <header class="fin-dashboard__header">
    <div>
      <h1>Dashboard Financeiro</h1>
      <p>Posição financeira consolidada da fazenda</p>
    </div>
  </header>

  <!-- Filters bar -->
  <section class="fin-dashboard__filters" aria-label="Filtros do dashboard">
    <label htmlFor="farm-filter">Fazenda</label>
    <select id="farm-filter">...</select>
    <label htmlFor="period-filter">Período</label>
    <select id="period-filter">...</select>
  </section>

  <!-- KPI grid (4 cards) -->
  <section class="fin-dashboard__kpis" aria-label="Indicadores financeiros">
    <article class="fin-dashboard__kpi-card"> ... </article>  <!-- Saldo -->
    <article class="fin-dashboard__kpi-card"> ... </article>  <!-- CP 30d -->
    <article class="fin-dashboard__kpi-card"> ... </article>  <!-- CR 30d -->
    <article class="fin-dashboard__kpi-card"> ... </article>  <!-- Resultado -->
  </section>

  <!-- Endividamento placeholder banner -->
  <div class="fin-dashboard__debt-placeholder" role="note"> ... </div>

  <!-- Charts row -->
  <section class="fin-dashboard__charts" aria-label="Gráficos financeiros">
    <div class="fin-dashboard__chart-card"> <!-- BarChart --> </div>
    <div class="fin-dashboard__chart-card"> <!-- PieChart --> </div>
  </section>

  <!-- Top 5 ranked lists -->
  <section class="fin-dashboard__top5" aria-label="Ranking financeiro">
    <div class="fin-dashboard__top5-panel"> <!-- Top 5 despesas --> </div>
    <div class="fin-dashboard__top5-panel"> <!-- Top 5 receitas --> </div>
  </section>

  <!-- Alerts -->
  <section class="fin-dashboard__alerts" aria-label="Alertas financeiros">
    ...
  </section>

</main>
```

**Grid breakpoints (mobile-first):**

- Default (mobile): single column
- ≥768px: KPI grid → 2×2; charts → side by side; Top 5 → side by side
- ≥1024px: KPI grid → 4 columns inline

## Animation

| Interaction                  | Duration                           | Curve       | Token                             |
| ---------------------------- | ---------------------------------- | ----------- | --------------------------------- |
| Card hover (box-shadow)      | 200ms                              | ease-out    | `--duration-fast`, `--ease-out`   |
| Page skeleton → content fade | 300ms                              | ease-out    | `--duration-normal`, `--ease-out` |
| Filter dropdown open         | 200ms                              | ease-out    | `--duration-fast`, `--ease-out`   |
| Recharts bar entrance        | Default recharts animation (500ms) | ease-out    | let recharts handle               |
| Skeleton pulse               | 1500ms infinite                    | ease-in-out | —                                 |

Card hover:

```css
.fin-dashboard__kpi-card:hover {
  box-shadow: var(--shadow-lg);
  transition: box-shadow 200ms var(--ease-out);
}
```

Never animate: text content changes, layout reflows, scroll.
`prefers-reduced-motion: reduce` → disable all transitions and animations (handled by existing global rule in tokens.css).

## Accessibility Checklist

- [ ] `<main>`, `<header>`, `<section>` with `aria-label` on all landmark sections
- [ ] All `<select>` elements have visible `<label htmlFor>` (not placeholder-only)
- [ ] KPI cards use `<article>` with descriptive title; value paired with unit in `aria-label`
- [ ] YoY badges: `aria-label="Variação de +12,4% vs ano anterior"` (not just visual arrow)
- [ ] Recharts containers: outer `<div aria-label="...">` wrapping `ResponsiveContainer`
- [ ] Chart colors never sole indicator: legend always present
- [ ] `:focus-visible` outline: 2px solid `--color-primary-500`, offset 2px (global rule)
- [ ] Skeleton pulse respects `prefers-reduced-motion` (global rule in tokens.css)
- [ ] Alert text uses `role="status"` for live updates when re-filtering
- [ ] Empty state has visible CTA button (not div with onClick)
