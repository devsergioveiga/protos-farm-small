**financial-dashboard.service.ts** — Create `getFinancialDashboard(ctx: RlsContext, query: FinancialDashboardQuery): Promise<FinancialDashboardOutput>`:

1. Use `withRlsContext(ctx, async (tx) => { ... })` pattern from bank-accounts.service.ts.
2. Use `new Date(Date.UTC(year, month - 1, 1))` for month boundaries (Phase 2 decision: UTC dates always).
3. Compute `today` and `in30Days = new Date(today.getTime() + 30 * 86400000)`.
4. Compute `prevYear = year - 1`, `prevMonthStart`, `prevMonthEnd` the same way.

**KPI 1 — totalBankBalance:**

- Query `BankAccountBalance` where account `isActive = true` and (if farmId provided, join through `BankAccountFarm` where `farmId = query.farmId`).
- Sum `currentBalance` using `Money(0)` accumulator with `Money.fromPrismaDecimal()`.
- For prevYear: query the same but use statement entries to reconstruct balance at prevMonthEnd — OR return `null` if the implementation is complex. Simpler approach: count total settled CP/CR in prev year same month to compute a delta. If no data at all, return `null`.
- **CRITICAL: totalBankBalance must ONLY be BankAccountBalance.currentBalance. Never add pending CP/CR.**

**KPI 2 — payablesDue30d:**

- Query `PayableInstallment` where `status IN ('PENDING', 'OVERDUE')` AND `dueDate <= in30Days` AND `dueDate >= today`.
- If farmId provided, join through `Payable.farmId`.
- Sum `amount` using Money accumulator.
- For prevYear: same query but with `prevYear` equivalent date range. Return `null` if no records.

**KPI 3 — receivablesDue30d:**

- Same pattern as KPI 2 but on `ReceivableInstallment` where `status = 'PENDING'` AND `dueDate <= in30Days` AND `dueDate >= today`.

**KPI 4 — monthResult:**

- Settled CP in month: `PayableInstallment` where `status = 'PAID'` AND `paidAt >= monthStart` AND `paidAt <= monthEnd`. Sum `amountPaid`.
- Settled CR in month: `ReceivableInstallment` where `status = 'RECEIVED'` AND `receivedAt >= monthStart` AND `receivedAt <= monthEnd`. Sum `receivedAmount`.
- `monthResult = totalReceivedInMonth - totalPaidInMonth`.
- For prevYear: same query with prevMonthStart/prevMonthEnd. Return `null` if both sums are 0 and no records exist.

**YoY computation:** For each KPI, if prevYear value is `null`, return `null`. The frontend handles display of "—". Do NOT compute percentage server-side — return raw values and let the frontend compute `((current - prev) / prev * 100)` for display.

**monthlyTrend:** Query last 6 months (from the selected month going back). For each month:

- revenues = sum of `ReceivableInstallment.receivedAmount` where `status = 'RECEIVED'` and `receivedAt` in that month
- expenses = sum of `PayableInstallment.amountPaid` where `status = 'PAID'` and `paidAt` in that month
- Return as `{ yearMonth: "YYYY-MM", revenues: number, expenses: number }`
- Apply farmId filter if provided.

**topExpenseCategories:** Group `Payable` by `category` for selected month, sum `totalAmount`, take top 5, compute percentage of total. Use the Payable category enum labels for `categoryLabel`.

**topPayablesByCategory:** Same grouping as topExpenseCategories but with `rank` (1-5) and `relativePercent` (% relative to rank 1).

**topReceivablesByClient:** Group `Receivable` by `clientName` for selected month, sum `totalAmount`, take top 5, add rank and relativePercent.

**alerts:**

- `overduePayablesCount`: Count of `PayableInstallment` where `status = 'OVERDUE'` (with farmId filter).
- `overduePayablesTotal`: Sum of `amount` for those overdue installments.
- `projectedBalanceNegative`: `totalBankBalance - payablesDue30d < 0`.

**financial-dashboard.routes.ts:**

```typescript
export const financialDashboardRouter = Router();

financialDashboardRouter.get(
  '/org/financial-dashboard',
  authenticate,
  checkPermission({ module: 'financial', action: 'read' }),
  async (req, res) => {
    try {
      const ctx = buildRlsContext(req);
      const now = new Date();
      const query: FinancialDashboardQuery = {
        farmId: req.query.farmId as string | undefined,
        year: req.query.year ? parseInt(req.query.year as string, 10) : now.getFullYear(),
        month: req.query.month ? parseInt(req.query.month as string, 10) : now.getMonth() + 1,
      };
      const result = await getFinancialDashboard(ctx, query);
      res.json(result);
    } catch (err) {
      handleError(err, res);
    }
  },
);
```

Use the `buildRlsContext` and `handleError` helpers following the exact pattern from bank-accounts.routes.ts.
