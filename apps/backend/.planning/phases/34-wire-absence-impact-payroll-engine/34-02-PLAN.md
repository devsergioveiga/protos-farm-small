    **First location (~line 979-996):** Add `fgtsBase` field. The `result` object comes from `calculateEmployeePayroll`, but in this context we are reading from `item` (PayrollRunItem from DB). The `fgtsBase` value needs to be stored in the DB item or re-derived. Since the PayrollRunItem schema does not have a `fgtsBase` column, compute it from available data:

    Actually, looking at the code more carefully — the PDF generation reads from `item` which is a PayrollRunItem with `lineItemsJson`, `grossSalary`, `fgtsAmount` etc. We need `fgtsBase` to be derivable. Two approaches:

    - Store `fgtsBase` in lineItemsJson metadata (adds complexity)
    - Compute: if `fgtsAmount / 0.08` gives us the base (since FGTS is always 8%)

    Use the second approach — it is mathematically exact and avoids schema changes:

    ```typescript
    fgtsBase: new Decimal(item.fgtsAmount.toString()).div('0.08').toDecimalPlaces(2).toNumber(),
    ```

    Apply this change at BOTH `generatePayslipPdf` call sites (~line 996 and ~line 1463). Replace:
    ```typescript
    fgtsMonth: new Decimal(item.fgtsAmount.toString()).toNumber(),
    ```
    With:
    ```typescript
    fgtsMonth: new Decimal(item.fgtsAmount.toString()).toNumber(),
    fgtsBase: new Decimal(item.fgtsAmount.toString()).div('0.08').toDecimalPlaces(2).toNumber(),
    ```

    Handle edge case where fgtsAmount is zero (avoid division by zero):
    ```typescript
    fgtsBase: new Decimal(item.fgtsAmount.toString()).isZero()
      ? 0
      : new Decimal(item.fgtsAmount.toString()).div('0.08').toDecimalPlaces(2).toNumber(),
    ```
